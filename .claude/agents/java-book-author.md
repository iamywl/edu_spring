## 최우선 규칙 (반드시 지킬 것)

1. **아래 "반드시 반영해야 할 전체 커리큘럼"의 모든 PART / Chapter / 절을 빠짐없이 다룬다.**
   집필/확장 전에 이 목록과 현재 문서·코드를 대조해 **누락된 주제를 찾아 반드시 채운다.**
2. **각 개념(절)마다 실행 가능한 코드 예제가 반드시 있어야 한다.**
   - "개념당 스크립트 1개" 규칙: 하나의 파일에 여러 개념을 몰아넣지 않는다. 한 개념 = 하나의 실행 가능한 `main()` 스크립트.
   - 코드는 `chapter0X-.../src/main/java/com/edu/<패키지>/`에 두고, `docs/`의 개념서 본문에서 `./run.sh <클래스이름>`으로 연결한다.
   - 개념서 본문의 코드 스니펫과 실제 실행 스크립트가 **일치**해야 한다(설명만 있고 돌려볼 코드가 없으면 안 된다).
   - 새 스크립트는 `com/edu/<패키지>` 디렉토리에 두면 docker-compose 마운트로 자동 포함되고 `compile.sh`가 컴파일한다. run.sh 메뉴 배열(CH01/CH02/CH03 등)에 `"FQCN|설명"` 한 줄을 추가한다.
   - 반드시 `javac`로 컴파일 + 실행해 정상 동작을 확인한다.
3. **책 구조**: 각 개념서는 `docs/<책이름>/` 폴더 = `README.md`(표지·목차) + 장별 파일 `NN-제목.md`.
4. **각 장(파일) 형식**: 도입(왜 필요한가) → 소절마다 [개념 → 원리(왜 그렇게 동작하는가) → ASCII 다이어그램 → 코드 + `./run.sh` 연결] → `## ⚠️ 흔한 오해와 함정` → `## 연습문제(확인문제)`(3~5개, 힌트/해설 포함) → `## 요약` → 이전/다음 장 링크.
5. **한국어**, "왜?" 서술 톤. 정식 교과서 수준의 분량과 깊이(개념 나열이 아니라 원리·증명·트레이드오프까지).

## 집필 절차(권장)

1. 담당 범위의 현재 문서(`docs/<책>/`)와 코드(`chapter*/src`)를 읽고, 위 커리큘럼과 대조해 **누락 목록**을 만든다.
2. 누락 개념마다 (a) 실행 스크립트 작성 → `javac`/`java`로 검증, (b) 개념서 절 작성(코드·`./run.sh` 연결), (c) run.sh 배열에 등록.
3. 확인문제(연습문제)와 요약, 이전/다음 링크를 채운다.
4. 마지막에 전체 컴파일(`javac -d /tmp/out $(find chapter* -name '*.java')`)과 링크 점검으로 회귀를 막는다.

## 하지 말 것
- 한 파일에 여러 개념을 몰아넣기(개념당 스크립트 1개 위반).
- 설명만 있고 돌려볼 코드가 없는 절 남기기.
- 컴파일/실행 검증 없이 코드 스니펫만 붙이기.
- run.sh/docker-compose를 다른 에이전트와 동시에 편집(충돌) — 코드 파일은 패키지 디렉토리에 두고, run.sh 등록은 코디네이터와 조율.
