## 1. 도입: 왜 바닥부터 구현하는가?

Java 표준 라이브러리에는 `ArrayList`, `LinkedList`, `ArrayDeque`, `Stack` 등 풍부한 자료구조가 이미 구현되어 있다. 그런데도 이 장에서 이들을 처음부터 직접 구현하는 이유는 무엇인가?

첫째, **Big-O 복잡도가 몸에 익는다.** `get(i)`가 왜 O(1)이고 `removeAt(i)`가 왜 O(n)인지, 코드를 직접 쓰고 나면 암기가 아닌 이해로 알게 된다.

둘째, **라이브러리를 올바르게 선택할 수 있다.** "리스트가 필요하면 `ArrayList`를 기본으로 쓰고, 앞·뒤 삽입이 빈번하면 `LinkedList`를 고려한다"는 조언의 근거를 이해하게 된다. (실제로 이 조언조차 캐시 지역성 때문에 항상 맞지는 않다는 것도 함께 배울 것이다.)

셋째, **면접·코딩테스트에서 필수다.** 연결 리스트 역순, 스택으로 수식 파싱, 큐를 이용한 BFS 등 문제가 자료구조 동작 원리를 반드시 요구한다.

이 장의 목표는 다음 네 가지 자료구조를 라이브러리 없이 구현하고, 각 연산의 시간·공간 복잡도를 분석하는 것이다.

| 자료구조 | Java 라이브러리 대응 | 핵심 특성 |
|---|---|---|
| 동적 배열 | `ArrayList` | 인덱스 O(1), 자동 확장 |
| 단일 연결 리스트 | `LinkedList` | 앞·끝 삽입·삭제 O(1) |
| 스택 | `Deque`(권장) / `Stack` | LIFO — 마지막 입력 먼저 출력 |
| 큐 | `ArrayDeque` / `LinkedList` | FIFO — 처음 입력 먼저 출력 |

## 2. 동적 배열 (Dynamic Array)

### 2.1 개념과 원리

Java의 배열은 생성 시점에 크기가 고정된다. 요소를 계속 추가해야 하는 상황이라면, 용량이 꽉 찼을 때 더 큰 배열을 새로 할당하고 기존 요소를 모두 복사하는 전략이 필요하다. 이것이 **동적 배열**의 핵심이다.

```
초기 상태 (capacity = 4, size = 2):
+----+----+----+----+
| 10 | 20 |    |    |
+----+----+----+----+
  0    1    2    3
  ^         ^
  data[0]  size=2, capacity=4 (빈 슬롯 2개)

add(30), add(40) 후 (capacity = 4, size = 4):
+----+----+----+----+
| 10 | 20 | 30 | 40 |
+----+----+----+----+
  0    1    2    3
                ^
               size=4, capacity=4 (빈 슬롯 0개 — 꽉 찼음)

add(50) 호출 → capacity 2배(=8) 확장 후 복사:
+----+----+----+----+----+----+----+----+
| 10 | 20 | 30 | 40 | 50 |    |    |    |
+----+----+----+----+----+----+----+----+
  0    1    2    3    4    5    6    7
                          ^
                         size=5, capacity=8
```

**중간 삽입(insert)과 중간 삭제(removeAt)는 O(n)이다.**

```
removeAt(1) — 인덱스 1('20') 삭제:

이전: [10][20][30][40][50]
           ^--- 삭제 대상

삭제 후 한 칸씩 왼쪽으로 이동 (shift):
        [30][40][50] ← 3번의 이동
이후: [10][30][40][50][  ]
size = 4
```

### 2.2 분할상환 분석 (Amortized Analysis)

"용량이 꽉 찼을 때 2배로 복사하면 O(n)이 걸리는데, 그러면 `add`가 O(n)이 아닌가?"라는 의문은 자연스럽다. **분할상환 분석**은 이 질문에 정밀하게 답한다.

**전제:** 용량이 꽉 찰 때마다 2배로 확장한다.

n개의 요소를 순서대로 추가한다고 하자. 확장이 발생하는 시점과 복사 비용은 다음과 같다.

| 확장 번호 | 확장 시 크기 | 복사 비용 |
|---|---|---|
| 1 | 1 → 2 | 1 |
| 2 | 2 → 4 | 2 |
| 3 | 4 → 8 | 4 |
| k | 2^(k-1) → 2^k | 2^(k-1) |

총 복사 비용의 합: 1 + 2 + 4 + ... + n/2 = n - 1 < n

n번의 `add` 연산에 대한 **총 비용**은 다음과 같다.
- n번의 단순 추가: O(n)
- 확장 시 복사 총합: O(n)

합계 O(n)을 n으로 나누면 **1회당 평균 비용 = O(1)** (분할상환 O(1), amortized O(1))

단, **최악의 경우(worst case)** 확장이 발생하는 순간 1회의 `add`는 O(n)이다. 이 구분은 복잡도 표에서 반드시 명시해야 한다.

**확장 계수(growth factor):** Java의 `ArrayList`는 확장 계수를 1.5로 사용한다(이 章 ⚠️ 절에서 상세 논의). 이론적으로 1보다 크기만 하면 분할상환 O(1)이 성립하며, 계수가 클수록 낭비 공간이 늘고 작을수록 복사 횟수가 는다.

### 2.3 Java 구현

```java
/**
 * 동적 배열 (ArrayList 뼈대)
 * 내부 고정크기 배열 + 꽉 차면 2배 확장
 */
public class DynamicArray {
    private int[] data;      // 내부 배열
    private int size;        // 현재 저장된 요소 수
    private int capacity;    // 내부 배열의 실제 크기

    public DynamicArray() {
        this(2);             // 초기 용량 2
    }

    public DynamicArray(int initialCapacity) {
        if (initialCapacity < 1) throw new IllegalArgumentException("용량은 1 이상이어야 합니다.");
        this.capacity = initialCapacity;
        this.data = new int[capacity];
        this.size = 0;
    }

    /** 끝에 추가 — 분할상환 O(1) */
    public void add(int value) {
        if (size == capacity) {
            resize();        // 꽉 찼으면 2배 확장
        }
        data[size++] = value;
    }

    /** i번째 위치에 삽입 — O(n) */
    public void insert(int index, int value) {
        checkIndexForAdd(index);
        if (size == capacity) resize();
        // index 이후 요소를 한 칸씩 오른쪽으로 이동
        for (int i = size; i > index; i--) {
            data[i] = data[i - 1];
        }
        data[index] = value;
        size++;
    }

    /** i번째 요소 제거 — O(n) */
    public int removeAt(int index) {
        checkIndex(index);
        int removed = data[index];
        // index 이후 요소를 한 칸씩 왼쪽으로 이동
        for (int i = index; i < size - 1; i++) {
            data[i] = data[i + 1];
        }
        data[--size] = 0;  // 참조 해제 (GC 친화적)
        return removed;
    }

    /** i번째 요소 조회 — O(1) */
    public int get(int index) {
        checkIndex(index);
        return data[index];
    }

    /** i번째 요소 수정 — O(1) */
    public void set(int index, int value) {
        checkIndex(index);
        data[index] = value;
    }

    public int size()     { return size; }
    public boolean isEmpty() { return size == 0; }

    /** 2배 확장 (핵심 로직) */
    private void resize() {
        int newCapacity = capacity * 2;
        System.out.printf("[resize] capacity %d → %d%n", capacity, newCapacity);
        int[] newData = new int[newCapacity];
        // 기존 요소 복사 — O(n)
        System.arraycopy(data, 0, newData, 0, size);
        data = newData;
        capacity = newCapacity;
    }

    private void checkIndex(int index) {
        if (index < 0 || index >= size)
            throw new IndexOutOfBoundsException("index=" + index + ", size=" + size);
    }
    private void checkIndexForAdd(int index) {
        if (index < 0 || index > size)
            throw new IndexOutOfBoundsException("index=" + index + ", size=" + size);
    }

    @Override
    public String toString() {
        StringBuilder sb = new StringBuilder("[");
        for (int i = 0; i < size; i++) {
            sb.append(data[i]);
            if (i < size - 1) sb.append(", ");
        }
        sb.append("] size=").append(size).append(", cap=").append(capacity);
        return sb.toString();
    }

    // -------------------------------------------------------
    // 사용 예
    // -------------------------------------------------------
    public static void main(String[] args) {
        DynamicArray arr = new DynamicArray(2);
        System.out.println("초기: " + arr);   // [] size=0, cap=2

        arr.add(10); arr.add(20);
        System.out.println("add(10,20): " + arr);  // [10, 20] size=2, cap=2

        arr.add(30);   // 여기서 resize: cap 2→4
        arr.add(40);
        System.out.println("add(30,40): " + arr);  // [10, 20, 30, 40] size=4, cap=4

        arr.add(50);   // 여기서 resize: cap 4→8
        System.out.println("add(50):    " + arr);  // [10, 20, 30, 40, 50] size=5, cap=8

        System.out.println("get(2): " + arr.get(2));      // 30
        arr.removeAt(1);
        System.out.println("removeAt(1): " + arr);        // [10, 30, 40, 50] size=4, cap=8
    }
}
```

**실행 출력 예:**

```
초기: [] size=0, cap=2
add(10,20): [10, 20] size=2, cap=2
[resize] capacity 2 → 4
add(30,40): [10, 20, 30, 40] size=4, cap=4
[resize] capacity 4 → 8
add(50):    [10, 20, 30, 40, 50] size=5, cap=8
get(2): 30
removeAt(1): [10, 30, 40, 50] size=4, cap=8
```

## 3. 단일 연결 리스트 (Singly Linked List)

### 3.1 개념과 원리

연결 리스트는 **노드(Node)**들이 참조(포인터)로 연결된 구조다. 각 노드는 값(value)과 다음 노드를 가리키는 참조(next)를 담는다. 배열처럼 메모리가 연속적이지 않아도 된다.

```
노드 하나의 구조:
+---------+------+
|  value  | next |---->  다음 노드
+---------+------+

단일 연결 리스트 [5 → 10 → 15 → 20]:

head
  |
  v
+---+--+   +----+--+   +----+--+   +----+------+
| 5 |  +-->| 10 |  +-->| 15 |  +-->| 20 | null |
+---+--+   +----+--+   +----+--+   +----+------+
                                      ^
                                     tail
```

**addFirst(99) 후:**

```
head
  |
  v
+----+--+   +---+--+   +----+--+   +----+--+   +----+------+
| 99 |  +-->| 5 |  +-->| 10 |  +-->| 15 |  +-->| 20 | null |
+----+--+   +---+--+   +----+--+   +----+--+   +----+------+
```

**removeFirst() 후 (99 제거):**

```
head
  |
  v
+---+--+   +----+--+   +----+--+   +----+------+
| 5 |  +-->| 10 |  +-->| 15 |  +-->| 20 | null |
+---+--+   +----+--+   +----+--+   +----+------+
```

`addFirst`와 `removeFirst`는 head 포인터만 변경하면 되므로 **O(1)**이다. tail 참조를 별도로 유지하면 `addLast`도 **O(1)**이다. 그러나 `get(i)`는 head에서 i번 따라가야 하므로 **O(n)**이다.

### 3.2 배열 vs 연결 리스트: 캐시 지역성

연결 리스트의 노드는 힙(heap) 전역에 흩어진다. CPU는 메모리를 접근할 때 캐시 라인(보통 64 바이트) 단위로 읽어온다. 배열은 요소가 연속 메모리에 있어 한 번의 캐시 로드에 여러 요소를 올릴 수 있지만, 연결 리스트는 노드마다 캐시 미스(cache miss)가 발생할 수 있다.

```
배열 (연속 메모리, 캐시 친화적):
RAM:  [10][20][30][40][50][60][70][80]
       ^-- 캐시 라인 하나로 다수 요소 적재

연결 리스트 (분산 메모리, 캐시 비친화적):
RAM:   0x100: [10|0x340]
       0x340: [20|0x820]
       0x820: [30|0x1A0]
              ↑ 매 노드 접근마다 캐시 미스 가능성
```

실제 벤치마크에서 순차 탐색(iteration)은 `ArrayList`가 `LinkedList`보다 수 배 빠른 경우가 많다. "삽입·삭제가 많으면 `LinkedList`가 유리하다"는 통념은 중간 위치에 접근하기 위한 O(n) 탐색 비용과 캐시 미스를 간과한 것이다. 실무에서는 **`ArrayList`를 기본으로 사용하고**, 앞에서의 삽입·삭제가 매우 빈번하거나 이터레이터 기반 삭제가 주된 작업일 때만 `LinkedList`(또는 `ArrayDeque`)를 검토한다.

### 3.3 Java 구현

```java
/**
 * 단일 연결 리스트 (LinkedList 뼈대)
 * head + tail 참조 유지 → addLast O(1) 보장
 */
public class SinglyLinkedList {

    /** 내부 노드 클래스 */
    private static class Node {
        int value;
        Node next;
        Node(int value) { this.value = value; }
    }

    private Node head;   // 첫 번째 노드
    private Node tail;   // 마지막 노드 (addLast O(1) 위해 유지)
    private int size;

    /** 앞에 추가 — O(1) */
    public void addFirst(int value) {
        Node newNode = new Node(value);
        newNode.next = head;
        head = newNode;
        if (tail == null) tail = head;   // 첫 노드이면 tail도 설정
        size++;
    }

    /** 뒤에 추가 — O(1) (tail 유지 덕분) */
    public void addLast(int value) {
        Node newNode = new Node(value);
        if (tail == null) {
            head = tail = newNode;
        } else {
            tail.next = newNode;
            tail = newNode;
        }
        size++;
    }

    /** 앞 제거 — O(1) */
    public int removeFirst() {
        if (head == null) throw new NoSuchElementException();
        int removed = head.value;
        head = head.next;
        if (head == null) tail = null;   // 리스트가 비었으면 tail도 null
        size--;
        return removed;
    }

    /** 뒤 제거 — O(n): tail의 이전 노드를 찾아야 함 */
    public int removeLast() {
        if (head == null) throw new NoSuchElementException();
        if (head == tail) {              // 요소가 1개
            int removed = head.value;
            head = tail = null;
            size--;
            return removed;
        }
        // tail 이전 노드 탐색
        Node prev = head;
        while (prev.next != tail) prev = prev.next;
        int removed = tail.value;
        prev.next = null;
        tail = prev;
        size--;
        return removed;
    }

    /** i번째 값 조회 — O(n) */
    public int get(int index) {
        checkIndex(index);
        Node cur = head;
        for (int i = 0; i < index; i++) cur = cur.next;
        return cur.value;
    }

    /** 특정 값 포함 여부 — O(n) */
    public boolean contains(int value) {
        Node cur = head;
        while (cur != null) {
            if (cur.value == value) return true;
            cur = cur.next;
        }
        return false;
    }

    public int size()       { return size; }
    public boolean isEmpty(){ return size == 0; }

    private void checkIndex(int index) {
        if (index < 0 || index >= size)
            throw new IndexOutOfBoundsException("index=" + index + ", size=" + size);
    }

    @Override
    public String toString() {
        StringBuilder sb = new StringBuilder("head → ");
        Node cur = head;
        while (cur != null) {
            sb.append("[").append(cur.value).append("]");
            if (cur.next != null) sb.append(" → ");
            cur = cur.next;
        }
        sb.append(" → null  (size=").append(size).append(")");
        return sb.toString();
    }

    // -------------------------------------------------------
    // 사용 예
    // -------------------------------------------------------
    public static void main(String[] args) {
        SinglyLinkedList list = new SinglyLinkedList();
        list.addLast(5);
        list.addLast(10);
        list.addLast(15);
        list.addLast(20);
        System.out.println(list);
        // head → [5] → [10] → [15] → [20] → null  (size=4)

        list.addFirst(1);
        System.out.println(list);
        // head → [1] → [5] → [10] → [15] → [20] → null  (size=5)

        System.out.println("removeFirst: " + list.removeFirst()); // 1
        System.out.println("get(2): " + list.get(2));             // 15
        System.out.println(list);
        // head → [5] → [10] → [15] → [20] → null  (size=4)
    }
}
```

## 4. 스택 (Stack)

### 4.1 개념과 원리

스택은 **LIFO(Last In, First Out)** 원칙으로 동작한다. 마지막으로 넣은 데이터가 가장 먼저 나온다. 접시를 쌓는 것을 떠올리면 된다.

```
push(A) → push(B) → push(C) 후:

         +---+
   top → | C |  ← pop() 시 C가 먼저 나옴
         +---+
         | B |
         +---+
         | A |
         +---+
         (바닥)

pop() → C 반환:

         +---+
   top → | B |
         +---+
         | A |
         +---+
```

연결 리스트의 head를 top으로 사용하면 `push`/`pop` 모두 O(1)로 구현할 수 있다.

**주요 활용 사례:**
- **콜 스택(call stack):** 재귀 함수 호출 추적
- **Undo/Redo:** 텍스트 에디터 실행 취소
- **괄호 유효성 검사:** `(`, `[`, `{` 짝 확인
- **DFS(깊이 우선 탐색):** 방문 순서 관리
- **수식 계산:** 후위 표기법(Postfix) 평가

### 4.2 Java 구현

```java
/**
 * 스택 (LIFO) — 연결 리스트 head를 top으로 활용
 */
public class Stack {

    private static class Node {
        int value;
        Node next;
        Node(int value) { this.value = value; }
    }

    private Node top;    // 스택의 최상단 노드
    private int size;

    /** 데이터 삽입 — O(1) */
    public void push(int value) {
        Node newNode = new Node(value);
        newNode.next = top;
        top = newNode;
        size++;
    }

    /** 최상단 데이터 제거 및 반환 — O(1) */
    public int pop() {
        if (isEmpty()) throw new EmptyStackException();
        int removed = top.value;
        top = top.next;
        size--;
        return removed;
    }

    /** 최상단 데이터 조회 (제거하지 않음) — O(1) */
    public int peek() {
        if (isEmpty()) throw new EmptyStackException();
        return top.value;
    }

    public int size()       { return size; }
    public boolean isEmpty(){ return top == null; }

    @Override
    public String toString() {
        StringBuilder sb = new StringBuilder("top → ");
        Node cur = top;
        while (cur != null) {
            sb.append("[").append(cur.value).append("]");
            if (cur.next != null) sb.append(" → ");
            cur = cur.next;
        }
        return sb.toString();
    }

    // -------------------------------------------------------
    // 활용 예: 괄호 유효성 검사
    // -------------------------------------------------------
    public static boolean isValidParentheses(String s) {
        Stack stack = new Stack();
        for (char c : s.toCharArray()) {
            if (c == '(' || c == '[' || c == '{') {
                stack.push(c);
            } else if (c == ')' || c == ']' || c == '}') {
                if (stack.isEmpty()) return false;
                char top = (char) stack.pop();
                if ((c == ')' && top != '(') ||
                    (c == ']' && top != '[') ||
                    (c == '}' && top != '{')) return false;
            }
        }
        return stack.isEmpty();
    }

    public static void main(String[] args) {
        Stack stack = new Stack();
        stack.push(10); stack.push(20); stack.push(30);
        System.out.println(stack);           // top → [30] → [20] → [10]
        System.out.println("pop: " + stack.pop()); // 30
        System.out.println("peek: " + stack.peek()); // 20
        System.out.println(stack);           // top → [20] → [10]

        System.out.println(isValidParentheses("({[]})")); // true
        System.out.println(isValidParentheses("({[}])"));  // false
    }
}
```

## 5. 큐 (Queue)

### 5.1 개념과 원리

큐는 **FIFO(First In, First Out)** 원칙으로 동작한다. 먼저 들어온 데이터가 먼저 나간다. 매표소 줄을 생각하면 된다.

```
enqueue(A) → enqueue(B) → enqueue(C):

  front                         rear
    ↓                             ↓
  [A] ← 먼저 들어옴    [B]    [C] ← 나중에 들어옴

dequeue() → A 반환:

  front               rear
    ↓                   ↓
  [B]                [C]
```

head(front)에서 제거하고, tail(rear)에서 추가하면 head/tail 연결 리스트로 두 연산 모두 **O(1)**이다.

**주요 활용 사례:**
- **작업 대기열(task queue):** 운영체제 프로세스 스케줄링
- **BFS(너비 우선 탐색):** 그래프 최단 경로
- **프린터 스풀(spool):** 인쇄 작업 순서 관리
- **메시지 큐:** 비동기 이벤트 처리

### 5.2 원형 배열 기반 큐 (Circular Buffer)

연결 리스트 대신 배열로 큐를 구현할 때 흔히 발생하는 문제가 있다. 일반 배열에서 `dequeue`(앞에서 제거)를 O(1)로 하려면 front 인덱스를 앞으로 당기면 되는데, 이렇게 하면 배열 앞부분의 공간이 낭비된다.

```
일반 배열 큐 — dequeue를 O(1)로 구현할 때:

  [_][_][_][C][D][E]   front=3, rear=5
   0  1  2  3  4  5
            ^--- 0~2 슬롯은 이미 사용했으나 재활용 불가
```

**원형 배열(ring buffer)**은 인덱스를 `capacity`로 나눈 나머지(%)를 사용해 배열을 원형으로 취급한다.

```
capacity = 6인 원형 큐:

         0
      +-----+
  5   |  D  |   1
 [E]  +-----+  [A] ← (이미 dequeue됨)
      |     |
  4   +-----+   2
 [D]  |  C  |  [B] ← (이미 dequeue됨)
      +-----+
  3   |  E  |
      +-----+

front=2 (다음에 dequeue될 위치)
rear=0  (다음에 enqueue될 위치)
size=3  (C, D, E)
```

**원형 큐 Java 구현:**

```java
/**
 * 원형 배열 기반 큐 (Ring Buffer Queue)
 * enqueue / dequeue 모두 O(1)
 */
public class CircularQueue {
    private int[] data;
    private int front;     // 다음에 dequeue될 위치
    private int rear;      // 다음에 enqueue될 위치
    private int size;
    private int capacity;

    public CircularQueue(int capacity) {
        this.capacity = capacity;
        this.data = new int[capacity];
        this.front = 0;
        this.rear = 0;
        this.size = 0;
    }

    /** 뒤에 추가 — O(1) */
    public void enqueue(int value) {
        if (size == capacity) throw new IllegalStateException("큐가 가득 찼습니다.");
        data[rear] = value;
        rear = (rear + 1) % capacity;   // 원형 인덱스 이동
        size++;
    }

    /** 앞에서 제거 — O(1) */
    public int dequeue() {
        if (isEmpty()) throw new NoSuchElementException();
        int removed = data[front];
        front = (front + 1) % capacity; // 원형 인덱스 이동
        size--;
        return removed;
    }

    /** 앞 조회 (제거 없음) — O(1) */
    public int peek() {
        if (isEmpty()) throw new NoSuchElementException();
        return data[front];
    }

    public int size()       { return size; }
    public boolean isEmpty(){ return size == 0; }
    public boolean isFull() { return size == capacity; }

    @Override
    public String toString() {
        StringBuilder sb = new StringBuilder("front[");
        for (int i = 0; i < size; i++) {
            sb.append(data[(front + i) % capacity]);
            if (i < size - 1) sb.append(", ");
        }
        sb.append("]rear  (size=").append(size)
          .append(", cap=").append(capacity).append(")");
        return sb.toString();
    }
}
```

### 5.3 Java 구현 (연결 리스트 기반)

```java
/**
 * 큐 (FIFO) — 연결 리스트 head/tail 활용
 * enqueue(뒤) / dequeue(앞) 모두 O(1)
 */
public class Queue {

    private static class Node {
        int value;
        Node next;
        Node(int value) { this.value = value; }
    }

    private Node front;  // dequeue 측 (앞)
    private Node rear;   // enqueue 측 (뒤)
    private int size;

    /** 뒤에 추가 — O(1) */
    public void enqueue(int value) {
        Node newNode = new Node(value);
        if (rear == null) {
            front = rear = newNode;
        } else {
            rear.next = newNode;
            rear = newNode;
        }
        size++;
    }

    /** 앞에서 제거 — O(1) */
    public int dequeue() {
        if (isEmpty()) throw new NoSuchElementException();
        int removed = front.value;
        front = front.next;
        if (front == null) rear = null;
        size--;
        return removed;
    }

    /** 앞 조회 — O(1) */
    public int peek() {
        if (isEmpty()) throw new NoSuchElementException();
        return front.value;
    }

    public int size()       { return size; }
    public boolean isEmpty(){ return front == null; }

    @Override
    public String toString() {
        StringBuilder sb = new StringBuilder("front[");
        Node cur = front;
        while (cur != null) {
            sb.append(cur.value);
            if (cur.next != null) sb.append(", ");
            cur = cur.next;
        }
        sb.append("]rear  (size=").append(size).append(")");
        return sb.toString();
    }

    // -------------------------------------------------------
    // 사용 예
    // -------------------------------------------------------
    public static void main(String[] args) {
        Queue queue = new Queue();
        queue.enqueue(10); queue.enqueue(20); queue.enqueue(30);
        System.out.println(queue);                    // front[10, 20, 30]rear
        System.out.println("dequeue: " + queue.dequeue()); // 10
        System.out.println("peek: " + queue.peek());       // 20
        System.out.println(queue);                    // front[20, 30]rear
    }
}
```

**스택과 큐의 차이 — 한 줄 요약:**

```
스택 (LIFO): push/pop 모두 같은 쪽(top)에서
큐  (FIFO): enqueue는 rear, dequeue는 front에서
```

## 6. 이중 연결 리스트와 Deque

이중 연결 리스트(Doubly Linked List)는 각 노드가 이전(prev)과 다음(next) 노드를 모두 참조한다. 이를 통해 `removeLast`도 O(1)로 처리할 수 있다.

```
이중 연결 리스트 [5 ↔ 10 ↔ 15]:

head                            tail
  |                               |
  v                               v
+----+------+   +------+----+------+   +------+----+
|null| 5 |  +-->+  10  +   +  +-->+  15  |null|
+----+------+   +------+----+------+   +------+----+
      <--prev       prev-->  <--prev
                             next = null
```

이중 연결 리스트는 **Deque(Double-Ended Queue, 덱)**의 기반이 된다. 덱은 앞·뒤 양쪽에서 삽입·삭제가 모두 O(1)이다. Java의 `LinkedList`는 실제로 이중 연결 리스트 기반의 덱을 구현한다.

```java
/** 이중 연결 리스트 노드 */
private static class DNode {
    int value;
    DNode prev, next;
    DNode(int value) { this.value = value; }
}

// 덱 핵심 연산 (이중 연결 리스트 사용):
// addFirst(v)  → head 앞에 새 노드 O(1)
// addLast(v)   → tail 뒤에 새 노드 O(1)
// removeFirst()→ head 노드 제거 O(1)
// removeLast() → tail 노드 제거 O(1) ← 단일 연결 리스트는 O(n)
```

Java `ArrayDeque`는 원형 배열 기반으로 덱을 구현해 캐시 지역성까지 챙긴다. 실무에서 스택·큐 모두 `ArrayDeque`가 권장되는 이유다.

## 8. 복잡도 표

### 동적 배열

| 연산 | 최선 | 평균 (분할상환) | 최악 | 공간 |
|---|---|---|---|---|
| `get(i)` / `set(i)` | O(1) | O(1) | O(1) | O(1) |
| `add(끝)` | O(1) | **O(1) amortized** | O(n) (확장 시) | O(1) |
| `insert(중간)` | O(1) | O(n) | O(n) | O(1) |
| `removeAt(중간)` | O(1) | O(n) | O(n) | O(1) |
| `contains` / `indexOf` | O(1) | O(n) | O(n) | O(1) |
| **공간 복잡도 (전체)** | — | — | — | **O(n)** |

> ⚠️ `add(끝)`의 **평균 O(1)**은 분할상환 기준이다. 확장이 발생하는 순간의 **최악은 O(n)**이다. 이 구분을 면접에서 반드시 언급해야 한다.

### 단일 연결 리스트

| 연산 | 시간 복잡도 | 비고 |
|---|---|---|
| `addFirst` | O(1) | head 포인터만 변경 |
| `removeFirst` | O(1) | head 포인터만 변경 |
| `addLast` | O(1) | tail 유지 시 |
| `removeLast` | O(n) | tail 이전 노드 탐색 필요 |
| `get(i)` | O(n) | head부터 순회 |
| `contains` | O(n) | 전체 순회 |
| **공간 복잡도** | — | **O(n)** |

### 스택

| 연산 | 시간 복잡도 | 구현 방식 |
|---|---|---|
| `push` | O(1) | head에 노드 추가 |
| `pop` | O(1) | head 노드 제거 |
| `peek` | O(1) | head 값 조회 |
| **공간 복잡도** | — | **O(n)** |

### 큐

| 연산 | 연결 리스트 | 원형 배열 | 일반 배열 |
|---|---|---|---|
| `enqueue` | O(1) | O(1) | O(1) |
| `dequeue` | O(1) | O(1) | **O(n)** (shift 발생) |
| `peek` | O(1) | O(1) | O(1) |
| **공간 복잡도** | O(n) | O(capacity) | O(n) |

> 일반 배열로 큐를 구현하면 `dequeue`마다 앞 요소를 제거하고 나머지를 당기는 shift가 필요해 O(n)이 된다. 이것이 **원형 배열(ring buffer)**이 필요한 이유다.

### 네 자료구조 종합 비교

| 자료구조 | 인덱스 접근 | 앞 삽입·삭제 | 끝 삽입·삭제 | 중간 삽입·삭제 |
|---|---|---|---|---|
| 동적 배열 | **O(1)** | O(n) | O(1) amortized | O(n) |
| 단일 연결 리스트 | O(n) | **O(1)** | O(1) / O(n) | O(n) + 탐색 |
| 이중 연결 리스트 | O(n) | **O(1)** | **O(1)** | O(n) + 탐색 |
| 스택 | — | — | O(1) (push/pop) | — |
| 큐 | — | O(1) (dequeue) | O(1) (enqueue) | — |

## 9. ⚠️ 흔한 오해와 함정

### ⚠️ 함정 1: "연결 리스트는 삽입·삭제가 항상 빠르다"

**반은 맞고, 반은 틀리다.** 연결 리스트의 앞·끝 삽입·삭제는 O(1)이 맞다. 그러나 **중간에 삽입하려면 먼저 해당 위치까지 탐색(O(n))해야 한다.** 예를 들어 인덱스 500번째 위치에 삽입하려면 head부터 499번을 따라가야 한다.

반면 `ArrayList`는 중간 삽입 시 shift가 O(n)이지만, **배열의 연속 메모리 덕분에 캐시 히트율이 높아 실제 수행 시간은 더 빠른 경우가 많다.** JVM 벤치마크에서 대부분의 경우 `ArrayList`가 `LinkedList`보다 빠르게 측정된다.

실무 결론: **`ArrayList`를 기본으로 사용하고, 명확한 성능 측정 근거 없이 `LinkedList`로 교체하지 않는다.**

### ⚠️ 함정 2: 배열 확장 계수 2 vs 1.5 — Java의 실제 선택

이 章에서 구현한 동적 배열은 용량이 꽉 차면 **2배**로 확장했다. 그런데 Java의 `ArrayList` 내부 소스를 보면 확장 계수가 **1.5**다.

```java
// ArrayList.java (OpenJDK) 내부 grow() 메서드 핵심
int newCapacity = oldCapacity + (oldCapacity >> 1); // 1.5배 (비트 시프트 이용)
```

- **계수 2:** 낭비 공간 최대 50%, 확장 빈도 낮음
- **계수 1.5:** 낭비 공간 최대 33%, 확장 빈도 약간 높음, 메모리 효율 우수

두 방법 모두 분할상환 O(1)이 성립한다. 단, 계수를 1에 가깝게 작게 잡으면 복사 비용이 기하급수적으로 커지므로 주의해야 한다.

### ⚠️ 함정 3: 일반 배열로 큐를 구현하면 dequeue가 O(n)

```java
// 잘못된 배열 큐: dequeue마다 shift
public int dequeue() {
    int val = data[0];
    for (int i = 0; i < size - 1; i++)
        data[i] = data[i + 1]; // O(n) shift 발생!
    size--;
    return val;
}
```

이 구현은 n번의 dequeue에 O(n²) 총 비용이 든다. **원형 배열(ring buffer)**을 사용해 front 인덱스만 이동하면 dequeue를 O(1)로 만들 수 있다. Java의 `ArrayDeque`가 바로 이 방식이다.

```java
// 올바른 원형 큐: front 인덱스만 이동 O(1)
public int dequeue() {
    int val = data[front];
    front = (front + 1) % capacity;
    size--;
    return val;
}
```

### ⚠️ 함정 4: 스택 오버플로우 (StackOverflowError)

연결 리스트 기반 스택은 이론상 메모리가 허용하는 한 무한히 커질 수 있다. 그러나 **Java의 실제 콜 스택**은 JVM이 관리하는 고정 크기 메모리 영역을 사용한다. 재귀 함수가 지나치게 깊이 호출되면 콜 스택이 꽉 차서 `StackOverflowError`가 발생한다. 재귀를 스택 자료구조로 변환하면 이 문제를 피할 수 있다.

```java
// StackOverflowError 발생 예
public static int badRecursion(int n) {
    return badRecursion(n - 1); // 종료 조건 없음 → 무한 재귀
}

// 스택 자료구조로 DFS 구현 → StackOverflowError 없음
public static void dfsIterative(Graph g, int start) {
    Stack<Integer> stack = new Stack<>();
    stack.push(start);
    while (!stack.isEmpty()) {
        int v = stack.pop();
        // 방문 처리 ...
        for (int neighbor : g.getNeighbors(v)) stack.push(neighbor);
    }
}
```

### ⚠️ 함정 5: tail 참조 없는 연결 리스트의 addLast는 O(n)

tail 포인터를 따로 유지하지 않으면 `addLast`마다 head부터 끝까지 순회해야 한다. 큐를 단순 연결 리스트로 구현할 때 tail을 빠뜨리면 enqueue가 O(n)이 되어 큐의 핵심 장점이 사라진다. 항상 head·tail 두 포인터를 관리하자.

### 문제 1 (★☆☆) — 동적 배열 remove(중간) 구현

**문제:** 위에서 구현한 `DynamicArray`에 `remove(int value)` 메서드를 추가하라. 이 메서드는 첫 번째로 발견되는 `value`를 찾아 제거하고, 제거 성공 여부를 boolean으로 반환한다. 없으면 false를 반환한다.

**힌트:** 먼저 선형 탐색으로 인덱스를 찾고, 찾으면 `removeAt(index)`를 호출한다.

<details>
<summary>해설 보기</summary>

```java
public boolean remove(int value) {
    for (int i = 0; i < size; i++) {
        if (data[i] == value) {
            removeAt(i);
            return true;
        }
    }
    return false;
}
```

시간 복잡도: O(n) (탐색 O(n) + 이동 O(n) = O(n))  
단, Java `ArrayList.remove(Object)`도 동일한 원리로 O(n)이다.

</details>

### 문제 2 (★★☆) — 스택 2개로 큐 만들기

**문제:** 스택 두 개만 사용해서 큐의 `enqueue`와 `dequeue`를 구현하라. 추가 배열이나 리스트는 사용할 수 없다.

**힌트:** `inbox` 스택과 `outbox` 스택을 사용한다. `enqueue`는 `inbox`에 push한다. `dequeue`는 `outbox`가 비어 있을 때만 `inbox`의 모든 요소를 `outbox`로 이동한다.

<details>
<summary>해설 보기</summary>

```java
public class QueueWithTwoStacks {
    private java.util.Deque<Integer> inbox  = new java.util.ArrayDeque<>();
    private java.util.Deque<Integer> outbox = new java.util.ArrayDeque<>();

    public void enqueue(int value) {
        inbox.push(value);   // O(1)
    }

    public int dequeue() {
        if (outbox.isEmpty()) {
            // inbox 전체를 outbox로 이동 → 순서가 역전되어 FIFO 실현
            while (!inbox.isEmpty()) outbox.push(inbox.pop());
        }
        if (outbox.isEmpty()) throw new java.util.NoSuchElementException();
        return outbox.pop(); // O(1)
    }
}
```

**복잡도 분석:**  
- `enqueue`: O(1)  
- `dequeue`: **분할상환 O(1)** — 각 요소는 inbox에서 outbox로 한 번만 이동된다. n번의 dequeue에 총 n번의 이동만 발생하므로 1회당 평균 O(1)이다.  
- 최악의 경우(outbox 비어 있을 때): O(n)

이 패턴은 동적 배열의 분할상환 O(1)과 동일한 논리 구조다.

</details>

### 문제 3 (★★☆) — 원형 배열 기반 큐 (Ring Buffer) 구현

**문제:** 위에서 작성한 `CircularQueue`를 확장해, 큐가 꽉 찼을 때 용량을 2배로 자동 확장하는 `ResizableCircularQueue`를 구현하라.

**힌트:** 확장 시 기존 요소를 front부터 차례로 새 배열 인덱스 0부터 복사한다. front와 rear를 재설정하는 것을 잊지 말자.

<details>
<summary>해설 보기</summary>

```java
public class ResizableCircularQueue {
    private int[] data;
    private int front, rear, size, capacity;

    public ResizableCircularQueue(int capacity) {
        this.capacity = capacity;
        data = new int[capacity];
    }

    public void enqueue(int value) {
        if (size == capacity) resize();
        data[rear] = value;
        rear = (rear + 1) % capacity;
        size++;
    }

    public int dequeue() {
        if (size == 0) throw new java.util.NoSuchElementException();
        int val = data[front];
        front = (front + 1) % capacity;
        size--;
        return val;
    }

    private void resize() {
        int newCapacity = capacity * 2;
        int[] newData = new int[newCapacity];
        // front부터 size개를 새 배열 인덱스 0부터 복사
        for (int i = 0; i < size; i++) {
            newData[i] = data[(front + i) % capacity];
        }
        data = newData;
        front = 0;
        rear = size;          // size개 채워진 직후 위치
        capacity = newCapacity;
    }
}
```

핵심은 `resize` 시 원형 구조의 순서를 올바르게 펼쳐(linearize) 복사하는 것이다. `(front + i) % capacity`가 원형 인덱스 계산의 핵심이다.

</details>

### 문제 4 (★★★) — 이중 연결 리스트 구현

**문제:** 각 노드가 `prev`와 `next` 두 참조를 갖는 이중 연결 리스트를 구현하라. 다음 연산을 모두 O(1)로 구현해야 한다: `addFirst`, `addLast`, `removeFirst`, `removeLast`. 추가로 `get(i)`도 구현하되, 인덱스가 중간보다 앞이면 head부터, 중간보다 뒤이면 tail부터 탐색해 효율을 높여라.

**힌트:** `sentinel` 더미 노드(head, tail에 더미 사용)를 활용하면 경계 조건 처리가 단순해진다. 인덱스 기반 탐색은 `index < size / 2`이면 앞에서, 아니면 뒤에서 순회한다.

<details>
<summary>해설 보기</summary>

```java
public class DoublyLinkedList {
    private static class Node {
        int value; Node prev, next;
        Node(int v) { this.value = v; }
    }

    private final Node head = new Node(0); // 더미 헤드
    private final Node tail = new Node(0); // 더미 테일
    private int size;

    public DoublyLinkedList() {
        head.next = tail;
        tail.prev = head;
    }

    private void linkBetween(Node before, Node newNode, Node after) {
        newNode.prev = before; newNode.next = after;
        before.next = newNode; after.prev = newNode;
        size++;
    }

    private int unlink(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
        size--;
        return node.value;
    }

    public void addFirst(int v) { linkBetween(head, new Node(v), head.next); }
    public void addLast(int v)  { linkBetween(tail.prev, new Node(v), tail); }
    public int removeFirst()    { return unlink(head.next); }
    public int removeLast()     { return unlink(tail.prev); }

    public int get(int index) {
        if (index < size / 2) {
            Node cur = head.next;
            for (int i = 0; i < index; i++) cur = cur.next;
            return cur.value;
        } else {
            Node cur = tail.prev;
            for (int i = size - 1; i > index; i--) cur = cur.prev;
            return cur.value;
        }
    }
}
```

더미 노드(sentinel)를 사용하면 "리스트가 비어 있을 때" 같은 경계 조건에서 null 체크가 불필요해 코드가 깔끔해진다. Java `LinkedList`도 동일한 더미 노드 패턴을 사용한다.

</details>

### 문제 5 (★★★) — 최솟값을 O(1)로 반환하는 스택 (MinStack)

**문제:** 일반 스택의 기능(`push`, `pop`, `peek`)에 더해, 현재 스택에서 **최솟값을 O(1)**로 반환하는 `getMin()` 메서드를 구현하라.

**힌트:** 보조 스택을 하나 더 사용한다. 보조 스택은 현재까지의 최솟값 히스토리를 추적한다. `push` 시 현재 값이 보조 스택 top보다 작거나 같으면 보조 스택에도 push한다. `pop` 시 제거되는 값이 보조 스택 top과 같으면 보조 스택도 pop한다.

<details>
<summary>해설 보기</summary>

```java
public class MinStack {
    private java.util.Deque<Integer> stack    = new java.util.ArrayDeque<>();
    private java.util.Deque<Integer> minStack = new java.util.ArrayDeque<>();

    public void push(int value) {
        stack.push(value);
        if (minStack.isEmpty() || value <= minStack.peek()) {
            minStack.push(value);
        }
    }

    public int pop() {
        int val = stack.pop();
        if (val == minStack.peek()) minStack.pop();
        return val;
    }

    public int peek()   { return stack.peek(); }
    public int getMin() { return minStack.peek(); } // O(1)
}
```

예를 들어 `push(5, 3, 7, 2, 4)`를 하면:  
- `stack`:    top → [4, 2, 7, 3, 5]  
- `minStack`: top → [2, 3, 5]  

`getMin()`은 minStack.peek() = 2를 O(1)에 반환한다. `pop()` 4번으로 4를 꺼내도 minStack은 변하지 않는다(4 ≠ minStack.top=2).

이 패턴은 LeetCode 155번 "Min Stack" 문제와 동일하며, 면접에서 자주 출제된다.

</details>

## 11. 요약

이 장에서 네 가지 핵심 자료구조를 라이브러리 없이 직접 구현하고 복잡도를 분석했다.

| 자료구조 | 핵심 연산 복잡도 | 선택 기준 |
|---|---|---|
| **동적 배열** | `get/set` O(1), `add(끝)` amortized O(1) | 인덱스 접근, 순차 탐색이 주 |
| **단일 연결 리스트** | `addFirst/removeFirst` O(1), `get` O(n) | 앞 삽입·삭제가 매우 빈번할 때 |
| **스택 (LIFO)** | `push/pop/peek` O(1) | 콜스택, Undo, DFS, 괄호 검사 |
| **큐 (FIFO)** | `enqueue/dequeue` O(1) | 작업 대기열, BFS, 프린터 스풀 |

**핵심 교훈:**

1. 동적 배열의 `add(끝)`은 **분할상환 O(1)**이나 **최악 O(n)**이다. 두 값을 구분해서 설명할 수 있어야 한다.
2. 연결 리스트는 중간 접근이 O(n)이고 캐시 미스 비용이 있어, **실제 성능은 ArrayList가 유리한 경우가 많다.**
3. 배열로 큐를 만들면 `dequeue`가 O(n)이다. **원형 배열(ring buffer)**이 필수다.
4. 스택과 큐는 **데이터를 꺼내는 방향**만 다르다: 스택은 top(LIFO), 큐는 front(FIFO).
5. Java 실무에서는 `ArrayList`, `ArrayDeque`를 기본으로 사용한다. `ArrayDeque`는 스택과 큐 모두 커버한다.

다음 장(06. 트리와 힙)에서는 선형 자료구조를 넘어 트리 구조로 나아가며, `get`·`insert`·`remove` 모두 O(log n)을 달성하는 방법을 배운다.
