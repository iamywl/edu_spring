## 1. 도입: 관계를 표현하는 자료구조

우리가 살아가는 세계는 연결로 가득하다. 서울 지하철 노선도에서 역과 역 사이의 연결, 인스타그램에서 팔로우로 이어진 사람들, 도로망 위에서 교차로와 도로, 웹 페이지 사이를 잇는 하이퍼링크까지. 이런 모든 "관계"를 수학적으로 추상화한 것이 바로 **그래프(Graph)**다.

배열은 순서(index)로 원소를 조회한다. 연결 리스트는 앞뒤 이웃을 사슬처럼 이었다. 트리는 부모-자식의 계층 구조였다. 그래프는 이보다 훨씬 자유롭다. 어느 정점에서 어느 정점으로든 연결될 수 있고, 사이클이 있어도 되며, 연결되지 않은 섬처럼 분리된 부분이 존재해도 된다. 그만큼 표현력이 강하고, 그만큼 다루기 까다롭다.

이 장에서는 그래프를 컴퓨터 메모리에 어떻게 저장하는지(표현 방법), 어떤 순서로 정점을 방문하는지(탐색), 그리고 출발지에서 목적지까지 가장 짧은 길을 어떻게 찾는지(최단 경로)를 차례로 공부한다. 5장의 큐와 스택, 6장의 힙과 우선순위 큐가 여기서 핵심 도구로 쓰인다. 앞 장들이 이 장의 준비였다고 해도 과언이 아니다.

## 2. 그래프의 기본 개념과 용어

### 2.1 정의

**그래프 G = (V, E)** 는 두 집합의 쌍이다.

- **V (Vertex, 정점)**: 개체의 집합. 지하철 노선도라면 각 역.
- **E (Edge, 간선)**: 정점 쌍의 집합. 두 역이 직접 연결되어 있다면 그 연결이 간선.

정점 수를 V (또는 |V|), 간선 수를 E (또는 |E|)로 표기한다. 알고리즘 복잡도에서 자주 등장하므로 기호에 익숙해지자.

### 2.2 방향 그래프 vs 무방향 그래프

```
무방향 그래프 (Undirected)        방향 그래프 (Directed, Digraph)
    A --- B                           A ---> B
    |     |                           |      |
    C --- D                           v      v
                                      C      D
간선 (A,B) = (B,A): 양방향           간선 (A,B) != (B,A): 단방향
```

- **무방향 그래프**: 친구 관계(나 ↔ 친구), 도로의 양방향 통행.
- **방향 그래프(유향 그래프)**: SNS 팔로우(나 → 셀럽), 웹 하이퍼링크, 작업 선후관계.

### 2.3 가중 그래프 vs 비가중 그래프

```
비가중 그래프            가중 그래프
  0 -- 1                  0 --4-- 1
  |    |                  |       |
  2 -- 3                  1      5
                          |       |
                          2 --3-- 3
```

- **비가중 그래프**: 연결 여부만 중요. 최단 경로 = 간선 수가 가장 적은 경로.
- **가중 그래프**: 간선에 비용/거리/시간 등의 **가중치(weight)**가 있음. 최단 경로 = 가중치 합이 가장 작은 경로.

### 2.4 핵심 용어 정리

| 용어 | 설명 |
|------|------|
| 차수 (Degree) | 정점에 연결된 간선의 수. 방향 그래프에서는 진입 차수(in-degree)와 진출 차수(out-degree)로 구분 |
| 경로 (Path) | 정점 v1, v2, ..., vk를 차례로 지나는 간선의 나열 |
| 단순 경로 | 같은 정점을 두 번 이상 방문하지 않는 경로 |
| 사이클 (Cycle) | 시작 정점과 끝 정점이 같은 경로 |
| 연결 그래프 | 모든 정점 쌍 사이에 경로가 존재하는 무방향 그래프 |
| 연결 요소 | 서로 연결된 정점들의 최대 집합 (connected component) |
| 희소 그래프 (Sparse) | E ≪ V²인 그래프. 현실 세계 대부분의 그래프가 해당 |
| 밀집 그래프 (Dense) | E가 V²에 가까운 그래프 |

## 3. 그래프 표현 방법

V개의 정점, E개의 간선을 가진 그래프를 컴퓨터 메모리에 저장하는 주요 방법은 두 가지다.

### 3.1 인접 리스트 (Adjacency List)

각 정점마다 **이웃 정점의 목록**을 저장하는 방식이다.

```
정점 0의 이웃: [1, 2]
정점 1의 이웃: [0, 3, 4]
정점 2의 이웃: [0, 4]
정점 3의 이웃: [1, 5]
정점 4의 이웃: [1, 2]
정점 5의 이웃: [3]

그래프 시각화:
    0
   / \
  1   2
 /|\   \
3 | 4   4
|   |
5   (1-4 공유)
```

Java 구현:

```java
import java.util.*;

// 인접 리스트 표현 (무방향 비가중 그래프)
public class AdjacencyList {
    private int V;
    private List<List<Integer>> adj;

    public AdjacencyList(int V) {
        this.V = V;
        adj = new ArrayList<>();
        for (int i = 0; i < V; i++) {
            adj.add(new ArrayList<>());
        }
    }

    // 무방향 간선 추가
    public void addEdge(int u, int v) {
        adj.get(u).add(v);
        adj.get(v).add(u);
    }

    public List<Integer> neighbors(int v) {
        return adj.get(v);
    }
}

// 가중치 그래프 - 내부 클래스로 간선 표현
class WeightedGraph {
    static class Edge {
        int to, weight;
        Edge(int to, int weight) {
            this.to = to;
            this.weight = weight;
        }
    }

    private int V;
    private List<List<Edge>> adj;

    public WeightedGraph(int V) {
        this.V = V;
        adj = new ArrayList<>();
        for (int i = 0; i < V; i++) {
            adj.add(new ArrayList<>());
        }
    }

    public void addEdge(int u, int v, int w) {
        adj.get(u).add(new Edge(v, w));
        adj.get(v).add(new Edge(u, w)); // 무방향이면 양쪽 추가
    }

    public List<Edge> neighbors(int v) {
        return adj.get(v);
    }
}
```

**공간 복잡도**: O(V + E). 각 정점을 한 번씩, 각 간선을 (무방향이면) 두 번씩 저장.

**장점**: 현실 그래프 대부분은 희소(E ≪ V²)하므로 메모리 효율이 좋다. 정점의 이웃을 순회하는 데 O(degree) 시간.

**단점**: 두 정점이 연결되었는지 O(1)에 확인 불가. 이웃 목록을 선형 탐색해야 하므로 O(degree).

### 3.2 인접 행렬 (Adjacency Matrix)

V × V 크기의 2차원 배열 `adj[u][v]`에 연결 여부(또는 가중치)를 저장하는 방식이다.

```
정점: 0, 1, 2, 3, 4, 5
간선: 0-1, 0-2, 1-3, 1-4, 2-4, 3-5

     0  1  2  3  4  5
  0 [0, 1, 1, 0, 0, 0]
  1 [1, 0, 0, 1, 1, 0]
  2 [1, 0, 0, 0, 1, 0]
  3 [0, 1, 0, 0, 0, 1]
  4 [0, 1, 1, 0, 0, 0]
  5 [0, 0, 0, 1, 0, 0]
```

Java 구현:

```java
public class AdjacencyMatrix {
    private int V;
    private int[][] adj; // 비가중: 0/1, 가중: 가중치 값

    public AdjacencyMatrix(int V) {
        this.V = V;
        adj = new int[V][V];
    }

    public void addEdge(int u, int v) {
        adj[u][v] = 1;
        adj[v][u] = 1; // 무방향
    }

    public void addWeightedEdge(int u, int v, int w) {
        adj[u][v] = w;
        adj[v][u] = w;
    }

    // 간선 존재 확인 O(1)
    public boolean hasEdge(int u, int v) {
        return adj[u][v] != 0;
    }
}
```

**공간 복잡도**: O(V²). 정점 수가 10,000이면 1억 칸 = 약 400MB (int 기준). 희소 그래프에서는 대부분의 칸이 0이라 낭비가 심하다.

**장점**: 두 정점의 연결 여부 또는 가중치를 O(1)에 확인 가능.

**단점**: 공간이 O(V²)이므로 V가 크면 사용 불가. 이웃 순회 시 O(V) (0인 칸도 다 확인해야 함).

### 3.3 표현 방법 선택 기준

```
V = 10,000, E = 50,000 (희소 그래프) 예시:
  인접 리스트: 약 (10,000 + 100,000) * 4바이트 ≈ 0.4 MB
  인접 행렬:   10,000 * 10,000 * 4바이트 = 400 MB

=> 실제 코딩 테스트와 업무에서 인접 리스트를 거의 항상 사용한다.
   인접 행렬은 V가 수백 이하이거나 "두 정점 연결 여부를 O(1)에 자주 물어봐야 하는" 경우에만 사용.
```

## 4. 너비 우선 탐색 (BFS)

### 4.1 개념과 원리

**BFS(Breadth-First Search, 너비 우선 탐색)**는 출발 정점에서 시작해 가장 가까운 정점들부터 차례로 방문하는 탐색 방식이다. 물에 돌을 던졌을 때 퍼져나가는 물결처럼, 거리 1인 정점을 모두 방문한 후 거리 2인 정점을 방문하고, 그 다음에 거리 3인 정점을 방문한다.

이를 가능하게 하는 자료구조가 바로 **큐(Queue)**다. 방문한 정점을 큐에 넣고, 큐에서 꺼낸 정점의 이웃을 다시 큐에 넣는 과정을 반복한다. 큐는 FIFO(First In First Out)이므로, 먼저 넣은(= 더 가까운) 정점이 먼저 나온다.

### 4.2 BFS 다이어그램 (물결 탐색)

```
그래프:
    0
   / \
  1   2
 / \   \
3   4   4  (정점 4는 1과 2 모두에 연결)
|
5

인접 리스트:
  0: [1, 2]
  1: [0, 3, 4]
  2: [0, 4]
  3: [1, 5]
  4: [1, 2]
  5: [3]

BFS(0) 수행 과정:

Step 0: 큐 = [0],  방문 = {0}
         0 꺼냄. 이웃 1, 2 → 큐에 추가

Step 1: 큐 = [1, 2],  방문 = {0, 1, 2}
         1 꺼냄. 이웃 0(방문됨), 3, 4 → 3, 4 큐에 추가

Step 2: 큐 = [2, 3, 4],  방문 = {0, 1, 2, 3, 4}
         2 꺼냄. 이웃 0(방문됨), 4(방문됨) → 추가 없음

Step 3: 큐 = [3, 4],  방문 = {0, 1, 2, 3, 4}
         3 꺼냄. 이웃 1(방문됨), 5 → 5 큐에 추가

Step 4: 큐 = [4, 5],  방문 = {0, 1, 2, 3, 4, 5}
         4 꺼냄. 이웃 1, 2 모두 방문됨 → 추가 없음

Step 5: 큐 = [5]
         5 꺼냄. 이웃 3(방문됨) → 추가 없음. 큐 비어 있으면 종료.

BFS(0) 방문 순서: [0, 1, 2, 3, 4, 5]

거리별 분류:
  거리 0: {0}
  거리 1: {1, 2}
  거리 2: {3, 4}
  거리 3: {5}
```

### 4.3 의사 코드

```
BFS(graph, start):
    queue ← 빈 큐
    visited ← 모두 false인 배열
    queue.enqueue(start)
    visited[start] ← true

    while queue가 비어있지 않으면:
        v ← queue.dequeue()
        v를 처리 (출력 등)
        for each neighbor w of v:
            if not visited[w]:
                visited[w] ← true
                queue.enqueue(w)
```

### 4.4 Java 코드

```java
import java.util.*;

public class BFS {

    // 기본 BFS - 방문 순서 반환
    public static List<Integer> bfs(List<List<Integer>> adj, int start) {
        int V = adj.size();
        boolean[] visited = new boolean[V];
        List<Integer> order = new ArrayList<>();
        Queue<Integer> queue = new LinkedList<>();

        visited[start] = true;
        queue.offer(start);

        while (!queue.isEmpty()) {
            int v = queue.poll();
            order.add(v);

            for (int w : adj.get(v)) {
                if (!visited[w]) {
                    visited[w] = true;
                    queue.offer(w);
                }
            }
        }
        return order;
    }

    // BFS + 최단 거리 계산 (비가중 그래프)
    public static int[] bfsDistance(List<List<Integer>> adj, int start) {
        int V = adj.size();
        int[] dist = new int[V];
        Arrays.fill(dist, -1); // -1 = 미방문
        Queue<Integer> queue = new LinkedList<>();

        dist[start] = 0;
        queue.offer(start);

        while (!queue.isEmpty()) {
            int v = queue.poll();
            for (int w : adj.get(v)) {
                if (dist[w] == -1) {
                    dist[w] = dist[v] + 1;
                    queue.offer(w);
                }
            }
        }
        return dist; // dist[v] = start에서 v까지 간선 수
    }

    // BFS + 경로 복원 (부모 배열 이용)
    public static List<Integer> bfsPath(List<List<Integer>> adj, int start, int end) {
        int V = adj.size();
        int[] parent = new int[V];
        Arrays.fill(parent, -1);
        boolean[] visited = new boolean[V];
        Queue<Integer> queue = new LinkedList<>();

        visited[start] = true;
        queue.offer(start);

        while (!queue.isEmpty()) {
            int v = queue.poll();
            if (v == end) break;

            for (int w : adj.get(v)) {
                if (!visited[w]) {
                    visited[w] = true;
                    parent[w] = v; // w에 오기 전 정점 기록
                    queue.offer(w);
                }
            }
        }

        // 경로 역추적
        if (parent[end] == -1 && start != end) return Collections.emptyList();
        List<Integer> path = new ArrayList<>();
        for (int v = end; v != -1; v = parent[v]) {
            path.add(0, v);
        }
        return path;
    }
}
```

### 4.5 BFS와 비가중 최단 경로

BFS는 비가중 그래프에서 출발 정점부터 각 정점까지의 **간선 수 기준 최단 경로**를 자연스럽게 구한다. 왜냐하면 BFS는 항상 거리 k인 정점을 모두 방문한 뒤에야 거리 k+1인 정점을 방문하기 때문이다. `dist` 배열에는 이 최단 거리가 담기고, `parent` 배열로 경로를 역추적할 수 있다.

```
경로 복원 예시:
  BFS(0 → 5):
  parent[1]=0, parent[2]=0, parent[3]=1, parent[4]=1, parent[5]=3

  5의 경로 역추적:
  5 → parent[5]=3 → parent[3]=1 → parent[1]=0
  결과: 0 → 1 → 3 → 5 (간선 3개, 최단 경로)
```

## 5. 깊이 우선 탐색 (DFS)

### 5.1 개념과 원리

**DFS(Depth-First Search, 깊이 우선 탐색)**는 한 방향으로 갈 수 있는 데까지 최대한 깊이 탐색한 후, 더 이상 갈 곳이 없으면 뒤로 돌아와(backtrack) 다른 방향을 탐색하는 방식이다. 미로에서 한 방향으로 벽에 부딪힐 때까지 걸어가다 막히면 되돌아오는 전략과 같다.

이를 가능하게 하는 자료구조는 **스택(Stack)**이다. 재귀 함수로 구현하면 함수 호출 스택이 곧 DFS의 스택 역할을 한다.

### 5.2 DFS 다이어그램 (경로 탐색)

```
같은 그래프에서 DFS(0) 수행:

    0
   / \
  1   2
 / \   \
3   4   4
|
5

Step 1: 0 방문 → 이웃 [1, 2] 중 1 선택
Step 2: 1 방문 → 이웃 [0(방문됨), 3, 4] 중 3 선택
Step 3: 3 방문 → 이웃 [1(방문됨), 5] 중 5 선택
Step 4: 5 방문 → 이웃 [3(방문됨)] → 더 이상 갈 곳 없음. 백트랙
Step 5: 3으로 돌아옴 → 남은 이웃 없음. 백트랙
Step 6: 1로 돌아옴 → 남은 이웃 4 선택
Step 7: 4 방문 → 이웃 [1(방문됨), 2(미방문)] → 2 선택
Step 8: 2 방문 → 이웃 [0(방문됨), 4(방문됨)] → 더 이상 갈 곳 없음. 백트랙
Step 9: 4로 돌아옴 → 남은 이웃 없음. 백트랙
Step 10: 1로 돌아옴 → 모든 이웃 방문됨. 백트랙
Step 11: 0으로 돌아옴 → 남은 이웃 2(방문됨) → 종료

DFS(0) 방문 순서: [0, 1, 3, 5, 4, 2]

(* 이웃 목록 순서와 구현 방식에 따라 달라질 수 있음)
```

### 5.3 의사 코드

```
DFS_Recursive(graph, v, visited):
    visited[v] ← true
    v를 처리 (출력 등)
    for each neighbor w of v:
        if not visited[w]:
            DFS_Recursive(graph, w, visited)

DFS_Iterative(graph, start):
    stack ← 빈 스택
    visited ← 모두 false
    stack.push(start)

    while stack이 비어있지 않으면:
        v ← stack.pop()
        if not visited[v]:
            visited[v] ← true
            v를 처리
            for each neighbor w of v (역순으로 추가하면 재귀와 같은 순서):
                if not visited[w]:
                    stack.push(w)
```

### 5.4 Java 코드 (재귀 + 반복)

```java
import java.util.*;

public class DFS {

    // 재귀 DFS
    public static void dfsRecursive(List<List<Integer>> adj, int v, boolean[] visited, List<Integer> order) {
        visited[v] = true;
        order.add(v);

        for (int w : adj.get(v)) {
            if (!visited[w]) {
                dfsRecursive(adj, w, visited, order);
            }
        }
    }

    // 재귀 DFS 호출 래퍼
    public static List<Integer> dfs(List<List<Integer>> adj, int start) {
        int V = adj.size();
        boolean[] visited = new boolean[V];
        List<Integer> order = new ArrayList<>();
        dfsRecursive(adj, start, visited, order);
        return order;
    }

    // 반복 DFS (스택 명시적 사용)
    public static List<Integer> dfsIterative(List<List<Integer>> adj, int start) {
        int V = adj.size();
        boolean[] visited = new boolean[V];
        List<Integer> order = new ArrayList<>();
        Deque<Integer> stack = new ArrayDeque<>();

        stack.push(start);

        while (!stack.isEmpty()) {
            int v = stack.pop();
            if (visited[v]) continue; // 이미 방문했으면 건너뜀
            visited[v] = true;
            order.add(v);

            // 역순으로 push해야 원래 순서대로 방문
            List<Integer> neighbors = adj.get(v);
            for (int i = neighbors.size() - 1; i >= 0; i--) {
                int w = neighbors.get(i);
                if (!visited[w]) {
                    stack.push(w);
                }
            }
        }
        return order;
    }

    // 연결 요소 개수 세기 (DFS 활용)
    public static int countComponents(List<List<Integer>> adj) {
        int V = adj.size();
        boolean[] visited = new boolean[V];
        int count = 0;

        for (int i = 0; i < V; i++) {
            if (!visited[i]) {
                dfsRecursive(adj, i, visited, new ArrayList<>());
                count++; // 새로운 연결 요소 발견
            }
        }
        return count;
    }
}
```

## 6. BFS와 DFS 비교: 자료구조가 알고리즘을 결정한다

```
BFS                          DFS
──────────────────────────────────────────────────────────
자료구조: 큐 (Queue, FIFO)    자료구조: 스택 (Stack, LIFO)
탐색 방식: 거리 순 (물결)     탐색 방식: 한 방향 끝까지 (한 우물)
방문 순서: 레벨 단위          방문 순서: 깊이 우선

코드의 핵심 차이:
  BFS: queue.offer(w);       DFS: stack.push(w);  (또는 재귀 호출)
       v = queue.poll();          v = stack.pop();

주요 용도:
  BFS: 비가중 최단 경로        DFS: 사이클 탐지
       두 정점 사이 최단 거리        위상 정렬
       레벨 단위 탐색               연결 요소
       이분 그래프 판정             백트래킹 (미로, N-Queen)
                                    강연결 요소

메모리 사용:
  BFS: 한 레벨 전체를 큐에 저장   DFS: 가장 깊은 경로만 스택에
       최악 O(V) (넓은 트리)            최악 O(V) (긴 경로)
```

핵심 교훈: **큐를 스택으로, 스택을 큐로 바꾸는 것만으로 탐색 방식이 달라진다.** 자료구조의 선택이 알고리즘의 성질을 결정한다. 이것이 자료구조를 공부하는 이유다.

## 7. 다익스트라 알고리즘: 최단 경로

### 7.1 문제 정의

BFS는 비가중 그래프의 최단 경로를 구한다. 그러나 현실의 지도 앱에서는 도로마다 이동 시간이 다르다. **가중치 있는 그래프**에서 한 정점에서 모든 정점까지의 최단 경로를 구하는 알고리즘이 **다익스트라(Dijkstra)**다.

### 7.2 예제 그래프

```
정점: 0, 1, 2, 3, 4
간선과 가중치:
  0 → 1: 4
  0 → 2: 1
  2 → 1: 2
  1 → 3: 1
  1 → 4: 5
  2 → 4: 8

그래프 시각화:
        4
   0 ──── 1
   │      │ \
 1 │    1 │  5
   │      │   \
   2 ─────────→ 3
    \     2  /
    8 \      /
       → 4
         (간선 2→1 = 2이므로 0→2→1 경로가 더 짧음)

0에서 시작한 최단 거리:
  0 → 0: 0
  0 → 1: 3  (0→2→1: 1+2=3, 직통 0→1: 4보다 짧음)
  0 → 2: 1
  0 → 3: 4  (0→2→1→3: 1+2+1=4)
  0 → 4: 8  (0→2→1→4: 1+2+5=8 또는 0→2→4: 1+8=9 중 8이 최소)
```

### 7.3 알고리즘 원리: 그리디 + 완화(Relaxation)

다익스트라는 두 가지 핵심 아이디어를 결합한다.

**아이디어 1: 완화(Relaxation)**
"현재 알고 있는 최단 거리보다 다른 정점 u를 경유하면 더 짧아지는가?"를 계속 확인하고 갱신한다.

```
완화 연산:
  if dist[u] + weight(u, v) < dist[v]:
      dist[v] = dist[u] + weight(u, v)
      parent[v] = u  // 경로 복원용
```

**아이디어 2: 탐욕적 선택(Greedy)**
아직 확정되지 않은 정점 중 `dist[v]`가 가장 작은 정점을 먼저 확정한다. 이 정점은 더 이상 갱신될 가능성이 없다(양수 가중치 전제). 가장 짧은 정점을 빠르게 꺼내려면 **우선순위 큐(최소 힙)**가 필요하다.

### 7.4 다익스트라 수행 과정

```
초기: dist = [0, INF, INF, INF, INF]
     우선순위 큐 = [(0, 0)]   // (거리, 정점)

Step 1: (0, 0) 꺼냄 → 정점 0 확정
  완화: 0→1: dist[1] = min(INF, 0+4) = 4
  완화: 0→2: dist[2] = min(INF, 0+1) = 1
  큐 = [(1, 2), (4, 1)]
  dist = [0, 4, 1, INF, INF]

Step 2: (1, 2) 꺼냄 → 정점 2 확정 (현재 최소)
  완화: 2→1: dist[1] = min(4, 1+2) = 3  ← 갱신!
  완화: 2→4: dist[4] = min(INF, 1+8) = 9
  큐 = [(3, 1), (4, 1)_old, (9, 4)]
  dist = [0, 3, 1, INF, 9]

Step 3: (3, 1) 꺼냄 → 정점 1 확정
  완화: 1→3: dist[3] = min(INF, 3+1) = 4
  완화: 1→4: dist[4] = min(9, 3+5) = 8  ← 갱신!
  큐 = [(4, 1)_old(무효), (4, 3), (8, 4), (9, 4)_old]
  dist = [0, 3, 1, 4, 8]

Step 4: (4, 1)_old 꺼냄 → 정점 1 이미 확정됨, 건너뜀 (lazy deletion)

Step 5: (4, 3) 꺼냄 → 정점 3 확정
  완화: 3의 이웃 없음 (이 예제에서)
  dist = [0, 3, 1, 4, 8]

Step 6: (8, 4) 꺼냄 → 정점 4 확정
  완화: 4의 이웃 없음
  dist = [0, 3, 1, 4, 8]

최종 결과: dist = [0, 3, 1, 4, 8]
  0→0: 0, 0→1: 3, 0→2: 1, 0→3: 4, 0→4: 8
```

### 7.5 Java 코드

```java
import java.util.*;

public class Dijkstra {

    static class Edge {
        int to, weight;
        Edge(int to, int weight) {
            this.to = to;
            this.weight = weight;
        }
    }

    /**
     * 다익스트라 알고리즘 - 우선순위 큐 (최소 힙) 구현
     * @param adj    가중치 인접 리스트
     * @param start  출발 정점
     * @return       dist[v] = start에서 v까지의 최단 거리 (-1이면 도달 불가)
     */
    public static long[] dijkstra(List<List<Edge>> adj, int start) {
        int V = adj.size();
        long[] dist = new long[V];
        Arrays.fill(dist, Long.MAX_VALUE); // 초기값: 무한대
        dist[start] = 0;

        // 우선순위 큐: (거리, 정점) - 거리가 작은 것이 먼저 나옴
        PriorityQueue<long[]> pq = new PriorityQueue<>(Comparator.comparingLong(a -> a[0]));
        pq.offer(new long[]{0, start});

        while (!pq.isEmpty()) {
            long[] curr = pq.poll();
            long d = curr[0];
            int v = (int) curr[1];

            // lazy deletion: 이미 더 짧은 경로로 확정된 정점이면 건너뜀
            if (d > dist[v]) continue;

            for (Edge e : adj.get(v)) {
                long newDist = dist[v] + e.weight;
                if (newDist < dist[e.to]) {
                    dist[e.to] = newDist;
                    pq.offer(new long[]{newDist, e.to}); // 갱신된 거리로 큐에 추가
                }
            }
        }

        return dist;
    }

    // 완화 연산 (핵심 로직만 분리)
    private static boolean relax(long[] dist, int u, int v, int weight, PriorityQueue<long[]> pq) {
        if (dist[u] != Long.MAX_VALUE && dist[u] + weight < dist[v]) {
            dist[v] = dist[u] + weight;
            pq.offer(new long[]{dist[v], v});
            return true; // 완화 성공
        }
        return false;
    }

    // 경로 복원 포함 버전
    public static long[] dijkstraWithPath(List<List<Edge>> adj, int start, int[] parent) {
        int V = adj.size();
        long[] dist = new long[V];
        Arrays.fill(dist, Long.MAX_VALUE);
        Arrays.fill(parent, -1);
        dist[start] = 0;

        PriorityQueue<long[]> pq = new PriorityQueue<>(Comparator.comparingLong(a -> a[0]));
        pq.offer(new long[]{0, start});

        while (!pq.isEmpty()) {
            long[] curr = pq.poll();
            long d = curr[0];
            int v = (int) curr[1];

            if (d > dist[v]) continue;

            for (Edge e : adj.get(v)) {
                long newDist = dist[v] + e.weight;
                if (newDist < dist[e.to]) {
                    dist[e.to] = newDist;
                    parent[e.to] = v; // 경로 복원: e.to에 오기 직전 정점
                    pq.offer(new long[]{newDist, e.to});
                }
            }
        }
        return dist;
    }

    // 경로 출력
    public static List<Integer> getPath(int[] parent, int end) {
        List<Integer> path = new ArrayList<>();
        for (int v = end; v != -1; v = parent[v]) {
            path.add(0, v);
        }
        return path;
    }

    public static void main(String[] args) {
        int V = 5;
        List<List<Edge>> adj = new ArrayList<>();
        for (int i = 0; i < V; i++) adj.add(new ArrayList<>());

        adj.get(0).add(new Edge(1, 4));
        adj.get(0).add(new Edge(2, 1));
        adj.get(2).add(new Edge(1, 2));
        adj.get(1).add(new Edge(3, 1));
        adj.get(1).add(new Edge(4, 5));
        adj.get(2).add(new Edge(4, 8));

        long[] dist = dijkstra(adj, 0);
        System.out.println("0에서 각 정점까지 최단 거리:");
        for (int i = 0; i < V; i++) {
            System.out.printf("  0 → %d: %d%n", i, dist[i]);
        }
        // 출력:
        // 0 → 0: 0
        // 0 → 1: 3  (0→2→1: 1+2=3)
        // 0 → 2: 1
        // 0 → 3: 4
        // 0 → 4: 8
    }
}
```

### 7.6 우선순위 큐를 쓰는 이유와 Lazy Deletion

우선순위 큐 없이 단순히 `dist` 배열 전체를 매번 선형 탐색해 최솟값을 찾으면 O(V²) 시간이 걸린다 (정점 V번 × 배열 탐색 V). 우선순위 큐(최소 힙)를 사용하면 최솟값 추출이 O(log V)이므로 전체 복잡도가 O((V+E) log V)로 개선된다.

그런데 한 가지 문제가 있다. dist[v]가 갱신될 때, 우선순위 큐 안에 이미 들어있는 (old_dist, v) 항목을 꺼내 수정하기가 어렵다(Java의 PriorityQueue는 특정 원소를 O(1)에 찾을 방법이 없다). 해결책이 **Lazy Deletion**이다. 갱신 시 새 (new_dist, v)를 큐에 추가하고, 나중에 꺼냈을 때 `d > dist[v]`이면 이미 더 좋은 경로로 처리된 것이므로 그냥 건너뛴다. 조금 비효율적이지만 구현이 간단하고 실전에서 충분히 빠르다.

## 8. 다익스트라 정당성 소절

### 8.1 왜 "가장 가까운 것 확정"이 옳은가

다익스트라가 올바른 이유를 귀납적으로 스케치해 보자.

**주장**: 우선순위 큐에서 꺼낸 정점 u에서의 `dist[u]`는 진짜 최단 거리다.

**증명 스케치 (귀류법)**:

1. 처음 꺼낸 정점은 출발점 s 자신이다. `dist[s] = 0`은 명백히 최단 거리.
2. 이미 확정된 정점들의 집합을 S라 하자. 지금 꺼낸 정점 u의 `dist[u]`가 최단 거리가 아니라고 가정하자.
3. 그러면 S를 통하지 않는 다른 경로 P가 존재하고, P 위에서 S에서 S 바깥으로 처음 넘어가는 간선 (x, y)가 있다 (x ∈ S, y ∉ S).
4. `dist[y] ≤ P의 비용 < dist[u]`이어야 하는데, 그러면 다익스트라는 u보다 y를 먼저 꺼냈을 것이다. 모순.
5. 따라서 u의 `dist[u]`는 최단 거리다.

이 논리가 성립하려면 **모든 가중치가 양수여야 한다**. 양수 가중치이기 때문에 간선 (x, y)를 지나면 경로 비용이 반드시 증가하고, 따라서 y를 경유하는 경로는 u보다 길 수밖에 없다는 보장이 생긴다.

### 8.2 왜 음수 가중치에서 깨지는가

```
음수 가중치 반례:
    0 --5--> 1
    |        |
    2       -4
    |        |
    v        v
    2 -------> ...

Step 1: (0, 0) 꺼냄, dist[1] = 5, dist[2] = 2
Step 2: (2, 2) 꺼냄 → 정점 2 확정. 이후 2→1 간선이 없다고 가정.
Step 3: (5, 1) 꺼냄 → 정점 1 확정. dist[1] = 5라고 확정.

그런데 만약 0→1→2→1 경로가 있고 1→2 가중치가 -4라면?
0→1(5) → 1→2(-4): 0→2 총 비용 = 1
정점 2가 이미 확정된 후에야 이 경로를 발견하지만, 확정은 취소 불가.
=> 다익스트라는 dist[2] = 2를 최단으로 잘못 확정함.

더 심각한 경우: 음수 사이클 (총 비용이 마이너스인 순환)
A --1--> B ---3--> C
^        |
|       -5
 \--------

A→B→C 경로: 1+3=4
A→B→A: 1+(-5)=-4  (음수 사이클)
이 사이클을 계속 돌면 거리가 -무한대로 발산 → "최단 경로"가 의미 없어짐.
```

**해결책**: 음수 가중치가 있으면 **벨만-포드(Bellman-Ford)** 알고리즘을 사용해야 한다. 음수 사이클이 있는지도 탐지할 수 있다.

## 9. DFS 응용: 사이클·위상정렬·연결요소

DFS는 단순 탐색 외에도 그래프의 중요한 구조적 성질을 파악하는 데 활용된다.

### 9.1 연결 요소 (Connected Components)

```java
// 각 정점이 어느 연결 요소에 속하는지 구분
public static int[] findComponents(List<List<Integer>> adj) {
    int V = adj.size();
    int[] component = new int[V];
    Arrays.fill(component, -1);
    int compId = 0;

    for (int i = 0; i < V; i++) {
        if (component[i] == -1) {
            // DFS로 연결 요소 전체에 compId 부여
            dfsLabel(adj, i, component, compId++);
        }
    }
    return component; // component[v] = v가 속한 연결 요소 번호
}

private static void dfsLabel(List<List<Integer>> adj, int v, int[] component, int id) {
    component[v] = id;
    for (int w : adj.get(v)) {
        if (component[w] == -1) {
            dfsLabel(adj, w, component, id);
        }
    }
}
```

### 9.2 사이클 탐지 (Cycle Detection)

무방향 그래프에서 DFS 중 방문된 정점을 다시 만나면 사이클이다. 단, 직접 부모로 되돌아가는 것은 사이클이 아니다.

```java
// 무방향 그래프 사이클 탐지
public static boolean hasCycle(List<List<Integer>> adj) {
    int V = adj.size();
    boolean[] visited = new boolean[V];
    for (int i = 0; i < V; i++) {
        if (!visited[i]) {
            if (dfsCycle(adj, i, visited, -1)) return true;
        }
    }
    return false;
}

private static boolean dfsCycle(List<List<Integer>> adj, int v, boolean[] visited, int parent) {
    visited[v] = true;
    for (int w : adj.get(v)) {
        if (!visited[w]) {
            if (dfsCycle(adj, w, visited, v)) return true;
        } else if (w != parent) {
            // 방문됐는데 부모가 아님 → 사이클 발견
            return true;
        }
    }
    return false;
}
```

방향 그래프에서는 "현재 DFS 경로 위에 있는 정점"을 구분해야 한다. `visited` 대신 `inStack` 배열을 별도로 관리한다.

### 9.3 위상 정렬 (Topological Sort)

방향 비순환 그래프(DAG)에서 간선 방향을 거스르지 않는 정점 순서를 구하는 것이 위상 정렬이다. 예: 대학 선수과목(자료구조를 듣기 전에 프로그래밍 기초를 들어야 한다).

DFS를 이용한 위상 정렬: 정점의 DFS가 완전히 끝나는 시점에 스택에 push하고, 마지막에 스택을 뒤집으면 위상 순서가 된다.

```java
public static List<Integer> topologicalSort(List<List<Integer>> adj) {
    int V = adj.size();
    boolean[] visited = new boolean[V];
    Deque<Integer> stack = new ArrayDeque<>();

    for (int i = 0; i < V; i++) {
        if (!visited[i]) {
            dfsTopoSort(adj, i, visited, stack);
        }
    }

    List<Integer> result = new ArrayList<>();
    while (!stack.isEmpty()) {
        result.add(stack.pop());
    }
    return result;
}

private static void dfsTopoSort(List<List<Integer>> adj, int v, boolean[] visited, Deque<Integer> stack) {
    visited[v] = true;
    for (int w : adj.get(v)) {
        if (!visited[w]) {
            dfsTopoSort(adj, w, visited, stack);
        }
    }
    stack.push(v); // DFS 완료 시점에 push
}
```

### 9.4 이분 그래프 판정 (Bipartite Check)

이분 그래프란 정점을 두 그룹 A, B로 나눠 모든 간선이 A↔B 사이에만 있는 그래프다. BFS로 정점을 0번 색, 1번 색으로 번갈아 칠하다가 같은 색끼리 인접하면 이분 그래프가 아니다.

```java
public static boolean isBipartite(List<List<Integer>> adj) {
    int V = adj.size();
    int[] color = new int[V];
    Arrays.fill(color, -1); // -1 = 미채색

    for (int i = 0; i < V; i++) {
        if (color[i] == -1) {
            if (!bfsColor(adj, i, color)) return false;
        }
    }
    return true;
}

private static boolean bfsColor(List<List<Integer>> adj, int start, int[] color) {
    Queue<Integer> queue = new LinkedList<>();
    color[start] = 0;
    queue.offer(start);

    while (!queue.isEmpty()) {
        int v = queue.poll();
        for (int w : adj.get(v)) {
            if (color[w] == -1) {
                color[w] = 1 - color[v]; // 반대 색
                queue.offer(w);
            } else if (color[w] == color[v]) {
                return false; // 같은 색끼리 인접 → 이분 그래프 아님
            }
        }
    }
    return true;
}
```

## 10. 더 나아가기: 벨만-포드, 플로이드-워셜, A*

이 교재에서 깊이 다루지는 않지만, 그래프 최단 경로 알고리즘에는 다익스트라 외에도 여러 가지가 있다. 간략히 소개한다.

### 10.1 벨만-포드 (Bellman-Ford)

음수 가중치 간선이 있어도 올바르게 동작하는 최단 경로 알고리즘이다. 모든 간선을 (V-1)번 반복해 완화하고, 한 번 더 완화가 발생하면 음수 사이클이 있음을 알 수 있다.

- 시간 복잡도: O(V × E)
- 다익스트라보다 느리지만 음수 가중치와 음수 사이클 탐지 가능.

```java
public static long[] bellmanFord(int V, int[][] edges, int start) {
    // edges[i] = {u, v, w}
    long[] dist = new long[V];
    Arrays.fill(dist, Long.MAX_VALUE);
    dist[start] = 0;

    // V-1번 완화
    for (int i = 0; i < V - 1; i++) {
        for (int[] e : edges) {
            int u = e[0], v = e[1], w = e[2];
            if (dist[u] != Long.MAX_VALUE && dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
            }
        }
    }

    // 음수 사이클 검사: V번째 완화가 발생하면 사이클 있음
    for (int[] e : edges) {
        int u = e[0], v = e[1], w = e[2];
        if (dist[u] != Long.MAX_VALUE && dist[u] + w < dist[v]) {
            System.out.println("음수 사이클 존재!");
            return null;
        }
    }
    return dist;
}
```

### 10.2 플로이드-워셜 (Floyd-Warshall)

모든 정점 쌍 사이의 최단 거리를 한 번에 구한다. 음수 가중치도 허용(단, 음수 사이클 없어야).

- 시간 복잡도: O(V³)
- 공간 복잡도: O(V²)
- 정점 수가 수백 이하인 경우에 적합.

```java
public static long[][] floydWarshall(long[][] graph) {
    int V = graph.length;
    long[][] dist = new long[V][V];
    for (long[] row : dist) Arrays.fill(row, Long.MAX_VALUE / 2);
    for (int i = 0; i < V; i++) dist[i][i] = 0;
    // 초기 간선 설정
    for (int u = 0; u < V; u++)
        for (int v = 0; v < V; v++)
            if (graph[u][v] != 0) dist[u][v] = graph[u][v];

    // k: 경유 정점
    for (int k = 0; k < V; k++)
        for (int i = 0; i < V; i++)
            for (int j = 0; j < V; j++)
                if (dist[i][k] + dist[k][j] < dist[i][j])
                    dist[i][j] = dist[i][k] + dist[k][j];

    return dist; // dist[i][j] = i에서 j까지 최단 거리
}
```

### 10.3 A* (에이스타)

다익스트라에 **휴리스틱(heuristic)** 함수를 더한 알고리즘이다. 목적지까지의 예상 거리 `h(v)`를 `dist[v] + h(v)` 형태로 우선순위 큐에 활용해 목적지 방향을 더 빠르게 탐색한다. 지도 내비게이션, 게임 경로 탐색에 널리 사용된다. 휴리스틱이 "실제 거리를 초과하지 않음(admissible)"을 보장하면 최단 경로를 찾는다.

## 12. 흔한 오해와 함정

## ⚠️ 흔한 오해와 함정

**함정 1: BFS로 가중치 있는 그래프의 최단 경로를 구하려 한다**

BFS는 간선 수 기준으로 가장 가까운 정점을 방문한다. 가중치가 있으면 간선 수가 적어도 가중치 합이 클 수 있다.

```
0 --1--> 1 --1--> 2    (BFS: 0→1→2 = 2간선)
|                      (하지만 0→2에 간선 하나의 비용이 100이라면?)
└──────────────────→2  (간선 1개지만 비용 100)

BFS는 "0→1→2"를 최단으로 선택하지만 실제 최단은 간선 수 기준이지 가중치 합 기준이 아님.
모든 간선 가중치가 같을 때만 BFS = 최단 경로.
```

**함정 2: DFS 재귀로 깊은 그래프를 탐색하면 스택 오버플로**

Java의 기본 스레드 스택 크기는 약 512KB~1MB다. 정점 수만 개인 선형 그래프(0→1→2→...→N)를 재귀 DFS로 탐색하면 재귀 깊이가 N에 달해 `StackOverflowError`가 발생한다. 해결책은 반복 DFS를 사용하거나, JVM 스택 크기를 늘리는 것이다.

```java
// 위험: 긴 선형 그래프에서 StackOverflowError
void dfs(int v) { dfs(v + 1); } // 재귀 N번

// 안전: 반복 DFS 사용
Deque<Integer> stack = new ArrayDeque<>();
stack.push(start);
while (!stack.isEmpty()) { ... }
```

**함정 3: 방문 표시를 하지 않으면 무한 루프**

사이클이 있는 그래프에서 `visited` 배열 없이 BFS나 DFS를 수행하면 같은 정점을 계속 방문하며 무한 루프에 빠진다.

```
그래프: 0 -- 1 -- 2 -- 0  (삼각형, 사이클)
방문 표시 없는 BFS:
  0 방문 → 1, 2 큐에 추가
  1 방문 → 0, 2 다시 큐에 추가
  2 방문 → 1, 0 다시 큐에 추가
  ... 무한 반복
```

**함정 4: 다익스트라에 음수 가중치 간선을 사용**

다익스트라는 "한번 확정된 정점은 더 이상 갱신되지 않는다"는 전제가 있다. 음수 간선이 있으면 나중에 발견되는 경로가 더 짧을 수 있어 이 전제가 깨진다. 음수 가중치 그래프에는 반드시 벨만-포드를 사용할 것.

**함정 5: 인접 행렬로 대규모 그래프를 표현하려 한다**

V = 100,000인 그래프의 인접 행렬은 100,000 × 100,000 = 10^10개의 int = 약 40TB. 명백히 불가능하다. V > 1,000 수준이면 인접 리스트를 사용해야 한다.

### 문제 1. 연결 요소 개수 세기 (난이도: ★☆☆)

무방향 그래프가 주어질 때, 연결 요소(connected component)의 개수를 반환하는 함수를 작성하라.

**입력 예시**:
```
V = 7, 간선: (0,1), (1,2), (3,4), (5,6)
```

**기대 출력**: 3 (연결 요소: {0,1,2}, {3,4}, {5,6})

**힌트**: 모든 정점을 순회하면서 아직 방문하지 않은 정점에서 BFS 또는 DFS를 시작한다. DFS/BFS 한 번이 끝날 때마다 카운터를 증가시킨다.

<details>
<summary>해설</summary>

```java
public static int countConnectedComponents(List<List<Integer>> adj) {
    int V = adj.size();
    boolean[] visited = new boolean[V];
    int count = 0;

    for (int i = 0; i < V; i++) {
        if (!visited[i]) {
            bfs(adj, i, visited); // 또는 dfs
            count++;
        }
    }
    return count;
}

private static void bfs(List<List<Integer>> adj, int start, boolean[] visited) {
    Queue<Integer> queue = new LinkedList<>();
    visited[start] = true;
    queue.offer(start);
    while (!queue.isEmpty()) {
        int v = queue.poll();
        for (int w : adj.get(v)) {
            if (!visited[w]) {
                visited[w] = true;
                queue.offer(w);
            }
        }
    }
}
```

핵심 아이디어: 연결 요소마다 BFS/DFS를 한 번 수행한다. 방문 표시로 이미 처리된 정점은 건너뛴다. 전체 복잡도는 O(V+E).

</details>

### 문제 2. 사이클 탐지 (난이도: ★★☆)

무방향 그래프에서 사이클이 존재하는지 판별하는 함수를 작성하라.

**입력 예시**:
```
그래프 A: V=4, 간선: (0,1),(1,2),(2,3)          → false (트리, 사이클 없음)
그래프 B: V=4, 간선: (0,1),(1,2),(2,0),(2,3)    → true  (0-1-2-0 사이클)
```

**힌트**: DFS 탐색 시 이미 방문한 정점을 만나더라도, 그 정점이 직접 부모(parent)이면 사이클이 아니다. 부모가 아닌 방문된 정점을 만나면 사이클.

<details>
<summary>해설</summary>

```java
public static boolean hasCycle(List<List<Integer>> adj) {
    int V = adj.size();
    boolean[] visited = new boolean[V];
    for (int i = 0; i < V; i++) {
        if (!visited[i]) {
            if (dfs(adj, i, visited, -1)) return true;
        }
    }
    return false;
}

private static boolean dfs(List<List<Integer>> adj, int v, boolean[] visited, int parent) {
    visited[v] = true;
    for (int w : adj.get(v)) {
        if (!visited[w]) {
            if (dfs(adj, w, visited, v)) return true;
        } else if (w != parent) {
            return true; // 사이클 발견
        }
    }
    return false;
}
```

주의: 방향 그래프의 사이클 탐지는 다르다. `inStack` 배열을 별도로 유지해야 한다.

</details>

### 문제 3. 위상 정렬 구현 (난이도: ★★☆)

방향 비순환 그래프(DAG)가 주어질 때, 위상 정렬 결과를 출력하라. 위상 정렬이 불가능한 경우(사이클 존재)는 빈 리스트를 반환하라.

**입력 예시**:
```
V=6, 간선: 5→2, 5→0, 4→0, 4→1, 2→3, 3→1
기대 출력 (가능한 여러 답 중 하나): [4, 5, 2, 0, 3, 1]
```

**힌트**: Kahn의 알고리즘(진입 차수 0인 정점부터 BFS)이나 DFS 후위순회(Post-order DFS) 방법을 사용할 수 있다.

<details>
<summary>해설 (Kahn's Algorithm, BFS 기반)</summary>

```java
public static List<Integer> topologicalSortKahn(List<List<Integer>> adj) {
    int V = adj.size();
    int[] inDegree = new int[V];

    // 진입 차수 계산
    for (int u = 0; u < V; u++)
        for (int v : adj.get(u))
            inDegree[v]++;

    // 진입 차수 0인 정점을 큐에
    Queue<Integer> queue = new LinkedList<>();
    for (int i = 0; i < V; i++)
        if (inDegree[i] == 0) queue.offer(i);

    List<Integer> result = new ArrayList<>();
    while (!queue.isEmpty()) {
        int v = queue.poll();
        result.add(v);
        for (int w : adj.get(v)) {
            inDegree[w]--;
            if (inDegree[w] == 0) queue.offer(w);
        }
    }

    // 결과 크기가 V 미만이면 사이클 존재
    return result.size() == V ? result : Collections.emptyList();
}
```

Kahn 알고리즘의 핵심: 진입 차수가 0인 정점(선수 과목 없는 과목)부터 차례로 제거하면서 위상 순서를 얻는다. 제거 후 이웃의 진입 차수를 감소시켜 새로 0이 된 정점을 큐에 추가한다.

</details>

### 문제 4. 이분 그래프 판정 (난이도: ★★☆)

주어진 무방향 그래프가 이분 그래프(Bipartite Graph)인지 판별하라. 이분 그래프는 정점을 두 그룹으로 나눠 같은 그룹 내 정점끼리는 간선이 없는 그래프다.

**입력 예시**:
```
그래프 A: V=4, 간선: (0,1),(1,2),(2,3),(3,0)  → true  (정사각형, 2색 가능)
그래프 B: V=3, 간선: (0,1),(1,2),(2,0)         → false (삼각형, 2색 불가)
```

**힌트**: BFS/DFS로 인접 정점에 반대 색을 칠한다. 같은 색끼리 인접하면 이분 그래프가 아니다. 홀수 길이 사이클이 있으면 이분 그래프가 될 수 없다.

<details>
<summary>해설</summary>

```java
public static boolean isBipartite(List<List<Integer>> adj) {
    int V = adj.size();
    int[] color = new int[V];
    Arrays.fill(color, -1);

    for (int i = 0; i < V; i++) {
        if (color[i] != -1) continue;
        Queue<Integer> queue = new LinkedList<>();
        color[i] = 0;
        queue.offer(i);

        while (!queue.isEmpty()) {
            int v = queue.poll();
            for (int w : adj.get(v)) {
                if (color[w] == -1) {
                    color[w] = 1 - color[v];
                    queue.offer(w);
                } else if (color[w] == color[v]) {
                    return false; // 같은 색끼리 인접
                }
            }
        }
    }
    return true;
}
```

이분 그래프 ⟺ 홀수 길이 사이클이 없는 그래프. 2-coloring이 가능하면 이분 그래프다.

</details>

### 문제 5. 벨만-포드 구현 및 음수 사이클 탐지 (난이도: ★★★)

음수 가중치 간선이 포함된 방향 그래프가 주어질 때, 출발점 s에서 모든 정점까지의 최단 거리를 구하라. 음수 사이클이 존재하면 "음수 사이클 있음"을 출력하라.

**입력 예시**:
```
V=5, s=0
간선: (0,1,6), (0,2,7), (1,3,5), (1,4,-4), (2,3,-3), (2,4,9),
     (3,1,-2), (4,0,2), (4,3,7)

기대 출력: dist = [0, 2, 7, 4, -2]
```

**힌트**: 모든 간선을 (V-1)번 완화한다. V번째 완화 시도에서 여전히 갱신이 발생하면 음수 사이클.

<details>
<summary>해설</summary>

```java
public static long[] bellmanFord(int V, int[][] edges, int start) {
    long[] dist = new long[V];
    Arrays.fill(dist, Long.MAX_VALUE);
    dist[start] = 0;

    // V-1번 완화
    for (int i = 0; i < V - 1; i++) {
        boolean updated = false;
        for (int[] e : edges) {
            int u = e[0], v = e[1], w = e[2];
            if (dist[u] != Long.MAX_VALUE && dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                updated = true;
            }
        }
        if (!updated) break; // 조기 종료 최적화
    }

    // V번째 완화 시도: 음수 사이클 탐지
    for (int[] e : edges) {
        int u = e[0], v = e[1], w = e[2];
        if (dist[u] != Long.MAX_VALUE && dist[u] + w < dist[v]) {
            System.out.println("음수 사이클 존재!");
            return null;
        }
    }
    return dist;
}
```

왜 (V-1)번인가? 최단 경로는 최대 (V-1)개의 간선을 포함한다 (더 많으면 사이클이 생기므로). 따라서 (V-1)번 완화하면 모든 최단 경로가 확정된다.

</details>

## 14. 요약

```
핵심 내용 정리:

그래프 G = (V, E): 정점과 간선의 집합.
  - 방향/무방향, 가중/비가중의 네 가지 조합
  - 현실 세계의 대부분의 관계망을 모델링

표현 방법:
  인접 리스트: O(V+E) 공간, 희소 그래프에 적합 (거의 항상 이것)
  인접 행렬:  O(V²) 공간, 연결 확인 O(1), 밀집 그래프나 V가 작을 때

BFS (너비 우선):
  - 큐(FIFO) 사용
  - 레벨 단위 탐색, 비가중 최단 경로
  - O(V+E) 시간, O(V) 공간

DFS (깊이 우선):
  - 스택(재귀 콜스택) 사용
  - 한 방향 끝까지, 백트래킹
  - O(V+E) 시간, O(V) 공간
  - 응용: 사이클 탐지, 위상 정렬, 연결 요소, 이분 그래프

다익스트라:
  - 우선순위 큐(최소 힙) 사용
  - 양수 가중치 그래프에서 단일 출발점 최단 경로
  - O((V+E) log V) 시간
  - 음수 가중치 불가 → 벨만-포드 사용

완화 (Relaxation):
  if dist[u] + w(u,v) < dist[v]: dist[v] 갱신
  → 다익스트라, 벨만-포드의 핵심 연산

자료구조와 알고리즘의 연결:
  BFS  = 큐  = 5장
  DFS  = 스택 = 5장
  다익스트라 = 우선순위 큐 = 6장
  각 자료구조의 성질이 탐색 방식을 결정한다.
```

## 15. 마치며: 전공자의 사고방식

### 이 교재를 마친 당신이 달라진 것들

7개 장을 모두 공부했다. 처음에는 Big-O가 낯설었고, 재귀가 무서웠고, 힙이 왜 필요한지 몰랐다. 이제 달라진 것들을 정리해 보자.

**코드를 보면 Big-O가 보인다.** 이중 for 루프를 보면 O(n²)을, 재귀 이진 분할을 보면 O(n log n)을, 한 번의 순회를 보면 O(n)을 즉각 읽어낸다. Java 표준 라이브러리를 쓸 때도 `ArrayList.get()`은 O(1)이지만 `LinkedList.get()`은 O(n)임을 알고, 상황에 맞는 자료구조를 선택한다.

**자료구조를 골라 쓸 수 있다.** "순서 중요하고 임의 접근 많으면 ArrayList, 삽입·삭제 빈번하면 LinkedList, 키-값 빠른 검색이면 HashMap, 순서 있는 범위 검색이면 TreeMap, 최솟값 빠르게 꺼내려면 PriorityQueue." 이 판단이 실력이다.

**재귀의 두 얼굴을 안다.** 재귀는 편리하지만 스택 공간을 소비하고, 깊은 재귀는 스택 오버플로 위험이 있다. 동시에 재귀는 분할정복의 도구이며, 병합 정렬과 퀵 정렬이 O(n log n)에 도달하는 방법도 재귀적 사고에서 나온다.

**지수 시간을 다항 시간으로 끌어내릴 수 있다.** 피보나치의 순진한 재귀는 O(2^n)이지만 메모이제이션을 더하면 O(n)이 된다. DP는 "이미 계산한 결과를 다시 계산하지 않는다"는 단순한 원칙이 알고리즘을 혁신한다. 중복 부분 문제를 발견하고 상태를 정의하는 능력이 DP의 핵심이다.

**그래프를 다룰 수 있다.** 현실의 네트워크, 지도, 의존 관계를 그래프로 모델링하고, BFS/DFS로 탐색하고, 다익스트라로 최단 경로를 찾는다. 큐가 BFS를 만들고, 스택이 DFS를 만들고, 우선순위 큐가 다익스트라를 만드는 것을 몸으로 안다.

### 다음 단계: 배움은 여기서 끝나지 않는다

이 교재는 입문이다. 전공자의 여정에서 다음 목적지들이 기다리고 있다.

**균형 트리 구현**: 이 책에서는 BST가 최악의 경우 O(n)으로 퇴화하는 문제를 언급했다. AVL 트리와 레드-블랙 트리는 회전(rotation)으로 균형을 유지해 O(log n)을 보장한다. Java의 `TreeMap`이 레드-블랙 트리로 구현되어 있다. 직접 구현해 보면 회전 연산의 아름다움을 느낄 수 있다.

**해시 함수 설계와 충돌 해결**: 좋은 해시 함수란 무엇인가? 개방 주소법(Open Addressing)과 체이닝(Chaining)은 각각 언제 유리한가? 부하율(load factor)을 어느 수준으로 유지해야 하는가? `HashMap`의 내부를 직접 만들어 보면 O(1) 평균이 어떻게 달성되는지 이해하게 된다.

**고급 그래프 알고리즘**: 최소 신장 트리(MST, Minimum Spanning Tree) - 크루스칼과 프림 알고리즘, 위상 정렬(Topological Sort), 강연결 요소(Strongly Connected Components) - 코사라주와 타잔 알고리즘. 이들은 단순 탐색을 넘어 그래프의 구조적 성질을 파악하는 도구다.

**NP-완전과 근사 알고리즘**: 모든 문제가 다항 시간에 풀리는 것은 아니다. 외판원 문제(TSP), 배낭 문제(Knapsack)처럼 지수 시간이 걸리는 NP-완전 문제들이 있다. 이를 "완벽하지 않지만 빠르게" 근사적으로 푸는 전략도 중요한 실전 무기다.

### 직접 구현하고 측정하라

이 책을 읽은 것만으로는 부족하다. 코드를 직접 짜고, 실행하고, 시간을 측정하라.

- 정렬 알고리즘 세 가지(삽입, 병합, 힙)를 직접 구현하고, 같은 입력에 대해 나노초 단위로 비교하라. Big-O가 상수를 숨긴다는 것이 실감된다.
- BST에 정렬된 입력을 넣고 `O(n²)`이 되는 것을 직접 확인하라. 그런 다음 AVL 트리로 바꿔 O(n log n)을 체험하라.
- 다익스트라와 벨만-포드를 같은 양수 가중치 그래프에서 비교하라. 음수 가중치 그래프에서 다익스트라가 틀리는 것을 반례로 확인하라.

이론과 실측이 만나는 지점에서 진짜 이해가 일어난다.

### 마지막으로

알고리즘과 자료구조는 프로그래밍의 문법이 아니라 **사고 방식**이다. 문제를 보고 "이건 그래프로 모델링하면 BFS 문제야", "이건 DP인데 상태가 뭐지?", "이 자료구조가 왜 이 복잡도를 갖는지"를 묻는 습관이 전공자를 비전공자로부터 구분한다.

코딩 테스트를 통과하는 것도 중요하지만, 그보다 더 중요한 것은 코드의 성능을 예측하고 트레이드오프를 이해하며 팀원에게 설명할 수 있는 능력이다. 이 교재가 그 능력의 첫 번째 기둥이 되길 바란다.

직접 구현하고, 측정하고, 틀리고, 다시 짜라. 그것이 전공자의 길이다.
