# CS 알고리즘 개념서: 비전공자에서 전공자로

> "라이브러리를 **쓸 줄 아는** 사람"과 "그 라이브러리를 **만들 수 있는** 사람"의 차이는
> 알고리즘과 자료구조를 아느냐에 있다. 이 책은 그 격차를 메운다.

---

## 이 책은 무엇인가

당신은 이미 Java를 배웠고 `ArrayList`, `HashMap`, `TreeMap`, `PriorityQueue`를 **써봤다**.
하지만 이런 질문에 답할 수 있는가?

- `ArrayList.get(i)`는 왜 빠르고, `LinkedList.get(i)`는 왜 느린가?
- `TreeMap`은 어떻게 항상 정렬된 상태를 유지하면서도 빠른가?
- `PriorityQueue`에서 최솟값이 어떻게 O(1)로 튀어나오는가?
- 왜 어떤 코드는 데이터가 커지면 갑자기 멈춰버리는가?

이 질문들의 답은 전부 **알고리즘과 자료구조**에 있다.
전공자(CS major)가 4년간 배우는 것의 핵심 뼈대를, 이 책은 "왜?"를 중심으로 압축한다.

## 학습 목표

이 책을 끝까지 통과하면 다음을 할 수 있다.

1. **Big-O를 유도한다** — 코드를 보고 O(n²)/O(log n)을 스스로 계산한다.
2. **탐색·정렬**의 여러 알고리즘과 그 트레이드오프(안정성·제자리·속도)를 안다.
3. **재귀와 DP**로 지수 시간 문제를 다항 시간으로 낮춘다.
4. 자료구조를 **밑바닥부터 구현**한다 — 동적 배열, 연결 리스트, 스택, 큐, BST, 힙, 그래프.
5. 위 구현이 표준 라이브러리(`ArrayList`/`TreeMap`/`PriorityQueue`)의 **원리임**을 안다.
6. 그래프 탐색(BFS/DFS)과 최단 경로(다익스트라)를 이해한다.

## 이 책을 읽는 법

- **반드시 예제를 직접 돌려보라.** 각 장에는 실행 가능한 Java 예제가 짝지어져 있다.
  이론을 읽는 것과, 화면에서 O(n²)이 폭발하는 것을 **눈으로 보는 것**은 완전히 다르다.
- 이미 배운 Java 컬렉션(`docs/JAVA_교재/심화/README.md` Chapter 5)과 계속 연결한다.
  "아, `HashMap`이 이렇게 생긴 거였구나"를 느끼는 것이 목표다.
- 각 장 끝의 **연습문제**를 손으로 풀어라. "안다"와 "만들 수 있다"의 거리는 오직 손으로 짠 코드가 메운다.

## 추천 학습 순서

```
  1 복잡도(Big-O) ─── 모든 장의 언어. 반드시 먼저.
        │
        ├──> 2 탐색 ─────┐
        │                ├──> 이 둘은 "절반 버리기"로 연결된다
        └──> 3 정렬 ─────┘
        │
        └──> 4 재귀와 DP ─── 분할 정복(정렬)의 재귀를 여기서 심화
                 │
                 └──> 5 자료구조 구현 ─── ArrayList/LinkedList/Stack/Queue의 정체
                          │
                          └──> 6 트리와 힙 ─── TreeMap/PriorityQueue의 정체
                                   │
                                   └──> 7 그래프 ─── 5·6의 큐·힙을 도구로 사용
```

1장은 나머지 전부의 공용어이므로 **가장 먼저** 읽는다.
이후 2·3장(탐색·정렬)을 함께 보면 "절반씩 버리기"라는 같은 아이디어가 두 곳에서 나타남을 느낄 수 있다.
5→6→7장은 자료구조가 뒤 장의 도구가 되므로 **순서대로** 읽는 것을 권한다.

## 실행 데모 안내

모든 예제는 다음 경로에 있다.

```
chapter-cs-algorithms/src/main/java/com/edu/algorithms/
```

각 파일은 **독립 실행 가능한** `main()`을 가지며, 실행하면 "말"이 아니라
**측정값과 다이어그램**으로 결과를 보여준다. 실행 방법은 두 가지다.

```bash
# 통합 Docker Java 환경(권장)에서:
./run.sh <ClassName>     # 예: ./run.sh SortingAlgorithms
./run.sh                 # 인자 없이 실행하면 대화형 메뉴
```

| 클래스 | 무엇을 보여주는가 | 연결 장 |
|--------|-----------------|--------|
| `./run.sh SearchAlgorithms` | 선형 vs 이진 탐색의 비교 횟수 O(n) vs O(log n), 정렬 전제를 깨면 실패하는 시연 | 1, 2 |
| `./run.sh SortingAlgorithms` | 5가지 정렬 구현, 크기별 시간 측정 표, 안정성 차이 | 1, 3 |
| `./run.sh RecursionAndDP` | 피보나치 O(2ⁿ) vs 메모 vs 타뷸레이션, 콜 스택 깊이, 0/1 배낭 DP, 그리디 실패 | 4 |
| `./run.sh DataStructuresFromScratch` | 동적 배열·연결 리스트·스택·큐 밑바닥 구현, 용량 확장 순간 출력 | 1, 5 |
| `./run.sh TreeAndHeap` | BST(삽입/탐색/중위순회/퇴화), MinHeap/PriorityQueue, 힙 정렬 | 6 |
| `./run.sh GraphAlgorithms` | 인접 리스트 그래프, BFS/DFS 방문 순서, 다익스트라 최단 거리 | 7 |

> 실습 환경 설정과 raw `javac`/`java` 실행법은 `chapter-cs-algorithms/README.md`를 참고하라.
