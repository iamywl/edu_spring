## 2.1 변수 선언

변수란 값을 저장할 수 있는 **이름 붙은 메모리 공간**이다. 자바는 *정적 타입 언어(statically typed)* 이므로, 변수를 쓰기 전에 반드시 "어떤 타입의" "무엇이라는 이름의" 공간인지 미리 선언해야 한다.

```
   메모리
 ┌──────────────┐
 │  age  │  25  │   ← "age"라는 이름이 붙은 int 크기의 공간
 └──────────────┘
```

선언에는 세 가지 형태가 있다.

```java
int age;          // 선언만: int 크기의 age 공간을 확보
age = 25;         // 대입: 그 공간에 25를 넣음
int year = 2026;  // 선언 + 초기화 (가장 흔한 형태)
```

값은 언제든 다시 바꿀 수 있어서 "변(變)수"라고 부른다. 같은 타입이면 `int x = 1, y = 2;` 처럼 콤마로 한 줄에 여러 개를 선언할 수도 있다.

> 💻 실습: `./run.sh VariableDeclaration`

## 2.2 정수 타입

정수를 담는 기본 타입은 **byte, short, int, long** 네 가지다. 왜 네 가지나 될까? 그릇(메모리)의 크기가 다르기 때문이다. 작은 값에 8바이트를 통째로 쓰면 메모리 낭비이고, 큰 값을 작은 그릇에 넣으면 넘친다. 상황에 맞게 골라 쓰라는 뜻이다.

```
byte  (1B)  -128 ~ 127
short (2B)  -32,768 ~ 32,767
int   (4B)  약 -21억 ~ 21억        ← 정수의 '기본' 타입
long  (8B)  약 -922경 ~ 922경
```

```java
int  i = 2_000_000_000;   // 밑줄(_)로 자릿수 구분 가능
long l = 9_000_000_000L;  // int 범위를 넘으면 접미사 L 필요
```

`long` 리터럴에 `L`을 붙이는 이유는, 접미사가 없는 정수 리터럴은 기본적으로 `int`로 해석되기 때문이다. `int` 범위를 넘는 값은 `L` 없이 쓰면 컴파일 에러가 난다. 진법은 `0b`(2진), `0`(8진), `0x`(16진) 접두사로 표현한다.

> 💻 실습: `./run.sh IntegerType`

## 2.3 문자 타입

`char`는 **하나의 문자**를 담는 2바이트 타입이다. 자바는 문자를 유니코드(Unicode) **정수 코드값**으로 저장한다. 즉 `char`는 사실상 "정수처럼도 다룰 수 있는" 특별한 타입이다.

```
 'A'  ↔  65      (유니코드 코드값)
 '가'  ↔  44032
```

문자 리터럴은 **작은따옴표** `' '`로 감싼다. (큰따옴표 `" "`는 문자열 String이다 — 헷갈리지 말 것.)

```java
char c1 = 'A';
char c2 = 65;        // 정수 코드값으로도 대입 가능 → 'A'
char c3 = '가';  // \u + 16진수 4자리 유니코드 이스케이프 → '가'
int  code = 'A';     // char → int, 코드값 65가 들어간다
```

> 💻 실습: `./run.sh CharType`

## 2.4 실수 타입

소수점이 있는 수를 담는 타입은 **float(4B)** 와 **double(8B)** 두 가지다. double이 유효자릿수가 약 15자리로 더 정밀하며, 실수의 *기본* 타입이다.

핵심 원리: 컴퓨터는 실수를 **2진 부동소수점(IEEE 754)** 으로 저장한다. 10진수 0.1을 2진수로는 딱 떨어지게 표현할 수 없어, **미세한 오차**가 생긴다.

```java
double d = 3.14;    // 접미사 없는 실수 리터럴은 double
float  f = 3.14f;   // float 은 접미사 f 필수 (없으면 컴파일 에러)

System.out.println(0.1 + 0.2);  // 0.30000000000000004  ← 오차!
```

그래서 돈처럼 정확해야 하는 계산에는 `double`이 아니라 `BigDecimal`을 쓴다(뒤 장에서 다룸). 지수 표기 `3.5e3`(= 3500.0)도 지원한다.

> 💻 실습: `./run.sh FloatType`

## 2.5 논리 타입

`boolean`은 **참(true)** 또는 **거짓(false)**, 오직 두 값만 가지는 타입이다. 조건을 표현하는 데 쓰이며 `if`/`while` 같은 제어문의 핵심이 된다.

C언어와 달리 자바에서는 `0`/`1`을 boolean으로 쓸 수 없다. 오직 `true`/`false`뿐이다.

```java
boolean isAdult = age >= 18;   // 비교 연산의 결과가 곧 boolean

boolean a = true, b = false;
a && b   // AND: 둘 다 참이어야 참 → false
a || b   // OR : 하나라도 참이면 참 → true
!a       // NOT: 뒤집기 → false
```

> 💻 실습: `./run.sh BooleanType`

## 2.6 문자열 타입

`String`은 여러 문자를 이어붙인 **문자열**을 담는다. 지금까지의 기본 타입(int, char 등)과 달리 String은 **클래스(참조 타입)** 이며, 리터럴은 **큰따옴표** `" "`로 감싼다.

중요한 성질 두 가지:

1. **불변(immutable)** — 한 번 만든 문자열의 내용은 바꿀 수 없다. `toUpperCase()` 등은 원본을 바꾸는 게 아니라 새 문자열을 만들어 돌려준다.
2. **`+`로 연결(concatenation)** — 문자열과 다른 값을 `+`로 이으면, 왼쪽에서 오른쪽으로 이어붙는다.

```java
String message = "안녕하세요, " + name + "님!";

"결과=" + 1 + 2   // "결과=12"  (왼→오, 이미 문자열이라 이어붙음)
1 + 2 + "=결과"   // "3=결과"   (숫자끼리 먼저 더한 뒤 연결)
```

`length()`, `substring()`, `charAt()`, `equals()` 등 유용한 메서드가 많다.

> 💻 실습: `./run.sh StringType`

## 2.7 자동 타입 변환

작은 그릇의 값을 **더 큰 그릇**에 대입하면, 자바가 알아서 변환해 준다. 값의 손실이 없으므로 별도 표기 없이 안전하게 일어난다. 이를 **자동(묵시적) 타입 변환**이라 한다.

```
 허용 방향 (작은 → 큰):
   byte → short → int → long → float → double
   char ─────────→ int
```

```java
byte b = 10;
int  i = b;      // 1바이트 → 4바이트, 자동 변환
double d = i;    // 정수 → 실수, 100 → 100.0
int code = 'A';  // char → int, 65
```

큰 그릇에 작은 값을 담는 것이므로 넘칠 일이 없다. 그래서 안전하고, 코드에 아무 표시도 하지 않는다.

> 💻 실습: `./run.sh AutoCasting`

## 2.8 강제 타입 변환

반대로 **큰 그릇의 값을 작은 그릇**에 넣으려면, 반드시 `(타입)`으로 명시해야 한다. 이를 **강제(명시적) 타입 변환**, 줄여서 **캐스팅(casting)** 이라 한다.

```
 ┌────────── int (4B) 300 ──────────┐
 │  넘치는 부분 잘림  │  하위 8비트  │  → byte (1B): 44
 └───────────────────┴──────────────┘
```

작은 그릇에 큰 물을 붓는 격이라 **넘치는 데이터가 잘려나갈 수 있다.** 그 위험을 개발자가 인지하고 있다는 뜻으로 `(타입)`을 직접 써 주는 것이다.

```java
int i = 65;
byte b = (byte) i;      // (byte)를 빼면 컴파일 에러

int big = 300;
byte lost = (byte) big; // 44 — 300이 아니다! 상위 비트 잘림

double d = 3.99;
int cut = (int) d;      // 3 — 반올림이 아니라 소수점 '버림'
```

> 💻 실습: `./run.sh ForcedCasting`

## 2.9 연산식에서 자동 타입 변환

서로 다른 타입끼리 연산하면, 자바는 두 값을 **더 큰 타입으로 맞춘 뒤** 계산한다. 또 하나 중요한 규칙: `byte`/`short`/`char`는 연산 시 무조건 **int로 승격**된 뒤 계산된다.

```
 우선순위:  double  >  float  >  long  >  int
 (하나라도 double이면 결과 double, ... 그 외는 모두 int)
```

```java
int i = 5;  double d = 2.0;
double r = i + d;          // 5 → 5.0으로 변환 후 계산, 결과 double

byte b1 = 10, b2 = 20;
int sum = b1 + b2;         // 결과가 int! byte로는 못 받는다

int a = 7, c = 2;
a / c        // 3   — 정수끼리 나눗셈은 몫만 (소수점 사라짐)
a / 2.0      // 3.5 — 한쪽이 실수면 실수 결과
(double)a / c// 3.5 — 미리 캐스팅해 정확한 나눗셈
```

"정수 나눗셈 함정"은 초보자가 가장 많이 겪는 버그다. 나눗셈에서 소수점이 필요하면 피연산자 중 하나를 실수로 만들어야 한다.

> 💻 실습: `./run.sh CastingInExpression`

## 2.10 문자열을 기본 타입으로 변환

키보드·파일·네트워크로 들어오는 값은 대부분 **문자열**이다. `"123"`은 문자열이라 그대로 계산할 수 없으므로, 숫자 타입으로 **변환**해야 한다. 각 Wrapper 클래스의 `parseXxx()` 정적 메서드를 쓴다.

```java
int    i = Integer.parseInt("12345");
double d = Double.parseDouble("3.141592");
long   l = Long.parseLong("9000000000");
boolean b = Boolean.parseBoolean("true");
```

왜 변환이 필요한지 보여주는 대표 예:

```java
String n1 = "10", n2 = "20";
n1 + n2                                   // "1020"  ← 문자열 연결!
Integer.parseInt(n1) + Integer.parseInt(n2)// 30      ← 진짜 덧셈
```

반대로 기본 타입 → 문자열은 `String.valueOf(x)` 또는 `"" + x`로 만든다.

> 💻 실습: `./run.sh StringToPrimitive`

## 2.11 변수 사용 범위

변수는 자신이 선언된 **중괄호 블록 `{ }` 안에서만** 유효하다. 이 유효 영역을 **스코프(scope)** 라 한다. 블록을 벗어나면 변수는 사라져 더 이상 접근할 수 없다.

```
 class {
   static int classLevel;    // 클래스 전체에서 유효
   main() {
     int local;              // main 블록 전체에서 유효
     { int inner; }          // 이 블록 안에서만 유효
     for(int j...) { }       // j는 for 안에서만 유효
     // 여기서 inner, j 는 접근 불가 → 컴파일 에러
   }
 }
```

이 규칙 덕분에 서로 다른 블록에서는 같은 이름을 써도 충돌하지 않고, 필요 없어진 변수가 자동으로 정리되어 메모리도 절약된다.

```java
{
    int inner = 20;              // 이 블록 안에서만 존재
}
// System.out.println(inner);   // 컴파일 에러: inner를 찾을 수 없음
```

> 💻 실습: `./run.sh VariableScope`

## 2.12 콘솔 출력

변수값을 화면(콘솔)에 출력하는 방법은 세 가지다.

| 메서드 | 동작 |
|--------|------|
| `System.out.println(x)` | 출력 후 **줄바꿈** |
| `System.out.print(x)` | 출력만 (줄바꿈 없음) |
| `System.out.printf(...)` | **형식(포맷)을 지정**해 출력 |

```java
System.out.println("이름: " + name + ", 나이: " + age);

// printf 형식 지정자: %d 정수, %f 실수, %s 문자열, %c 문자, %b 논리, %n 줄바꿈
System.out.printf("정수 %d, 실수 %.2f, 문자열 %s%n", 10, 3.14, "자바");
```

`printf`는 자릿수 맞춤(`%5d`), 소수점 자리 제한(`%.2f`) 같은 정돈된 출력에 유용하다.

> 💻 실습: `./run.sh ConsolePrint`

## 2.13 키보드 입력

사용자가 키보드로 입력한 값은 **`Scanner`** 클래스로 읽어 변수에 저장한다.

```java
import java.util.Scanner;

Scanner sc = new Scanner(System.in);  // 입력원을 키보드(System.in)로 지정
System.out.print("나이: ");
int age = sc.nextInt();               // 사용자가 입력할 때까지 기다린다(블로킹)
sc.close();
```

타입별 읽기 메서드가 나뉘어 있다.

| 메서드 | 읽는 것 |
|--------|---------|
| `nextInt()` | 정수 하나 |
| `nextDouble()` | 실수 하나 |
| `next()` | 공백 전까지 문자열 하나 |
| `nextLine()` | 줄 전체 |

> 참고: 예제 클래스는 자동 실행 시 입력 대기로 멈추지 않도록, `System.in` 대신 미리 준비한 문자열을 입력원으로 삼아 시연한다. 실제로는 `new Scanner(System.in)`으로만 바꾸면 키보드 입력이 된다.

> 💻 실습: `./run.sh KeyboardInput`

## ⚠️ 흔한 오해와 함정

- **정수 나눗셈의 소수점 실종.** `7 / 2`는 `3.5`가 아니라 `3`이다. 정수끼리 나누면 몫만 남는다. 소수 결과가 필요하면 `7 / 2.0` 또는 `(double) a / b`처럼 한쪽을 실수로 만들어야 한다.
- **실수는 정확하지 않다.** `0.1 + 0.2 == 0.3`은 `false`다. 부동소수점 오차 때문이다. 돈 계산엔 `double` 대신 `BigDecimal`을 쓴다.
- **작은따옴표 vs 큰따옴표.** `'A'`는 문자(char), `"A"`는 문자열(String)이다. `'AB'`처럼 두 글자를 작은따옴표로 감싸면 에러다.
- **캐스팅은 반올림이 아니다.** `(int) 3.99`는 `4`가 아니라 `3`이다. 소수점 이하를 **버린다.**
- **`byte + byte`의 결과는 int.** 작은 타입은 연산 시 int로 승격되므로 `byte b3 = b1 + b2;`는 컴파일 에러다.
- **`long` 리터럴의 L, `float` 리터럴의 f.** int 범위를 넘는 정수엔 `L`, float 변수엔 `f`를 붙이지 않으면 에러가 난다.
- **문자열의 `+`는 덧셈이 아니다.** `"10" + "20"`은 `30`이 아니라 `"1020"`이다. 계산하려면 `parseInt`로 숫자 변환이 먼저다.

**과제 1. 내 프로필 출력 (쉬움)**

정수 나이, 실수 키(cm), 문자 혈액형(예: `'A'`), 논리 값 성인 여부(`boolean`), 문자열 이름을 각각 **변수로 선언·초기화**한 뒤, `이름: OOO, 나이: OO, 키: OOO.O, 혈액형: O형, 성인: true` 형태의 한 줄로 출력하는 프로그램(클래스)을 작성하라. 값은 자유롭게 정한다.

- 힌트: `int` / `double` / `char` / `boolean` / `String` 변수 선언과 `System.out.println`에서의 `+` 문자열 연결.
> 답안 비교: `./run.sh VariableDeclaration`

**과제 2. 큰 수와 오버플로 관찰 (쉬움→응용)**

`int` 최댓값(`Integer.MAX_VALUE`)을 변수에 담아 출력하고, 거기에 `1`을 더한 결과도 출력하라. 그런 다음 `100억`처럼 int 범위를 넘는 값을 **올바른 타입과 접미사**로 선언해 출력하는 프로그램을 작성하라. int에 `1`을 더했을 때 왜 음수(최솟값)로 뒤집히는지 주석으로 한 줄 설명을 덧붙여라.

- 힌트: `Integer.MAX_VALUE`, `long`과 리터럴 접미사 `L`, 오버플로 순환.
> 답안 비교: `./run.sh IntegerType`

**과제 3. 온도 변환기 — 문자열 입력과 정수 나눗셈 함정 (응용)**

문자열로 주어진 섭씨 온도(예: `String c = "100";`)를 숫자로 **변환**한 뒤, 화씨 = 섭씨 × 9 / 5 + 32 공식으로 화씨 온도를 계산해 출력하는 프로그램을 작성하라. 단, `9 / 5`를 그대로 쓰면 정수 나눗셈으로 `1`이 되어 결과가 틀린다는 점을 확인하고, **실수 나눗셈이 되도록** 식을 고쳐 정확한 값(섭씨 100 → 화씨 212.0)이 나오게 하라.

- 힌트: `Integer.parseInt`, 그리고 `9 / 5` 대신 `9.0 / 5` 또는 `(double)` 캐스팅으로 실수 나눗셈 유도.
> 답안 비교: `./run.sh StringToPrimitive`

**과제 4. 강제 캐스팅으로 데이터가 잘리는 순간 (응용)**

`int big = 300;`을 `byte`로 강제 캐스팅해 출력하고(값이 300이 아님을 확인), `double price = 3.99;`를 `int`로 캐스팅한 값과 `Math.round(price)` 값을 함께 출력하는 프로그램을 작성하라. 캐스팅이 **반올림이 아니라 버림**이라는 것을 출력으로 드러내라.

- 힌트: `(byte)`, `(int)` 명시적 캐스팅과 `Math.round`. 소수점 버림 vs 반올림 비교.
> 답안 비교: `./run.sh ForcedCasting`

**과제 5. 변수 스코프 실험 (응용)**

`main` 안에 별도의 중괄호 블록 `{ }`을 하나 만들고, 그 안에서만 쓰는 변수 `inner`를 선언·출력하라. 블록 밖에서 `inner`를 출력하는 줄을 **주석으로** 남기고, 그 줄의 주석을 풀면 왜 컴파일 에러가 나는지 한 줄로 설명하는 주석을 덧붙인 프로그램을 작성하라.

- 힌트: 중괄호 블록과 스코프 규칙 — 변수는 선언된 블록 안에서만 유효하다.
> 답안 비교: `./run.sh VariableScope`

## 확인문제

**1.** `int` 타입의 표현 범위를 벗어나는 큰 정수 리터럴(예: `9000000000`)을 저장하려면 어떤 타입과 접미사가 필요한가?

<details><summary>힌트 / 해설</summary>

`long` 타입을 쓰고 리터럴 뒤에 접미사 `L`을 붙인다: `long l = 9000000000L;`.
접미사가 없는 정수 리터럴은 `int`로 해석되어 범위를 넘으면 컴파일 에러가 나기 때문이다.
</details>

**2.** 다음 코드의 출력은?

```java
System.out.println(7 / 2);
System.out.println(7 / 2.0);
```

<details><summary>힌트 / 해설</summary>

`3` 과 `3.5`.
첫 줄은 정수끼리 나눗셈이라 몫만 남는다(3). 둘째 줄은 한쪽이 실수(`2.0`)라 결과도 실수(3.5)가 된다.
</details>

**3.** `double d = 3.99;` 일 때 `(int) d`의 값은? 왜 그런가?

<details><summary>힌트 / 해설</summary>

`3`. 실수 → 정수 캐스팅은 **반올림이 아니라 소수점 이하를 버린다(절삭).** 따라서 3.99는 3이 된다.
</details>

**4.** `char`가 "정수처럼도 다룰 수 있는 타입"이라고 하는 이유는 무엇인가?

<details><summary>힌트 / 해설</summary>

자바는 문자를 **유니코드 정수 코드값**으로 저장하기 때문이다. 예를 들어 `'A'`는 코드값 65로 저장되며, `int code = 'A';`처럼 int로 자동 변환되면 65가 들어간다. 반대로 `char c = 65;`도 가능해 'A'가 된다.
</details>

**5.** 문자열 `"100"`과 `"200"`을 더해 `300`을 얻으려면 어떻게 해야 하나? 그냥 `+`로 더하면 어떻게 되는가?

<details><summary>힌트 / 해설</summary>

`Integer.parseInt("100") + Integer.parseInt("200")` 처럼 먼저 숫자로 변환한 뒤 더해야 300이 된다.
그냥 `"100" + "200"`으로 더하면 문자열 연결이 일어나 `"100200"`이 된다.
</details>

## 요약

- **변수**는 값을 담는 이름 붙은 메모리 공간이고, 자바는 쓰기 전에 **타입 + 이름**으로 선언해야 하는 정적 타입 언어다.
- 기본 타입 8가지: 정수 `byte/short/int/long`, 실수 `float/double`, 문자 `char`, 논리 `boolean`. 그릇의 크기와 용도가 다르다.
- **String**은 기본 타입이 아닌 참조 타입이며 불변이고, `+`로 연결한다.
- 타입 변환: 작은 → 큰은 **자동**(안전), 큰 → 작은은 **강제 캐스팅**(데이터 손실 가능). 연산식에선 더 큰 타입으로 맞춰지고 작은 타입은 int로 승격된다.
- 문자열로 들어온 값은 `parseXxx()`로 숫자 타입으로 변환해야 계산할 수 있다.
- 변수는 선언된 **블록 안에서만** 유효하다(스코프).
- 출력은 `println`/`print`/`printf`, 입력은 `Scanner`로 처리한다.
