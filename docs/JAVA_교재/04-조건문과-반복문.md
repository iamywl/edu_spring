# Chapter 04 조건문과 반복문

프로그램은 기본적으로 위에서 아래로 한 줄씩 순서대로 실행된다. 하지만 실제 프로그램은 "점수가 60점 이상이면 합격", "10개의 데이터를 하나씩 처리", "원하는 값을 찾으면 멈춤"처럼 **상황에 따라 실행 순서를 바꾸어야** 한다. 이렇게 실행 순서를 조절하는 문법을 **제어문(control statement)**이라고 한다.

제어문은 크게 두 종류로 나뉜다.

- **조건문(selection)** : 조건에 따라 실행할 코드를 고른다. → `if`, `switch`
- **반복문(iteration)** : 같은 코드를 여러 번 실행한다. → `for`, `while`, `do-while`

그리고 반복문 안에서 흐름을 세밀하게 조절하는 **분기문(branch)**으로 `break`(즉시 탈출)와 `continue`(이번 회차 건너뛰기)가 있다.

이번 장에서는 이 문법들을 하나씩 살펴본다. 각 소절에는 실행 가능한 실습 코드가 연결되어 있으니, 개념을 읽은 뒤 직접 실행해 결과를 눈으로 확인하며 익히자.

---

## 4.1 실행 흐름 제어

코드의 실행 순서를 다루는 방식을 **제어 흐름(control flow)**이라고 한다. 제어 흐름은 세 가지 기본 형태로 이루어진다.

1. **순차(sequence)** : 특별한 제어문이 없으면 작성한 순서대로 위에서 아래로 한 문장씩 실행된다.
2. **조건(selection)** : 조건이 참일 때만 특정 코드를 실행한다. 갈림길에서 하나의 길을 고르는 것과 같다.
3. **반복(iteration)** : 같은 코드를 정해진 횟수나 조건만큼 되풀이한다.

우리가 작성하는 거의 모든 프로그램은 이 세 가지의 조합으로 만들어진다. 아래 코드는 세 가지 흐름이 한 프로그램 안에서 어떻게 어우러지는지 보여준다.

```java
// [1] 순차: 위에서 아래로
System.out.println("첫 번째 문장");
System.out.println("두 번째 문장");

// [2] 조건: 참일 때만 실행
int temperature = 30;
if (temperature >= 28) {
    System.out.println("덥습니다.");
}

// [3] 반복: 같은 작업을 되풀이
for (int i = 1; i <= 3; i++) {
    System.out.println("반복 " + i + "회차");
}
```

> 💻 실습: `./run.sh ExecutionFlow`

## 4.2 if 문

`if` 문은 **조건이 참(true)일 때만** 코드 블록을 실행한다. 조건식의 결과는 반드시 `boolean`(참/거짓)이어야 한다. 형태는 목적에 따라 네 가지로 나뉜다.

- **단순 if** : 조건이 참이면 실행, 거짓이면 아무것도 하지 않는다.
- **if-else** : 참이면 앞 블록, 거짓이면 `else` 블록을 실행한다. (두 갈래)
- **else if 사슬** : 여러 조건을 위에서부터 순서대로 검사해, **가장 먼저 참이 되는** 블록 하나만 실행한다.
- **중첩 if** : `if` 안에 또 `if`를 넣어 조건을 단계적으로 확인한다.

```java
int point = 82;
if (point >= 90) {
    System.out.println("등급: A");
} else if (point >= 80) {
    System.out.println("등급: B");   // 82 → 여기서 걸려 실행됨
} else if (point >= 70) {
    System.out.println("등급: C");
} else {
    System.out.println("등급: F");
}
```

`else if` 사슬은 위에서부터 검사하다가 처음으로 참이 되는 곳에서 멈춘다. 그래서 조건의 **순서가 중요**하다. 예컨대 위 코드에서 `>= 80`을 `>= 90`보다 위에 두면, 95점도 B로 판정되어 버린다.

> 💻 실습: `./run.sh IfStatement`

## 4.3 switch 문

`switch` 문은 **하나의 값을 여러 case와 비교**하여, 일치하는 곳부터 실행한다. "이 값이 A인가? B인가? C인가?"처럼 **같은지(==)만** 따질 때, 긴 `else if` 사슬보다 훨씬 읽기 쉽다.

**전통 switch**는 각 `case` 끝에 `break`를 붙여 흐름을 끊는다. 만약 `break`를 생략하면 다음 `case`로 그대로 흘러 내려가는데, 이를 **fall-through**라고 한다. 이 성질을 일부러 이용하면 여러 `case`를 하나로 묶을 수 있다.

```java
int month = 4;
switch (month) {
    case 3:
    case 4:
    case 5:
        System.out.println("봄");   // 3,4,5월이 하나로 묶임
        break;
    default:
        System.out.println("그 외");
}
```

**switch 표현식(JDK 14+)**은 `case 값 -> 결과;` 형태로 값을 바로 돌려줄 수 있다. `break`가 필요 없고 fall-through도 일어나지 않아 실수가 줄어든다. 결과를 계산하는 데 여러 문장이 필요하면 `{ ... yield 값; }` 블록을 쓴다.

```java
int grade = 2;
String result = switch (grade) {
    case 1 -> "1학년";
    case 2 -> "2학년";   // grade=2 → 이 값이 result에 들어감
    default -> "학년 미상";
};
```

> 💻 실습: `./run.sh SwitchStatement`

## 4.4 for 문

`for` 문은 **반복 횟수가 정해져 있을 때** 가장 흔히 쓰는 반복문이다. 구성 요소는 세 가지다.

```java
for (초기화; 조건; 증감) { 반복할 코드 }
```

- **초기화** : 반복 시작 전 딱 한 번만 실행된다. 보통 카운터 변수를 선언한다.
- **조건** : 매 반복 전에 검사한다. 참이면 본문을 실행하고, 거짓이 되면 반복을 끝낸다.
- **증감** : 본문을 실행한 뒤 매번 실행된다. 보통 카운터를 1씩 바꾼다.

```java
for (int i = 1; i <= 5; i++) {
    System.out.println("i = " + i);   // 1,2,3,4,5 출력
}
```

배열이나 컬렉션의 요소를 **처음부터 끝까지 하나씩** 꺼낼 때는 **향상된 for(for-each)**가 더 간결하다. 인덱스 변수 없이 요소를 바로 다룰 수 있다.

```java
String[] fruits = {"사과", "바나나", "체리"};
for (String fruit : fruits) {
    System.out.println("과일: " + fruit);
}
```

`for` 안에 `for`를 넣는 **중첩 for**는 구구단이나 표(2차원) 같은 반복에 쓰인다. 참고로 `for (;;)`처럼 조건을 비우면 무한 반복이 되므로 반드시 `break`로 탈출해야 한다.

> 💻 실습: `./run.sh ForStatement`

## 4.5 while 문

`while` 문은 **조건이 참인 동안** 코드를 반복한다. `for`와 달리 반복 횟수를 미리 정하지 않고, **어떤 조건이 만족될 때까지** 돌릴 때 적합하다.

```java
while (조건) { 반복할 코드 }
```

핵심은 두 가지다. 첫째, **조건을 먼저 검사**하므로 처음부터 거짓이면 본문이 한 번도 실행되지 않는다. 둘째, 반복 안에서 **언젠가 조건을 거짓으로 만들어 주어야** 한다. 그러지 않으면 영원히 도는 **무한 루프**에 빠진다.

```java
int i = 1;
while (i <= 5) {
    System.out.println("i = " + i);
    i++;   // 이 증가가 없으면 무한 루프! 조건을 거짓으로 만드는 역할
}
```

`i++`를 빠뜨리면 `i`는 계속 1에 머물러 조건이 영원히 참이 된다. `while`을 쓸 때는 항상 "이 조건은 언제 거짓이 되는가?"를 스스로 물어야 한다. 반복 종료를 `boolean` 플래그 변수로 제어하는 방식도 자주 쓰인다.

> 💻 실습: `./run.sh WhileStatement`

## 4.6 do-while 문

`do-while` 문은 `while`과 비슷하지만 **실행 순서가 반대**다. 본문을 **먼저 한 번 실행한 뒤** 조건을 검사한다. 그래서 조건이 처음부터 거짓이어도 **최소 한 번은 반드시** 실행된다.

```java
do { 반복할 코드 } while (조건);   // 끝에 세미콜론(;) 필수
```

| 구분 | 순서 | 최소 실행 횟수 |
|------|------|----------------|
| `while` | 조건 검사 → 실행 | 0번 |
| `do-while` | 실행 → 조건 검사 | 1번 |

```java
int b = 10;
do {
    System.out.println("최소 1회 실행됨 b=" + b);   // 조건이 거짓이어도 1번 출력
    b++;
} while (b < 5);
```

"메뉴를 보여주고 입력을 받는" 것처럼 **일단 한 번은 실행한 뒤** 계속할지 판단하는 상황에 잘 맞는다. 문법 끝에 붙는 세미콜론(`;`)을 빠뜨리기 쉬우니 주의하자.

> 💻 실습: `./run.sh DoWhileStatement`

## 4.7 break 문

`break` 문은 반복문(또는 `switch`)을 **즉시 빠져나온다**. 원하는 조건을 만족하는 순간 더 이상 반복할 필요가 없을 때 사용한다. 예를 들어 특정 값을 찾으면 나머지는 확인할 필요 없이 멈추면 된다.

기본 `break`는 자신을 감싸는 **가장 안쪽 반복문 하나만** 탈출한다. 그래서 중첩 반복에서 안쪽 `for`에 `break`를 쓰면 바깥 `for`는 계속 돈다.

```java
for (int i = 1; i <= 10; i++) {
    if (i == 5) {
        System.out.println("5 발견 → 중단");
        break;   // for를 즉시 탈출 (6~10은 실행 안 됨)
    }
    System.out.println("i = " + i);
}
```

중첩 반복을 **한 번에 모두** 탈출하고 싶다면 **레이블 break**를 쓴다. 바깥 반복문에 `이름:` 형태로 이름을 붙인 뒤 `break 이름;`으로 지정한다.

```java
outer:
for (int row = 1; row <= 3; row++) {
    for (int col = 1; col <= 3; col++) {
        if (row == 2 && col == 2) {
            break outer;   // 'outer'가 붙은 바깥 for 전체를 탈출
        }
    }
}
```

> 💻 실습: `./run.sh BreakStatement`

## 4.8 continue 문

`continue` 문은 현재 반복의 **남은 부분을 건너뛰고 다음 회차로** 넘어간다. `break`가 반복을 완전히 끝낸다면, `continue`는 **이번 회차만** 건너뛸 뿐 반복 자체는 계속된다. "짝수는 건너뛰고 홀수만 처리" 같은 필터링에 유용하다.

```java
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) {
        continue;   // 짝수면 아래를 건너뛰고 다음 i로
    }
    System.out.println("홀수 i = " + i);
}
```

`continue`가 다음으로 이동하는 위치는 반복문마다 다르다. `for`에서는 **증감식**으로, `while`에서는 **조건 검사**로 되돌아간다. 이 차이 때문에 `while`에서는 주의가 필요하다 — 증감을 `continue` 뒤에 두면 그 문장을 영영 못 만나 무한 루프에 빠질 수 있다.

중첩 반복에서 바깥 반복의 다음 회차로 바로 점프하고 싶다면 `break`처럼 **레이블 continue**(`continue 이름;`)를 쓴다.

> 💻 실습: `./run.sh ContinueStatement`

## ⚠️ 흔한 오해와 함정

- **`=`와 `==`를 헷갈린다.** `if (x = 5)`는 대입이고 `if (x == 5)`가 비교다. 자바에서는 `boolean`이 아닌 대입식을 조건에 쓰면 컴파일 오류가 나 어느 정도 막아 주지만, `boolean` 변수에서는 `if (flag = true)`가 실수로 통과할 수 있으니 주의하자.
- **switch에서 break를 빠뜨린다.** 전통 `switch`에서 `break`를 잊으면 아래 `case`로 계속 흘러(fall-through) 의도치 않은 코드까지 실행된다. fall-through를 원하는 게 아니라면 각 `case`에 `break`를 꼭 붙이자. (화살표 `->` 표현식에는 이 문제가 없다.)
- **`while`/`for` 조건을 거짓으로 만드는 것을 잊어 무한 루프에 빠진다.** 반복 안에서 카운터 증감이나 플래그 변경을 반드시 넣어야 한다. `while`에서 `continue`를 쓸 때는 증감식을 `continue` **앞**에 두는지 특히 확인하자.
- **중괄호 `{}`를 생략한 뒤 실수한다.** `if (조건) 문장;`처럼 중괄호를 생략하면 **바로 다음 한 문장만** 조건에 속한다. 나중에 문장을 한 줄 더 추가하면 그 줄은 조건과 무관하게 항상 실행되어 버린다. 가독성과 안전을 위해 항상 `{}`를 쓰자.
- **`break`와 `continue`를 혼동한다.** `break`는 반복을 **완전히 끝내고**, `continue`는 **이번 회차만 건너뛴다.** 목적에 맞는 것을 골라야 한다.
- **기본 `break`가 모든 반복을 빠져나온다고 착각한다.** 중첩 반복에서 기본 `break`는 **가장 안쪽 하나만** 탈출한다. 바깥까지 나가려면 레이블을 써야 한다.

## ✍️ 직접 작성해보기

지금까지는 완성된 코드를 **읽고 실행**하며 개념을 익혔다. 이제는 직접 **키보드로 타이핑**해 볼 차례다. 코드는 눈으로 볼 때와 손으로 칠 때 남는 것이 완전히 다르다. 아래 과제를 순서대로 풀어 보자. 쉬운 것에서 시작해 점점 응용으로 나아간다.

**푸는 방법**

1. 각 과제의 **스펙(무엇을 만들지)**과 **힌트**를 읽는다.
2. 여러분만의 클래스를 직접 작성하고 실행해 결과가 스펙과 맞는지 확인한다.
3. 막히거나 다 풀었다면, 각 과제 끝의 `> 답안 비교` 명령으로 **모범 답안 코드를 실행**해 내 결과와 비교해 본다. 답안 클래스는 이 장의 실습 코드와 같은 위치에 있다.

> 팁: 먼저 종이나 주석으로 "어떤 조건일 때 무엇을 할지"를 말로 적어 본 뒤 코드로 옮기면 훨씬 수월하다. 정답을 바로 열어 보지 말고, 최소 한 번은 스스로 작성해 실행까지 해 보자.

### 과제 1. 학점 판별기 (조건문)

시험 점수(정수 하나)를 변수에 담아 두고, 그 점수에 해당하는 **학점을 출력하는 프로그램**을 작성하라. 기준은 다음과 같다: 90점 이상 A, 80점 이상 B, 70점 이상 C, 60점 이상 D, 그 미만은 F. 예를 들어 점수가 82이면 `등급: B`가 출력되어야 한다.

- 힌트: `if` / `else if` / `else` 사슬을 사용한다. 조건은 **높은 점수부터** 검사해야 95점이 A로 올바르게 판정된다(순서가 중요하다).
- 힌트: 점수 변수 값을 몇 번 바꿔 가며 경계값(90, 80, 60, 59 등)에서 결과가 맞는지 확인해 보자.

> 답안 비교: `./run.sh IfStatement`

### 과제 2. 요일 이름 변환기 (switch)

1부터 7까지의 정수(요일 번호)를 변수에 담아 두고, 1이면 `월요일`, 2이면 `화요일` … 7이면 `일요일`을 출력하는 프로그램을 작성하라. 1~7 이외의 값이 들어오면 `잘못된 요일 번호`를 출력하라.

- 힌트: `switch` 문을 사용한다. 전통 방식이라면 각 `case`에 `break`를 붙이는 것을 잊지 말고, 화살표(`->`) 표현식을 쓰면 `break` 없이 더 깔끔하게 작성할 수 있다.
- 힌트: 범위를 벗어난 값 처리는 `default`가 담당한다.

> 답안 비교: `./run.sh SwitchStatement`

### 과제 3. 구구단 한 단 출력 (for 반복)

정수 하나(예: 7)를 변수에 담아 두고, 그 수의 **구구단 한 단**을 `7 x 1 = 7`부터 `7 x 9 = 63`까지 출력하는 프로그램을 작성하라.

- 힌트: `for (int i = 1; i <= 9; i++)` 형태의 반복문 안에서 `단 * i` 값을 계산해 출력한다.
- 응용: 바깥에 `for`를 하나 더 감싸(중첩 for) 2단부터 9단까지 **전체 구구단**을 출력하도록 확장해 보자.

> 답안 비교: `./run.sh ForStatement`

### 과제 4. 1부터 N까지의 합 (while 반복)

정수 N(예: 100)을 변수에 담아 두고, **1부터 N까지 모든 정수의 합**을 `while` 문으로 계산해 출력하는 프로그램을 작성하라. N이 100이면 결과는 `5050`이어야 한다.

- 힌트: 카운터 변수 `i`를 1로 두고, `합`에 `i`를 더한 뒤 `i++`로 하나씩 늘린다. 반복 조건은 `i <= N`이다.
- 힌트: **`i++`(또는 조건을 거짓으로 만드는 문장)를 반드시 넣어야** 무한 루프에 빠지지 않는다. `while`을 쓸 때는 "이 조건은 언제 거짓이 되는가?"를 스스로 물어보자.

> 답안 비교: `./run.sh WhileStatement`

### 과제 5. 소수 판별기 (break 활용)

2 이상의 정수 하나를 변수에 담아 두고, 그 수가 **소수(prime)인지 아닌지 판별해 출력하는 프로그램**을 작성하라. 소수이면 `N은 소수입니다`, 아니면 `N은 소수가 아닙니다`를 출력한다. (소수는 1과 자기 자신 외에는 약수가 없는 2 이상의 수다.)

- 힌트: 2부터 `N-1`까지 반복하며 `N % i == 0`인 약수를 찾는다. **하나라도 나누어떨어지면** 소수가 아니므로, 그 순간 `break`로 반복을 멈추고 결과를 확정하면 된다(끝까지 돌 필요가 없다).
- 힌트: "약수를 찾았는가"를 `boolean` 플래그 변수로 기억해 두면, 반복이 끝난 뒤 소수 여부를 판단하기 쉽다.

> 답안 비교: `./run.sh BreakStatement`

### 과제 6. FizzBuzz (continue·조건 활용)

1부터 30까지 반복하며, 3의 배수이면 `Fizz`, 5의 배수이면 `Buzz`, 3과 5의 **공배수**(즉 15의 배수)이면 `FizzBuzz`, 그 외에는 그 숫자 자체를 출력하는 프로그램을 작성하라.

- 힌트: 15의 배수를 **가장 먼저** 검사해야 3의 배수 조건에 먼저 걸리지 않는다(조건 순서에 주의).
- 응용: 4의 배수는 아예 출력에서 제외하도록 `continue`를 추가해 보자. 짝수만/홀수만 처리하는 필터링처럼, `continue`는 "이번 회차만 건너뛰기"에 쓰인다.

> 답안 비교: `./run.sh ContinueStatement`

## 요약

- **제어 흐름**은 순차·조건·반복 세 가지 형태로 이루어지며, 대부분의 프로그램은 이들의 조합이다. (`ExecutionFlow`)
- **`if` 문**은 조건이 참일 때 실행한다. `if-else`, `else if` 사슬, 중첩 `if`로 확장되며, `else if`는 **먼저 참이 되는** 블록 하나만 실행하므로 순서가 중요하다. (`IfStatement`)
- **`switch` 문**은 하나의 값을 여러 `case`와 비교한다. 전통 방식은 `break`로 끊으며 생략 시 **fall-through**가 일어난다. 화살표 표현식(`->`)은 값을 돌려주고 fall-through가 없다. (`SwitchStatement`)
- **`for` 문**은 반복 횟수가 정해진 반복에 쓰며(초기화·조건·증감), 배열/컬렉션 순회에는 **for-each**가 간결하다. (`ForStatement`)
- **`while` 문**은 조건이 참인 동안 반복하고 **조건을 먼저** 검사한다. 조건을 거짓으로 만들지 않으면 무한 루프에 빠진다. (`WhileStatement`)
- **`do-while` 문**은 본문을 **먼저 실행한 뒤** 조건을 검사하므로 **최소 1회**는 실행된다. (`DoWhileStatement`)
- **`break`**는 반복을 즉시 끝내고, 기본형은 **가장 안쪽 하나만** 탈출한다. 레이블로 바깥까지 탈출할 수 있다. (`BreakStatement`)
- **`continue`**는 **이번 회차만 건너뛰고** 다음 회차로 넘어간다. `while`에서는 증감 위치에 주의해야 한다. (`ContinueStatement`)
