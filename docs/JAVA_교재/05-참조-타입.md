# Chapter 05. 참조 타입

지금까지 다룬 `int`, `double`, `boolean` 같은 기본 타입은 변수 안에 **값 자체**를 담는다. 하지만 실제 프로그램은 문자열, 배열, 객체처럼 크고 복잡한 데이터를 다뤄야 한다. 이런 데이터는 변수 안에 통째로 넣기 어렵기 때문에, 자바는 데이터를 메모리 어딘가(힙)에 만들어 두고 변수에는 그 **위치(번지)** 만 저장한다. 이렇게 "값이 있는 곳을 가리키는" 타입을 **참조 타입(reference type)** 이라 부른다.

참조 타입을 제대로 이해하지 못하면 `==` 가 이상하게 동작하고, 배열을 복사했는데 원본이 같이 바뀌고, 난데없이 `NullPointerException` 이 터진다. 이 장에서는 참조 타입이 메모리에서 어떻게 동작하는지를 그림으로 이해하고, 문자열·배열·열거 타입까지 하나씩 실습한다.

> 이 장의 모든 실습은 `chapter-java-textbook` 디렉터리에서 `./run.sh <클래스명>` 으로 실행한다.
> 참조 타입이 등장하는 근본 배경(메모리, 스택/힙, 값 vs 참조)의 더 깊은 논의는

> [JAVA_개념서 1장 「컴퓨터는 어떻게 프로그램을 실행하는가」](심화/01-컴퓨터는-어떻게-프로그램을-실행하는가.md)를 함께 보라.

## 5.1 데이터 타입 분류

자바의 데이터 타입은 크게 **기본 타입(primitive type)** 과 **참조 타입(reference type)** 두 부류로 나뉜다.

| 구분 | 종류 | 변수에 저장되는 것 |
| --- | --- | --- |
| 기본 타입 | 정수(`byte` `short` `int` `long`), 실수(`float` `double`), 문자(`char`), 논리(`boolean`) | 값 자체 |
| 참조 타입 | 배열, 열거(`enum`), 클래스, 인터페이스 | 객체의 번지(주소) |

핵심 차이는 딱 한 문장이다. **기본형은 값을 직접 담고, 참조형은 값이 있는 곳을 가리킨다.**

이 차이는 "복사"할 때 극명하게 드러난다.

```java
int a = 10;
int b = a;     // 값 10이 복사됨 → a와 b는 독립
b = 99;        // a는 여전히 10

int[] arr1 = { 1, 2, 3 };
int[] arr2 = arr1;   // 번지가 복사됨 → 둘이 같은 배열을 가리킴
arr2[0] = 99;        // arr1[0]도 99로 바뀐다!
```

`b = a` 는 값을 복제하지만, `arr2 = arr1` 은 "가리키는 위치"만 복제하므로 두 변수가 **같은 객체**를 공유한다.

## 5.2 메모리 사용 영역

참조 타입을 이해하려면 JVM이 메모리를 어떻게 나눠 쓰는지를 알아야 한다. JVM은 실행 시 메모리를 크게 세 영역으로 구분한다.

- **메서드 영역(Method Area)** — 클래스 정보, `static` 필드/상수, 메서드 코드. 프로그램 전체가 공유한다.
- **스택 영역(Stack)** — 메서드가 호출될 때마다 프레임이 쌓인다. 지역 변수와 매개변수가 저장되며, 메서드가 끝나면 프레임이 사라진다.
- **힙 영역(Heap)** — `new` 로 생성된 객체(배열, 인스턴스)가 저장된다. 참조가 사라지면 가비지 컬렉터가 정리한다.

여기서 가장 중요한 사실: **기본형 지역 변수는 값이 스택에 직접 저장되지만, 참조형 지역 변수는 "힙에 있는 객체의 번지"가 스택에 저장된다.**

```
       스택(Stack)                          힙(Heap)
  ┌───────────────────┐             ┌────────────────────────┐
  │ localValue = 100  │  (값 자체)   │                        │
  │                   │             │   int[]{ 1, 2, 3 }      │
  │ localRef  ───────────────────▶  │   (0x100 번지의 배열)   │
  │            (번지) │             │                        │
  └───────────────────┘             └────────────────────────┘
   메서드가 끝나면 소멸           참조가 사라지면 GC가 수거
```

`localValue` 는 100이라는 값을 스택 안에 그대로 갖지만, `localRef` 는 배열 자체가 아니라 **배열이 놓인 힙 번지**를 갖는다. 배열의 실체는 힙에 있다.

이 구조 때문에 **메서드에 참조를 넘기면 같은 힙 객체를 공유**한다.

```java
static void modify(int[] arr) {
    arr[0] = 42;              // 넘겨받은 번지가 가리키는 힙 객체를 수정
}
// ...
int[] heapArray = { 7, 0, 0 };
modify(heapArray);            // 번지가 복사되지만 가리키는 객체는 동일
// heapArray[0] == 42  ← 메서드 안의 변경이 밖에 반영됨
```

> 💻 실습: `./run.sh MemoryArea`

## 5.3 참조 타입의 ==, != 연산

`==` 와 `!=` 는 기본 타입에서는 "값"을 비교하지만, **참조 타입에서는 "번지(주소)"를 비교한다.** 즉 두 변수가 같은 객체를 가리키는지를 묻는다.

- **동일성(identity)** — `==` : 두 변수가 *같은 객체*를 가리키는가?
- **동등성(equality)** — `equals()` : 두 객체의 *내용*이 같은가?

```java
int[] a = { 1, 2, 3 };
int[] b = { 1, 2, 3 };   // 내용은 같지만 별개의 객체
int[] c = a;             // a와 같은 객체를 가리킴

a == b   // false  (내용이 같아도 다른 객체)
a == c   // true   (같은 객체를 가리킴)
```

```
  a ──▶ [1,2,3] (0x100)      c ──▶ (0x100, a와 동일)
  b ──▶ [1,2,3] (0x200)      a==b → 0x100 vs 0x200 → false
                             a==c → 0x100 vs 0x100 → true
```

그래서 문자열의 **내용**을 비교할 때는 반드시 `equals()` 를 써야 한다. `==` 로 비교하면 "같은 객체인가"를 묻게 되어 예상과 다른 결과가 나온다.

## 5.4 null과 NullPointerException

참조 타입 변수에는 "아무 객체도 가리키지 않는다"는 의미로 `null` 을 대입할 수 있다.

```java
String text = null;   // 어떤 객체도 가리키지 않는 상태
```

문제는 `null` 인 변수를 통해 필드나 메서드에 접근할 때다. 가리키는 객체가 없으니 접근할 대상이 없고, 실행 중 **`NullPointerException`(NPE)** 이 발생한다.

```java
String text = null;
int len = text.length();   // 💥 NullPointerException
```

NPE는 자바에서 가장 흔한 런타임 예외다. 방지법은 두 가지다.

```java
// 방법 1: 사용 전에 null 검사
static int safeLength(String s) {
    if (s == null) return 0;
    return s.length();
}

// 방법 2: Objects.requireNonNull()로 시작 시점에 명확히 거부
static void register(String name) {
    Objects.requireNonNull(name, "name은 null일 수 없습니다");
    // ... 이후 로직에서는 name이 절대 null이 아님이 보장됨
}
```

방법 1은 null을 조용히 허용하고, 방법 2는 null을 만나면 즉시 예외를 던져 잘못된 사용을 조기에 드러낸다. "null이 절대 오면 안 되는 자리"라면 방법 2가 버그를 빨리 찾게 해준다.

> 💻 실습: `./run.sh NullAndNPE`

> 예외의 전반적인 처리 전략은 [JAVA_개념서 7장 「예외처리 - 실패에 대비하다」](심화/07-예외처리-실패에-대비하다.md)에서 다룬다.

## 5.5 문자열(String) 타입

`String` 은 참조 타입이며 문자열을 다루는 가장 기본 클래스다. `String` 에는 초보자를 자주 헷갈리게 하는 두 가지 특성이 있다.

**① 문자열 리터럴과 String Pool.** `"abc"` 같은 리터럴은 힙 안의 특별한 공간인 **문자열 상수 풀(String Pool)** 에 보관되어 재사용된다. 같은 리터럴은 풀에서 **같은 객체**를 공유한다. 반면 `new String("abc")` 는 풀과 별개로 힙에 새 객체를 만든다.

```
  "java" 리터럴들           new String("java")
  ┌──────────────┐         ┌──────────────┐
  │ String Pool  │         │  힙(별도 객체) │
  │  "java"◀─ a  │         │   "java" ◀─ c │
  │        ◀─ b  │         └──────────────┘
  └──────────────┘
  a == b → true (풀 공유)   a == c → false (다른 객체)
  a.equals(c) → true (내용은 같음)
```

`intern()` 을 쓰면 `new` 로 만든 문자열도 풀의 객체로 맞출 수 있다.

```java
String a = "java";
String c = new String("java");
String d = c.intern();   // 풀에 있는 "java"를 반환
// a == d → true
```

**② 불변성(immutable).** `String` 은 한 번 만들어지면 내용을 바꿀 수 없다. 그래서 문자열을 "변경"하는 연산은 실제로는 **새 객체를 만든다.**

```java
String s = "Hello";
String upper = s.concat(" World");  // s는 그대로, 새 문자열이 생성됨
// s     == "Hello"        (변하지 않음)
// upper == "Hello World"  (새 객체)
```

## 5.6 배열(array) 타입

배열은 **같은 타입**의 값을 여러 개 담는 참조 타입이다.

```java
int[] arr;                 // 선언
arr = new int[3];          // 생성 (길이 3, 기본값으로 채워짐)
int[] arr2 = { 1, 2, 3 };  // 선언 + 초기화
```

`new` 로 생성하면 각 요소가 타입별 기본값으로 채워진다.

| 타입 | 기본값 |
| --- | --- |
| 정수형(`int` 등) | `0` |
| 실수형(`double` 등) | `0.0` |
| `boolean` | `false` |
| `char` | `''`(공백) |
| 참조형(`String`, 클래스) | `null` |

인덱스는 **0부터 시작**하며, 유효 범위는 `0 ~ length-1` 이다. 배열 길이는 `arr.length` 로 얻는다(괄호 없는 필드).

```java
String[] fruits = { "사과", "바나나", "포도" };
for (int i = 0; i < fruits.length; i++) {
    System.out.println(fruits[i]);
}
```

범위를 벗어난 인덱스(`fruits[3]`)에 접근하면 `ArrayIndexOutOfBoundsException` 이 발생한다.

> 💻 실습: `./run.sh ArrayType`

## 5.7 다차원 배열

다차원 배열은 **"배열의 배열"** 이다. 자바에는 진짜 2차원 격자가 없다. 대신 바깥 배열의 각 요소가 다시 **안쪽 배열을 가리키는** 구조다.

```java
int[][] m = new int[2][3];   // 2행 3열
```

```
  m (바깥 배열)          안쪽 배열들
  ┌──────┐
  │ m[0] ─────▶ [ 0, 0, 0 ]   (길이 3)
  │ m[1] ─────▶ [ 0, 0, 0 ]   (길이 3)
  └──────┘
  m.length == 2 (행 개수)
  m[0].length == 3 (열 개수)
```

각 행이 별개의 배열이므로, **행마다 길이가 다른** 가변 배열(ragged array)도 만들 수 있다.

```java
int[][] ragged = new int[3][];   // 열은 나중에 각각 생성
ragged[0] = new int[] { 1 };
ragged[1] = new int[] { 1, 2 };
ragged[2] = new int[] { 1, 2, 3 };
```

2차원 배열을 순회할 때는 바깥 for로 행을, 안쪽 for로 열을 돈다. 이때 열의 개수는 반드시 `matrix[r].length` 로 각 행마다 따로 얻어야 가변 배열에서도 안전하다.

> 💻 실습: `./run.sh MultiDimArray`

## 5.8 객체를 참조하는 배열

`String` 이나 사용자 정의 클래스처럼 참조 타입을 담는 배열은, 각 요소에 "값"이 아니라 **"객체의 번지(참조)"** 가 저장된다.

`new Book[3]` 처럼 생성하면 요소는 아직 아무 객체도 가리키지 않아 모두 `null` 이다. 요소에 객체를 대입해야 비로소 그 요소가 해당 객체를 가리킨다.

```
  books ──▶ ┌─────────┐
            │ books[0]│───▶ Book("자바")
            │ books[1]│───▶ Book("스프링")
            │ books[2]│───▶ Book("JPA")
            └─────────┘
```

주의할 점: **두 요소가 같은 객체를 가리키면, 하나를 통해 바꾼 게 다른 쪽에도 보인다.**

```java
Book shared = new Book("공유책");
shelf[0] = shared;
shelf[1] = shared;            // 둘 다 같은 객체
shelf[0].title = "제목변경";   // shelf[0]으로 바꿨지만
// shelf[1].title 도 "제목변경"  ← 같은 객체이기 때문
```

이것은 버그가 아니라 참조의 자연스러운 성질이다. 5.3의 `==` 규칙과 같은 원리다.

> 💻 실습: `./run.sh ObjectReferenceArray`

## 5.9 배열 복사

배열은 한 번 만들면 **크기를 바꿀 수 없다.** 더 큰 배열이 필요하면 새 배열을 만들어 내용을 복사한다.

```java
// System.arraycopy(원본, 원본시작, 대상, 대상시작, 개수)
int[] src = { 1, 2, 3, 4, 5 };
int[] dest = new int[5];
System.arraycopy(src, 0, dest, 0, src.length);

// Arrays.copyOf(원본, 새 길이) — 길면 기본값으로 채움
int[] bigger = Arrays.copyOf(src, 7);   // { 1,2,3,4,5,0,0 }
```

**얕은 복사(shallow copy) 함정.** 참조형 배열을 복사하면 요소의 **"번지"만** 복사된다. 복사본과 원본이 **같은 요소 객체를 공유**하게 된다.

```
  boxes ──▶ [ ● , ● ]          copy = Arrays.copyOf(boxes, 2)
              │   │            copy  ──▶ [ ● , ● ]
              ▼   ▼                        │   │
         Box(10) Box(20) ◀──────────────────┘   │  (같은 객체를 가리킴)
             ▲                                   │
             └───────────────────────────────────┘

  copy[0].value = 999;  →  boxes[0].value 도 999 (요소 객체 공유)
  boxes == copy → false  (배열 껍데기 자체는 다른 객체)
```

즉 배열 껍데기는 새로 생기지만, 그 안이 가리키는 요소 객체들은 원본과 공유된다. 요소 객체까지 독립적으로 복제하려면 각 요소를 직접 새로 만드는 **깊은 복사(deep copy)** 가 필요하다.

> 💻 실습: `./run.sh ArrayCopy`

## 5.10 향상된 for 문 (enhanced for / for-each)

배열이나 컬렉션의 요소를 처음부터 끝까지 순서대로 꺼낼 때 쓰는 간결한 반복문이다.

```java
for (타입 변수 : 배열) { ... }
```

인덱스 변수·조건·증감식이 없어 코드가 짧고 실수가 적다. "읽기" 위주 순회에 특히 좋다.

```java
int[] scores = { 90, 85, 100 };
int sum = 0;
for (int s : scores) {   // 각 요소가 순서대로 s에 담김
    sum += s;
}
```

한 가지 기억할 점: **반복 변수는 요소의 복사본이다.** 그래서 기본형 배열을 순회하며 반복 변수를 바꿔도 원본 배열은 변하지 않는다.

```java
for (int s : scores) {
    s = 0;   // 반복 변수만 0이 됨. scores 요소는 그대로다.
}
```

값을 실제로 바꿔야 한다면 인덱스 기반 for(`scores[i] = 0`)를 써야 한다. 2차원 배열도 for-each를 중첩해 순회할 수 있다.

> 💻 실습: `./run.sh EnhancedForArray`

## 5.11 main(String[] args) 매개변수

프로그램 시작점인 `main` 메서드는 `String[] args` 를 매개변수로 받는다. 이는 **명령행 인자(command-line arguments)** 를 받기 위한 것이다.

```
java MainStringArgs 사과 바나나 포도
    → args = { "사과", "바나나", "포도" }
```

두 가지를 기억하자.

- 인자가 없으면 `args.length` 는 **0**이다. `null` 이 아니라 **길이 0인 배열**이므로 `args.length == 0` 로 안전하게 검사할 수 있다.
- 모든 인자는 **문자열**이다. 숫자로 쓰려면 `Integer.parseInt` 등으로 변환해야 한다.

```java
String[] numberArgs = { "10", "20", "30" };
int total = 0;
for (String s : numberArgs) {
    total += Integer.parseInt(s);   // "10" → 10
}
// total == 60
```

> 💻 실습: `./run.sh MainStringArgs`
> (이 예제는 비대화형 환경이라 `main` 안에서 예시 `args` 배열을 직접 만들어 시연한다.)

## 5.12 열거(enum) 타입

열거 타입은 **정해진 몇 개의 상수만** 값으로 가질 수 있는 참조 타입이다. 요일, 방향, 신호등 색처럼 값의 범위가 한정된 경우에 쓴다.

```java
enum TrafficLight { RED, YELLOW, GREEN }

TrafficLight light = TrafficLight.RED;
```

`int` 상수(`RED=0, YELLOW=1 ...`)로 대신할 수도 있지만, enum을 쓰면 컴파일러가 "정해진 값만" 들어가도록 강제해 오타나 잘못된 값을 막아준다.

주요 기능:

- `values()` — 모든 상수를 배열로 반환 (순회에 사용)
- `valueOf("GREEN")` — 문자열로부터 해당 상수를 얻음
- `name()` — 상수 이름, `ordinal()` — 선언 순서(0부터)
- `switch` 의 `case` 라벨로 바로 사용 가능

```java
for (TrafficLight t : TrafficLight.values()) {
    switch (t) {
        case RED:    System.out.println("정지"); break;
        case YELLOW: System.out.println("주의"); break;
        case GREEN:  System.out.println("진행"); break;
    }
}
```

enum 상수는 각각 **하나의 고유한 객체**이며 프로그램 내에서 유일하게 존재한다. 그래서 `==` 로 안전하게 비교할 수 있다(5.3에서 본 참조 비교의 함정이 enum에는 없다).

> 💻 실습: `./run.sh EnumType`

## ⚠️ 흔한 오해와 함정

- **"`==` 로 문자열을 비교하면 되겠지"** — 리터럴끼리는 풀을 공유해 우연히 `true` 가 나올 수 있지만, `new String()` 이나 런타임에 만들어진 문자열은 `false` 가 된다. **문자열 내용 비교는 항상 `equals()`.**

- **"배열을 다른 변수에 대입하면 복사되겠지"** — `arr2 = arr1` 은 번지만 복사한다. 둘은 같은 배열을 가리키므로 한쪽을 바꾸면 다른 쪽도 바뀐다. 진짜 복사는 `Arrays.copyOf` 나 `System.arraycopy`.

- **"`Arrays.copyOf` 면 완전히 독립된 복사겠지"** — 기본형 배열은 그렇지만, 참조형 배열은 **얕은 복사**라 요소 객체를 원본과 공유한다(5.9).

- **"참조형 배열을 만들면 요소도 만들어지겠지"** — `new Book[3]` 은 껍데기만 만든다. 요소는 전부 `null` 이고, 이 상태에서 `books[0].title` 에 접근하면 NPE가 난다.

- **"향상된 for로 배열 값을 바꿀 수 있겠지"** — 기본형 배열에서 반복 변수는 **복사본**이라 원본이 안 바뀐다. 값을 수정하려면 인덱스 for를 써야 한다.

- **"`args` 가 비면 `null` 이겠지"** — 인자가 없어도 `args` 는 **길이 0인 배열**이다. `args == null` 이 아니라 `args.length == 0` 로 검사한다.

- **"`String` 을 바꾸면 그 변수의 내용이 바뀌겠지"** — `String` 은 불변이다. `concat`, `replace`, `toUpperCase` 등은 원본을 그대로 두고 **새 문자열을 반환**한다. 반환값을 받지 않으면 아무 일도 일어나지 않은 것처럼 보인다.

### 과제 2. 2차원 배열 만들고 순회하기 (쉬움 → 응용)

**스펙**

- `int[][] matrix = new int[3][3];` 로 생성한다(생성 직후 값은 모두 `0`).

- 바깥 `for` 는 행(`r < matrix.length`), 안쪽 `for` 는 열(`c < matrix[r].length`).
- 값 채우기는 `int value = 1;` 을 두고 `matrix[r][c] = value++;` 로 하면 편하다.

### 과제 3. 배열 복사와 얕은 복사 확인하기 (응용)

배열을 복사하는 두 가지 방법을 써 보고, **참조형 배열의 얕은 복사**가 무엇인지 눈으로 확인하는 프로그램을 작성하라.

**스펙**

- `int[] src = { 1, 2, 3, 4, 5 };` 를 만든다.
- `System.arraycopy(...)` 로 `src` 전체를 같은 길이의 새 배열 `dest` 에 복사해 출력한다.
- `java.util.Arrays.copyOf(src, 7)` 로 **길이를 7로 늘린** 복사본을 만들고, 늘어난 칸이 기본값 `0` 으로 채워지는지 확인한다.
- (핵심) 값 필드를 가진 간단한 클래스(예: `static class Box { int value; }`)로 참조형 배열을 만들고 `Arrays.copyOf` 로 복사한 뒤, **복사본의 요소를 바꾸면 원본 요소도 바뀌는지**를 출력으로 확인한다. 또한 `boxes == copy` 는 `false` 임도 출력한다.

**힌트**

- `System.arraycopy(원본, 원본시작, 대상, 대상시작, 개수)` 순서에 주의한다.
- 얕은 복사: 배열 껍데기는 새로 생기지만 안의 요소 객체(번지)는 원본과 **공유**된다. 그래서 `copy[0].value = 999;` 하면 `boxes[0].value` 도 999가 된다.

### 과제 4. ==(동일성)과 equals(동등성) 차이 확인하기 (응용)

참조 타입에서 `==` 와 `equals()` 가 어떻게 다른지 직접 출력해 비교하는 프로그램을 작성하라.

**스펙**

- 내용이 같은 두 배열 `int[] a = { 1, 2, 3 };`, `int[] b = { 1, 2, 3 };` 와, `a` 를 대입한 `int[] c = a;` 를 만든다.
- `a == b`(다른 객체 → `false`)와 `a == c`(같은 객체 → `true`)를 출력한다.
- 문자열로도 확인한다: `String s1 = "hello";`, `String s2 = "hello";`(리터럴 → 풀 공유), `String s3 = new String("hello");`(new → 별도 객체).
- `s1 == s2`(true), `s1 == s3`(false), `s1.equals(s3)`(true)를 각각 출력해 **번지 비교 vs 내용 비교**의 차이를 드러낸다.

**힌트**

- `==` 는 "같은 객체를 가리키는가"(번지)를, `equals()` 는 "내용이 같은가"를 묻는다.
- 리터럴 `"hello"` 끼리는 String Pool을 공유해 `==` 가 `true` 지만, `new String("hello")` 는 힙에 새 객체라 `==` 가 `false` 다. 내용 비교는 언제나 `equals()`.

> 답안 비교: `./run.sh ReferenceEquality`

### 과제 5. null 안전 처리와 NPE 방지 (응용)

`null` 참조가 왜 `NullPointerException` 을 일으키는지 확인하고, 이를 **안전하게 처리**하는 메서드를 직접 작성하라.

**스펙**

- `String text = null;` 을 만든 뒤 `text.length()` 를 호출해 `NullPointerException` 이 나는 것을 `try-catch` 로 잡아 메시지를 출력한다.
- `static int safeLength(String s)` 메서드를 작성한다. `s` 가 `null` 이면 `0` 을, 아니면 `s.length()` 를 반환한다. `safeLength(null)` 과 `safeLength("java")` 를 각각 호출해 출력한다.
- (응용) `static void register(String name)` 메서드를 작성한다. 시작 부분에서 `java.util.Objects.requireNonNull(name, "name은 null일 수 없습니다")` 로 검증하고, 통과하면 등록 메시지를 출력한다. `register(null)`(예외 발생 → catch)과 `register("홍길동")`(정상)을 모두 실행한다.

**힌트**

- 방법 1(`if (s == null) return 0;`)은 null을 조용히 허용하고, 방법 2(`requireNonNull`)는 null을 만나면 즉시 예외를 던져 잘못된 사용을 조기에 드러낸다.
- `Objects.requireNonNull` 은 `java.util.Objects` 를 import 해야 한다.

> 답안 비교: `./run.sh NullAndNPE`

### 과제 6. enum 정의하고 활용하기 (응용)

정해진 상수만 값으로 갖는 열거 타입을 **직접 정의**하고, `values()` · `valueOf()` · `switch` 로 활용하는 프로그램을 작성하라.

**스펙**

- 신호등 색을 나타내는 `enum TrafficLight { RED, YELLOW, GREEN }` 을 정의한다(클래스 내부 또는 파일 내부).

- `TrafficLight.values()` 로 모든 상수를 순회하며 각 상수의 `ordinal()`(선언 순서, 0부터)과 `name()`(이름)을 출력한다.

**힌트**

- enum 상수는 각각 유일한 객체라 `==` 로 안전하게 비교된다(문자열과 달리 함정이 없다).
- `for (TrafficLight t : TrafficLight.values()) { ... }` 로 순회하고, `switch (t) { case RED: ... }` 처럼 `case` 라벨에 상수 이름을 그대로 쓴다.

향상된 for의 반복 변수 `x` 는 배열 요소 그 자체인가, 복사본인가?
</details>

<details>
<summary>해설</summary>

**출력: `[1, 2, 3]`** (변하지 않음)

향상된 for에서 반복 변수 `x` 는 각 요소의 **복사본**이다. `x` 를 바꿔도 원본 배열 요소는 그대로다. 실제로 값을 바꾸려면 인덱스 for를 써야 한다.

```java
for (int i = 0; i < arr.length; i++) {
    arr[i] = arr[i] * 10;
}
```

(→ 5.10)

**문제 5.** `enum` 상수를 `==` 로 비교해도 안전한 이유는 무엇인가? 문자열은 `==` 대신 `equals()` 를 권장하는 것과 대비해 설명하시오.

<details>
<summary>힌트</summary>
enum 상수는 프로그램 안에 몇 개의 객체로 존재하는가?
</details>

<details>
<summary>해설</summary>

**enum의 각 상수는 JVM 안에 유일한 객체로 딱 하나씩만 존재하기 때문이다.**

예컨대 `TrafficLight.RED` 는 프로그램 전체에서 언제나 같은 하나의 객체를 가리킨다. 그래서 `==`(번지 비교)로 비교해도 항상 의도대로 동작한다.

반면 문자열은 리터럴이면 풀을 공유하지만 `new String()` 이나 런타임 생성 문자열은 내용이 같아도 서로 다른 객체가 될 수 있다. 그래서 문자열은 내용 비교인 `equals()` 를 써야 안전하다. (→ 5.3, 5.5, 5.12)

</details>

## 요약

- **기본 타입은 값을, 참조 타입은 객체의 번지를 저장한다.** 복사할 때 이 차이가 드러난다(값 복사 vs 번지 복사). — 5.1
- JVM 메모리는 **메서드 영역·스택·힙**으로 나뉜다. 지역 변수는 스택에, `new` 로 만든 객체는 힙에 있고, 참조 변수는 스택에서 힙 객체의 번지를 가리킨다. — 5.2
- 참조 타입의 `==` 는 **번지(같은 객체인지)** 를 비교한다. 내용 비교는 `equals()`. — 5.3
- `null` 인 변수의 멤버에 접근하면 **NullPointerException**. `null` 검사나 `Objects.requireNonNull()` 로 방지한다. — 5.4
- `String` 은 **불변**이며 리터럴은 **String Pool** 을 공유한다. `new String()` 은 별도 객체, `intern()` 은 풀 객체로 통일한다. — 5.5
- 배열은 같은 타입의 값을 담는 참조 타입이며, `new` 시 **기본값**으로 채워지고 크기는 고정이다(`length`, 인덱스 0부터). — 5.6
- 다차원 배열은 **배열의 배열**이라 행마다 길이가 다른 가변 배열이 가능하다. — 5.7
- 참조형 배열의 요소는 **번지**를 담으며, 생성 직후엔 `null` 이다. 같은 객체를 가리키는 요소는 함께 바뀐다. — 5.8
- 배열 복사(`System.arraycopy`, `Arrays.copyOf`)에서 참조형은 **얕은 복사**라 요소 객체를 공유한다. — 5.9
- **향상된 for** 의 반복 변수는 복사본이라 기본형 배열 원본을 바꾸지 못한다. — 5.10
- `main` 의 `args` 는 명령행 인자를 담는 `String[]` 이며, 인자가 없으면 **길이 0인 배열**(null 아님)이다. — 5.11
- **enum** 은 정해진 상수만 갖는 참조 타입으로, 각 상수가 유일한 객체라 `==` 로 안전하게 비교된다. — 5.12
