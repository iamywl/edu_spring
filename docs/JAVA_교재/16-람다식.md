## 16.1 람다식이란?

**람다식**은 "메소드(함수) 하나를 식(expression)처럼 간결하게 표현"하는 방법이다. 자바에서 어떤 **동작을 넘겨주려면** 예전에는 **익명 클래스(anonymous class)** 를 써야 했다. 그런데 익명 클래스는 정작 중요한 코드는 한 줄인데도 `new`, 클래스명, 메소드 선언, 중괄호 같은 **껍데기**가 너무 길어 가독성이 떨어진다.

람다식은 이 껍데기를 걷어내고 **`(매개변수) -> 실행할 코드`** 형태만 남긴다. 아래는 같은 동작을 익명 클래스와 람다식으로 각각 구현한 것이다.

```java
@FunctionalInterface
interface Greeting {
    String say(String name);            // 추상 메소드 딱 1개
}

// [1] 익명 클래스 : 껍데기가 길다
Greeting g1 = new Greeting() {
    @Override
    public String say(String name) {
        return "안녕하세요, " + name + "님!";
    }
};

// [2] 람다식 : 핵심 코드만 남긴다
Greeting g2 = (String name) -> { return "반갑습니다, " + name + "님!"; };

// [3] 더 축약 : 타입 생략 + 문장 하나면 중괄호/return 생략
Greeting g3 = name -> "환영합니다, " + name + "님!";
```

람다식이 담기는 그릇은 **함수형 인터페이스(functional interface)** 다. 이는 **추상 메소드가 딱 1개**인 인터페이스로, 람다식은 바로 그 유일한 추상 메소드의 구현을 대신한다. `@FunctionalInterface` 어노테이션을 붙이면 컴파일러가 "추상 메소드 1개" 규칙을 검사해 준다(필수는 아니지만 권장). 자바가 미리 제공하는 표준 함수형 인터페이스도 많은데, 예컨대 `Supplier<T>`는 매개변수 없이 값 하나를 공급한다.

## 16.2 매개변수 없는 람다식

함수형 인터페이스의 추상 메소드가 **매개변수를 받지 않을** 때는, 람다식의 매개변수 자리를 **빈 괄호 `()`** 로 표현한다.

```java
() -> 실행할 코드
```

실행문이 하나면 중괄호와 `return`을 생략할 수 있고, 여러 줄이면 중괄호 `{ }`로 감싸고 `return`을 명시한다.

```java
@FunctionalInterface
interface MessageMaker { String make(); }   // 매개변수 없음, 리턴 있음

MessageMaker m1 = () -> "매개변수 없는 람다식입니다.";   // 한 줄
MessageMaker m2 = () -> {                                // 여러 줄
    String part1 = "여러 줄";
    String part2 = "실행 가능";
    return part1 + " " + part2;
};
```

대표적인 표준 함수형 인터페이스 두 가지를 알아 두면 좋다. **`Runnable`** 은 `void run()` — 입력도 리턴도 없이 **동작만** 수행한다. **`Supplier<T>`** 는 `T get()` — 입력은 없지만 값 하나를 **공급**한다.

```java
Runnable task = () -> System.out.println("동작만 수행");
Supplier<Integer> six = () -> 6;   // 항상 6을 공급
```

## 16.3 매개변수 있는 람다식

추상 메소드가 매개변수를 받을 때는 괄호 안에 매개변수를 쓴다. `(x) -> ...`는 1개, `(x, y) -> ...`는 2개다. 여기에 세 가지 **축약 규칙**이 적용된다.

- 매개변수 **타입은 대부분 생략**할 수 있다(컴파일러가 문맥으로 추론).
- 매개변수가 **정확히 1개**이면 **괄호도 생략**할 수 있다. 예) `x -> ...`
- 매개변수가 **0개이거나 2개 이상**이면 괄호는 **반드시** 필요하다.

```java
@FunctionalInterface interface IntPrinter { void print(int value); }

IntPrinter p1 = (int value) -> System.out.println("받은 값: " + value); // 타입 명시
IntPrinter p2 = value -> System.out.println("제곱: " + value * value);  // 타입/괄호 생략
```

값을 **소비만 하고 리턴하지 않는** 표준 함수형 인터페이스로 **`Consumer<T>`** (`void accept(T)`, 입력 1개)와 **`BiConsumer<T, U>`** (`void accept(T, U)`, 입력 2개)가 있다.

```java
Consumer<String> shout = text -> System.out.println(text.toUpperCase() + "!!!");
BiConsumer<String, Integer> repeat = (word, count) ->
        System.out.println(word + " x " + count);
```

## 16.4 리턴값 있는 람다식

람다식이 값을 리턴하는 방법은 두 가지다.

1. **중괄호 `{ }`** 를 쓰고 그 안에서 **명시적으로 `return`** 한다. → `(a, b) -> { return a + b; }`
2. **중괄호 없이 식(expression) 하나**만 쓴다. 이때는 그 식의 결과가 **자동으로 리턴**되며 `return` 키워드를 **쓰지 않는다(써서는 안 된다).** → `(a, b) -> a + b`

```java
@FunctionalInterface interface IntCalculator { int calc(int a, int b); }

IntCalculator add = (a, b) -> { int r = a + b; return r; }; // 중괄호 + return
IntCalculator mul = (a, b) -> a * b;                        // 식 하나 → 자동 리턴
```

값을 **리턴하는** 표준 함수형 인터페이스가 특히 자주 쓰인다. **`Function<T, R>`** 는 `R apply(T)` — 입력 1개, 리턴 1개. **`BiFunction<T, U, R>`** 는 입력 2개, 리턴 1개. **`Predicate<T>`** 는 `boolean test(T)` — 조건 검사에 쓰이며 boolean을 리턴한다.

```java
Function<String, Integer> length = s -> s.length();
Predicate<Integer> isEven = n -> n % 2 == 0;
```

## 16.5 메소드 참조

람다식이 하는 일이 **이미 존재하는 메소드 하나를 그대로 호출**하는 것뿐이라면, **메소드 참조(method reference)** 로 더 짧게 쓸 수 있다. 메소드 참조는 결국 람다식의 축약 표현이다.

```java
s -> Integer.parseInt(s)   // 람다
Integer::parseInt          // 메소드 참조 (같은 의미)
```

메소드 참조에는 세 종류가 있다(생성자 참조는 16.6에서 다룬다).

- **정적 메소드 참조** : `클래스명::정적메소드` — 예) `Integer::parseInt`
- **인스턴스 메소드 참조** : `클래스명::인스턴스메소드` — **첫 번째 인자가 호출 대상**이 된다. 예) `String::length` (`s.length()`와 같음)
- **특정 객체의 메소드 참조** : `객체참조::인스턴스메소드` — 특정 객체에 고정. 예) `System.out::println`

```java
Function<Integer, Integer> square = MathUtil::square;   // 정적
Function<String, Integer> length = String::length;      // 인스턴스 (첫 인자가 호출 대상)
Consumer<String> printer = System.out::println;         // 특정 객체
```

> 💻 실습: `./run.sh MethodReference`

## 16.6 생성자 참조

람다식이 하는 일이 **새 객체를 생성**하는 것뿐이라면, **생성자 참조(constructor reference)** 로 축약할 수 있다. 형태는 **`클래스명::new`** 다.

```java
() -> new User()          // 람다  →  User::new  (기본 생성자)
name -> new User(name)    // 람다  →  User::new  (인자 1개 생성자)
```

흥미로운 점은, 겉모습이 똑같이 `User::new`인데도 **어떤 생성자가 호출될지는 "대상 함수형 인터페이스의 시그니처"가 결정**한다는 것이다. 인자가 0개인 `Supplier<User>`에 담으면 `new User()`가, 인자가 1개인 `Function<String, User>`에 담으면 `new User(String)`이 호출된다.

```java
Supplier<User> defaultFactory = User::new;          // new User()      호출
Function<String, User> namedFactory = User::new;    // new User(name)  호출

User u1 = defaultFactory.get();
User u2 = namedFactory.apply("홍길동");
```

이렇게 생성자 참조는 객체를 찍어내는 **팩토리(factory)** 처럼 재사용할 수 있어, 컬렉션의 각 요소로부터 객체를 만들 때 특히 편리하다.

> 💻 실습: `./run.sh ConstructorReference`

## ⚠️ 흔한 오해와 함정

- **함수형 인터페이스에 추상 메소드가 2개면 람다를 못 쓴다.** 람다식은 "유일한 추상 메소드 1개"를 구현하는 것이므로, 추상 메소드가 2개 이상인 인터페이스에는 람다를 담을 수 없다. `@FunctionalInterface`를 붙여 두면 이 실수를 컴파일 시점에 잡을 수 있다.
- **식(expression) 람다에 `return`을 붙인다.** 중괄호 없는 `(a, b) -> a + b`는 결과가 **자동 리턴**된다. 여기에 `return`을 쓰면 컴파일 오류다. `return`을 쓰려면 반드시 중괄호 `{ }`로 감싸야 한다.
- **매개변수가 2개인데 괄호를 생략한다.** 괄호 생략은 **매개변수가 정확히 1개일 때만** 가능하다. `a, b -> ...`는 오류이며 `(a, b) -> ...`로 써야 한다. 0개일 때도 `()`가 필요하다.
- **인스턴스 메소드 참조에서 "누가 호출 대상인지"를 헷갈린다.** `String::length`는 **첫 번째 인자가 호출 대상**이 되어 `s -> s.length()`와 같다. 반면 `System.out::println`은 **특정 객체(`System.out`)에 고정**된 형태다. 둘의 차이를 구분하자.
- **람다 안에서 바깥 지역 변수를 바꾸려 한다.** 람다가 캡처하는 지역 변수는 **사실상 final(effectively final)** 이어야 한다. 즉 캡처한 변수의 값을 람다 안팎에서 바꾸면 컴파일 오류가 난다. (자세한 이유는 개념서 6장의 "변수 캡처" 참고.)
- **`User::new`가 항상 기본 생성자를 부른다고 생각한다.** 어떤 생성자가 불릴지는 **대상 함수형 인터페이스의 인자 개수/타입**이 결정한다. 같은 `User::new`라도 담기는 그릇에 따라 다른 생성자가 호출된다.

## 요약

- **람다식**은 함수 하나를 `(매개변수) -> 실행코드` 형태로 간결히 표현한 것으로, **익명 클래스의 껍데기를 걷어낸** 축약 표현이다. 람다가 담기는 그릇은 **추상 메소드가 1개인 함수형 인터페이스**다. (`LambdaIntro`)
- **매개변수 없는 람다**는 빈 괄호 `() -> ...`로 쓴다. `Runnable`(입력·리턴 없음), `Supplier<T>`(리턴만 있음)가 대표적이다. (`NoParamLambda`)
- **매개변수 있는 람다**는 `(x)`, `(x, y)`로 쓰며, 타입은 대부분 생략 가능하고 **매개변수가 1개일 때만** 괄호도 생략할 수 있다. `Consumer`/`BiConsumer`는 값을 소비만 한다. (`ParamLambda`)
- **리턴값 있는 람다**는 중괄호 + `return`을 쓰거나, 식 하나만 써서 **자동 리턴**한다. `Function`, `BiFunction`, `Predicate`가 값을 리턴하는 표준 인터페이스다. (`ReturnLambda`)
- **메소드 참조**는 이미 있는 메소드를 그대로 호출하는 람다의 축약이다. 정적(`클래스명::정적`), 인스턴스(`클래스명::인스턴스`, 첫 인자가 호출 대상), 특정 객체(`객체::인스턴스`) 세 종류가 있다. (`MethodReference`)
- **생성자 참조**는 `클래스명::new`로 객체 생성을 축약하며, **어떤 생성자가 호출될지는 대상 함수형 인터페이스의 시그니처**가 결정한다. (`ConstructorReference`)
