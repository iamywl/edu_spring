# 들어가며: 왜 "왜?"를 물어야 하는가

> "코드를 외우는 것이 아니라, 왜 그런지를 이해하는 것이 진짜 실력이다."

---

## 이 책의 목적

프로그래밍 책을 펼치면 대부분 이렇게 시작한다.

```java
int a = 10;
```

그리고 "변수 a에 10을 대입한다"라고 설명한다.

하지만 정작 중요한 질문에는 답하지 않는다.

- **왜** 변수라는 게 필요한가?
- **왜** `int`라고 써야 하는가?
- 컴퓨터 내부에서는 **실제로** 무슨 일이 벌어지는가?

이 책은 다르다. 이 책의 목표는 단 하나다.

> **"왜?"에 답하는 것.**

코드를 100줄 외우는 것보다, "왜 그렇게 동작하는가?"를 이해하는 것이 훨씬 가치 있다.
문법은 검색하면 나온다. 하지만 개념은 머릿속에 있어야 한다.

한 가지 예를 들어보자. 초보자는 `0.1 + 0.2`가 `0.3`이 아니라는 사실을 마주치면
"자바가 이상하다"거나 "버그다"라고 생각한다. 하지만 그 이유(§2.6 IEEE-754)를 이해한 사람은
같은 현상을 파이썬, 자바스크립트, C++ 어디서 만나도 당황하지 않는다.
**하나의 "왜?"가 열 개의 언어를 관통하는 것**이다. 이 책은 그런 "왜?"들을 모아둔 책이다.

---

## Java를 배우기 전에 알아야 할 것

프로그래밍을 배운다는 것은 **새로운 언어를 배우는 것**과 같다.

영어를 배울 때 문법만 외우면 말을 못 한다.
문법의 **이유**를 이해하고, 그 언어로 **생각하는 법**을 익혀야 한다.

Java도 마찬가지다. 문법을 외우는 것이 아니라,
**컴퓨터가 생각하는 방식**을 이해하고,
**그 방식으로 문제를 해결하는 법**을 익히는 것이다.

```
  문법을 외운 사람                 원리를 이해한 사람
  ┌────────────────┐              ┌────────────────┐
  │ "이렇게 하니까  │              │ "이건 이래서    │
  │  동작하더라"    │              │  이렇게 동작해" │
  │                 │              │                 │
  │ 새 상황 → 막힘   │              │ 새 상황 → 응용   │
  └────────────────┘              └────────────────┘
```

---

## 이 책의 대상 독자

- Java를 **처음** 배우는 사람 (문법책을 봤지만 "왜?"가 궁금했던 사람)
- 코드는 짜지만 **내부 동작 원리**가 궁금한 입문~초급 개발자
- 컴퓨터공학 **전공 기초**를 Java로 다지려는 학생
- **기술 면접**을 준비하며 "그냥 외운" 지식을 진짜 이해로 바꾸고 싶은 사람

이미 실무 경험이 있는 개발자라도, 평소 블랙박스로 넘겨왔던 JVM/바이트코드/GC/타입소거 같은
주제를 다시 정리하는 데 도움이 될 것이다.

## 이 책의 구성과 특징

이 책의 모든 장은 같은 형식을 따른다. 그래서 어느 장을 펼쳐도 길을 잃지 않는다.

- **도입**: 이 장이 왜 필요한가? (문제 제기)
- **개념별 소절**: 정의 → **원리(왜 그렇게 동작하는가)** → ASCII 다이어그램 → 코드/예제
- **⚠️ 흔한 오해와 함정**: 초보자가 반드시 밟는 지뢰들
- **연습문제**: 힌트와 해설이 붙은 문제 (직접 풀어봐야 는다)
- **요약**: 그 장의 핵심을 3~5줄로

또한 개념만 설명하고 끝내지 않는다. 핵심 개념 곁에는 이런 표시가 붙어 있다.

> 실행 데모: `./run.sh FloatingPointBits`  📁 `chapter01-java-basics/src/main/java/com/edu/basics/FloatingPointBits.java`

이것은 그 개념을 **직접 눈으로 확인할 수 있는 실행 예제**가 프로젝트에 준비되어 있다는 뜻이다.
글로 "0.1 + 0.2는 0.3이 아니다"를 읽는 것과, 터미널에서 실제로 `0.30000000000000004`가
찍히는 것을 보는 것은 완전히 다른 경험이다. 개념을 읽었다면 반드시 데모를 돌려보자.

## 이 책의 목차와 추천 학습 순서

각 장은 앞 장의 개념 위에 쌓이도록 배치되어 있다. **순서대로 읽는 것**을 강력히 권한다.

1. **1장 컴퓨터는 어떻게 프로그램을 실행하는가?** — JVM, 바이트코드, JIT, 클래스 로딩, GC. 모든 것의 토대.
2. **2장 데이터를 담는 그릇 - 변수** — 기본형/참조형, 2의 보수, IEEE-754, char, pass-by-value.
3. **3장 프로그램의 흐름을 제어하다** — 조건문, 반복문, 흐름 제어.
4. **4장 객체지향이란 무엇인가?** — 이 책의 심장. 캡슐화/상속/다형성/추상화.
5. **5장 데이터를 효율적으로 다루다 - 컬렉션** — List/Set/Map, Big-O, HashMap 내부, 트리, 제네릭.
6. **6장 함수형 프로그래밍으로의 진화** — Lambda, Stream, Optional.
7. **7장 예외처리 - 실패에 대비하다** — 예외 계층, Checked/Unchecked, Best Practice.
8. **8장 자바의 현대적 기능들** — record, sealed, 패턴 매칭, switch expression.
9. **마무리 - 자바 학습 로드맵** — 초급에서 실무까지의 지도.

전체 목차와 각 장의 자세한 안내는 [README(책 표지)](./README.md)에서 확인할 수 있다.
