# Java 개념서: 왜 그렇게 동작하는가?

> "코드를 외우는 것이 아니라, 왜 그런지를 이해하는 것이 진짜 실력이다."

```
  ┌────────────────────────────────────────────────────────┐
  │                                                        │
  │        Java 개념서                                     │
  │        ─────────────                                   │
  │        왜 그렇게 동작하는가?                           │
  │                                                        │
  │        문법이 아니라 원리를,                           │
  │        암기가 아니라 이해를.                           │
  │                                                        │
  └────────────────────────────────────────────────────────┘
```

---

## 이 책은 무엇인가

대부분의 프로그래밍 책은 "이렇게 쓰면 된다"까지만 말한다.
이 책은 거기서 한 걸음 더 나아가 **"왜 그렇게 되는가?"**에 답한다.

- `0.1 + 0.2`는 왜 `0.3`이 아닌가?
- Java는 왜 "한 번 작성하면 어디서든 실행"되는가?
- HashMap은 왜 평균 O(1)로 빠른가?
- Java는 왜 항상 pass-by-value인가?

이런 질문들에 다이어그램과 비유, 그리고 직접 돌려볼 수 있는 실행 데모로 답한다.

## 학습 목표

이 책을 끝까지 읽으면 다음을 할 수 있게 된다.

- JVM이 소스코드를 실행하는 전 과정(바이트코드 → JIT → GC)을 하나의 그림으로 설명한다.
- 기본형과 참조형, `==`와 `equals()`, pass-by-value를 정확히 구분한다.
- 객체지향 4대 특성(캡슐화·상속·다형성·추상화)을 "왜?"의 관점에서 이해하고 설계에 적용한다.
- 상황에 맞는 컬렉션을 고르고, Big-O를 스스로 유도한다.
- Lambda·Stream·Optional로 선언형 코드를 작성한다.
- 예외를 올바르게 설계하고, record·sealed 같은 현대 기능을 활용한다.

## 이 책의 대상

- Java를 **처음** 배우는 사람
- 문법은 알지만 **내부 원리**가 궁금한 입문~초급 개발자
- 컴퓨터공학 **전공 기초**를 다지려는 학생
- **기술 면접**을 준비하는 사람

## 추천 학습 순서

각 장은 앞 장의 개념 위에 쌓이도록 설계되었다. **번호 순서대로** 읽는 것을 강력히 권한다.

```
  들어가며 → 1장 → 2장 → 3장 → 4장(핵심) → 5장 → 6장 → 7장 → 8장 → 마무리
     │         │                    │
     │      토대를 놓는 장          심장이 되는 장
     │      (반드시 먼저)          (가장 꼼꼼히)
     └── 이 책을 왜/어떻게 읽는지
```

- 급하다면 최소한 **1장 → 2장 → 4장**은 순서대로 읽자. 나머지 장의 이해가 여기서 갈린다.
- 각 장은 독립적으로 읽어도 되지만, 뒤 장이 앞 장의 개념(예: 5장의 트리 ↔ 5장 HashMap treeify, 8장 sealed ↔ 4장 다형성)을 참조한다.

## 각 장의 형식

모든 장은 같은 틀을 따른다. 어느 장을 펼쳐도 길을 잃지 않는다.

1. **도입** — 이 장이 왜 필요한가 (문제 제기)
2. **개념별 소절** — 정의 → 원리(왜) → ASCII 다이어그램 → 코드/예제
3. **⚠️ 흔한 오해와 함정**
4. **연습문제** — 힌트/해설 포함
5. **요약** — 핵심 3~5줄
6. **이전/다음 장 링크**
