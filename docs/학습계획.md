## 학습 효과를 높이는 팁

```bash
# 읽기만 하지 말고, 반드시 Docker로 실행
docker compose up --build

# 코드를 수정하고 다시 실행하며 차이를 확인
```

### 2. 3단계 학습법을 따르라
```
1단계: 개념서로 "왜?"를 이해한다
2단계: 교육자료의 코드를 보며 "어떻게?"를 파악한다
3단계: 직접 코드를 수정/추가하며 "내 것"으로 만든다
```

- 오늘 배운 핵심 개념 3줄 요약
- 헷갈렸던 부분 1가지
- 내일 확인할 것 1가지

- 주중 5일 학습 + 주말 복습이 이상적

## CS 기반 심화 트랙 (비전공자 → 전공자 수준)

위 8주 과정은 "Java/Spring으로 동작하는 것을 만든다"에 초점이 있다.
**전공자 수준의 기초**(자료구조·알고리즘, 운영체제, 데이터베이스 이론, 네트워크)를 쌓으려면
아래 CS 트랙을 병행/후행 학습한다. 각 트랙은 개념서 + 직접 실행·측정하는 예제로 구성된다.

| 트랙 | 개념서 | 실습 | 언제 |
|------|--------|------|------|
| 자료구조·알고리즘 | `docs/CS_알고리즘_개념서/README.md` | `chapter-cs-algorithms/` (`./run.sh SearchAlgorithms`/`SortingAlgorithms`/`RecursionAndDP`/`DataStructuresFromScratch`/`TreeAndHeap`/`GraphAlgorithms`) | 3주차(컬렉션)와 병행하면 시너지 |
| 운영체제·동시성 | `docs/CS_운영체제_개념서/README.md` | `chapter-cs-concurrency/` (`./run.sh RaceConditionDemo`/`SynchronizationDemo`/`DeadlockDemo`/`ExecutorAndFutures`/`ProducerConsumer`) | Spring 스레드/트랜잭션 학습 전 |
| 데이터베이스 이론 | `docs/CS_데이터베이스_개념서/README.md` | `chapter-cs-database/labs/*.sql` (인프라 PostgreSQL) | 5주차(JPA) 전에 읽으면 밑단이 보임 |
| 컴퓨터 네트워크 | `docs/CS_네트워크_개념서/README.md` | `chapter-cs-network/` (`./run.sh RawHttpClient`, `labs/observe_http.sh`) | 4주차(REST) 전에 |

> 개념서(JAVA/SPRING)에도 머신 레이어 심화 섹션이 추가됨: 바이트코드/JIT/GC/클래스로딩,
> 2의 보수·IEEE-754·타입 소거, @Transactional 프록시 원리, ACID/격리수준/B-tree 인덱스,
> HMAC·대칭/비대칭 암호. "왜 이렇게 동작하는가"를 코드로 확인하며 읽는다.
