## 1. 도입

Java 표준 라이브러리에서 가장 많이 쓰이는 두 자료구조—`TreeMap`과 `PriorityQueue`—의 내부를 파고들면 모두 **트리** 기반 구조가 나온다. `TreeMap`은 **레드블랙 트리(Red-Black Tree)**로, `PriorityQueue`는 **이진 힙(Binary Heap)**으로 구현되어 있다.

이 두 자료구조가 왜 배열이나 연결 리스트보다 빠른가? 핵심은 탐색 공간을 매 단계마다 절반씩 줄이는 **로그 시간** 연산에 있다. 배열의 이진 탐색이 정렬된 배열을 전제로 하는 것과 달리, 트리 구조는 **삽입·삭제 중에도 정렬 순서를 유지**하므로 동적인 데이터셋에 적합하다.

이 장에서는 다음 순서로 진행한다.

- 이진 탐색 트리(BST)의 구조, 삽입·탐색·삭제 연산과 그 복잡도
- 트리 순회 4종(전위/중위/후위/레벨)
- 정렬된 순서로 삽입했을 때 발생하는 BST 퇴화 현상
- 퇴화를 막는 자가균형 트리: AVL 트리와 레드블랙 트리
- 완전이진트리를 배열 하나로 표현하는 최소 힙(MinHeap)
- sift-up / sift-down 연산과 build-heap의 O(n) 비밀
- 힙 정렬(Heap Sort)

## 2. 트리의 기본 개념

**트리(Tree)**는 노드(Node)와 간선(Edge)으로 구성된 계층적 자료구조이다. 배열이나 연결 리스트가 선형(1차원)인 것과 달리, 트리는 **비선형(Non-linear)**이며 하나의 루트(Root)에서 분기한다.

```
         루트(Root)
            50
           /  \
         30    70        ← 레벨 1 (루트의 자식)
        /  \  /  \
       20  40 60  80     ← 레벨 2 (리프 노드)
```

### 주요 용어

| 용어 | 설명 |
|------|------|
| 루트(Root) | 최상위 노드 (부모 없음) |
| 부모(Parent) | 특정 노드의 바로 위 노드 |
| 자식(Child) | 특정 노드의 바로 아래 노드 |
| 리프(Leaf) | 자식이 없는 노드 |
| 높이(Height) | 루트에서 가장 깊은 리프까지의 간선 수 |
| 깊이(Depth) | 루트에서 특정 노드까지의 간선 수 |
| 차수(Degree) | 노드의 자식 수 |

**이진 트리(Binary Tree)**는 각 노드가 최대 2개의 자식(왼쪽, 오른쪽)을 갖는 트리이다.

**완전 이진 트리(Complete Binary Tree)**는 마지막 레벨을 제외한 모든 레벨이 꽉 차 있고, 마지막 레벨의 노드가 왼쪽부터 채워진 트리이다. 힙은 완전 이진 트리의 성질을 이용해 배열 하나로 표현된다.

## 3. 이진 탐색 트리(BST)

### 3.1 BST 구조와 성질

**이진 탐색 트리(Binary Search Tree, BST)**는 다음 성질을 만족하는 이진 트리이다.

> 임의의 노드 N에 대해:  
> **왼쪽 서브트리의 모든 값 < N의 값 < 오른쪽 서브트리의 모든 값**

이 성질 덕분에 탐색 시 루트에서 시작해 값을 비교할 때마다 탐색 범위가 절반으로 줄어든다.

```
          50
         /  \
        30   70
       /  \  / \
      20  40 60  80
```

위 트리에서 임의의 노드를 선택해 BST 성질을 검증할 수 있다.
- 노드 50: 왼쪽(20, 30, 40) 모두 < 50, 오른쪽(60, 70, 80) 모두 > 50
- 노드 30: 왼쪽(20) < 30, 오른쪽(40) > 30

### Java 노드 구조

```java
class Node {
    int value;
    Node left, right;

    Node(int value) {
        this.value = value;
        this.left = null;
        this.right = null;
    }
}

class BST {
    Node root;

    BST() {
        root = null;
    }
}
```

### 3.2 탐색

BST에서 값 `target`을 찾는 과정은 다음과 같다.

1. 현재 노드가 null이면 탐색 실패 (값 없음)
2. `target == 현재 노드 값` → 탐색 성공
3. `target < 현재 노드 값` → 왼쪽 자식으로 이동
4. `target > 현재 노드 값` → 오른쪽 자식으로 이동

**contains(60) 탐색 과정 (위 트리 기준)**

```
단계 1: 루트 50 확인 → 60 > 50 → 오른쪽으로
단계 2: 노드 70 확인 → 60 < 70 → 왼쪽으로
단계 3: 노드 60 확인 → 60 == 60 → 탐색 성공!
```

비교 횟수: 3회 (트리 높이에 비례)

**의사코드**

```
function search(node, target):
    if node == null:
        return false
    if target == node.value:
        return true
    if target < node.value:
        return search(node.left, target)
    else:
        return search(node.right, target)
```

**Java 구현**

```java
// 재귀 버전
public boolean contains(int target) {
    return containsRecursive(root, target);
}

private boolean containsRecursive(Node node, int target) {
    if (node == null) return false;
    if (target == node.value) return true;
    if (target < node.value) return containsRecursive(node.left, target);
    return containsRecursive(node.right, target);
}

// 반복 버전 (스택 오버플로 방지)
public boolean containsIterative(int target) {
    Node current = root;
    while (current != null) {
        if (target == current.value) return true;
        current = (target < current.value) ? current.left : current.right;
    }
    return false;
}
```

### 3.3 삽입

삽입은 탐색과 동일한 경로를 따라 내려가다가 null 위치에 새 노드를 놓는다.

**의사코드**

```
function insert(node, value):
    if node == null:
        return new Node(value)
    if value < node.value:
        node.left = insert(node.left, value)
    else if value > node.value:
        node.right = insert(node.right, value)
    // value == node.value: 중복은 무시하거나 정책에 따라 처리
    return node
```

**Java 구현**

```java
public void insert(int value) {
    root = insertRecursive(root, value);
}

private Node insertRecursive(Node node, int value) {
    if (node == null) return new Node(value);
    if (value < node.value) {
        node.left = insertRecursive(node.left, value);
    } else if (value > node.value) {
        node.right = insertRecursive(node.right, value);
    }
    // 중복 값은 무시
    return node;
}
```

**삽입 과정 시각화**: 50, 30, 70, 20, 40, 60, 80을 순서대로 삽입

```
삽입 50:    50

삽입 30:    50
           /
          30

삽입 70:    50
           /  \
          30   70

삽입 20:    50
           /  \
          30   70
         /
        20

삽입 40:    50
           /  \
          30   70
         /  \
        20   40

삽입 60:    50
           /  \
          30   70
         /  \  /
        20  40 60

삽입 80:    50
           /  \
          30   70
         /  \  / \
        20  40 60  80   ← 완성
```

### 3.4 삭제 (3가지 경우)

BST 삭제는 삽입보다 복잡하다. 삭제할 노드의 자식 수에 따라 3가지 경우를 처리해야 한다.

#### 경우 1: 리프 노드 삭제

자식이 없으므로 단순히 제거한다.

```
삭제 전:         삭제 20 후:
    30               30
   /  \                \
  20   40              40
```

#### 경우 2: 자식이 1개인 노드 삭제

삭제할 노드를 유일한 자식 노드로 대체한다.

```
삭제 전:         삭제 30 후:
    50               50
   /  \             /  \
  30   70          20   70
 /
20
```

#### 경우 3: 자식이 2개인 노드 삭제 (후계자 방법)

가장 복잡한 경우이다. **중위 후계자(In-order Successor)**—오른쪽 서브트리에서 가장 작은 값—를 찾아 삭제할 노드의 값을 교체하고, 후계자를 삭제한다.

왜 중위 후계자인가? BST 성질(왼쪽 < 현재 < 오른쪽)을 유지하면서 노드를 교체하려면, 현재 노드보다 크면서 오른쪽 서브트리에서 가장 작은 값이 적합하다.

```
삭제 전:         삭제 50 (루트) 후:
     50               60
    /  \             /  \
   30   70          30   70
       /  \              \
      60   80            80

후계자 60 (70의 왼쪽 서브트리 최솟값)이 50의 자리를 대체
```

**Java 구현 (삭제 전체)**

```java
public void delete(int value) {
    root = deleteRecursive(root, value);
}

private Node deleteRecursive(Node node, int value) {
    if (node == null) return null;  // 값이 존재하지 않음

    if (value < node.value) {
        node.left = deleteRecursive(node.left, value);
    } else if (value > node.value) {
        node.right = deleteRecursive(node.right, value);
    } else {
        // 삭제 대상 노드 발견
        // 경우 1: 리프 노드
        if (node.left == null && node.right == null) {
            return null;
        }
        // 경우 2: 자식이 1개
        if (node.left == null) return node.right;
        if (node.right == null) return node.left;

        // 경우 3: 자식이 2개 → 중위 후계자로 교체
        Node successor = findMin(node.right);  // 오른쪽 서브트리의 최솟값
        node.value = successor.value;           // 값 교체
        node.right = deleteRecursive(node.right, successor.value); // 후계자 삭제
    }
    return node;
}

private Node findMin(Node node) {
    while (node.left != null) node = node.left;
    return node;
}
```

### 3.5 중위순회와 정렬

BST를 **중위순회(In-order Traversal)**하면 저장된 모든 값이 **오름차순**으로 출력된다. 이것은 BST의 가장 중요한 성질 중 하나이다.

중위순회 규칙: **왼쪽 서브트리 → 현재 노드 → 오른쪽 서브트리**

```
트리:
     50
    /  \
   30   70
  /  \  / \
 20  40 60  80

중위순회 순서:
  20 → 30 → 40 → 50 → 60 → 70 → 80
```

**Java 구현**

```java
public void inorder() {
    inorderRecursive(root);
    System.out.println();
}

private void inorderRecursive(Node node) {
    if (node == null) return;
    inorderRecursive(node.left);    // 1. 왼쪽
    System.out.print(node.value + " ");  // 2. 현재
    inorderRecursive(node.right);   // 3. 오른쪽
}
```

## 4. 트리 순회 4종

트리 순회(Traversal)는 모든 노드를 한 번씩 방문하는 방법이다. 방문 순서에 따라 4가지로 분류된다.

### 4.1 전위순회 (Pre-order)

**현재 노드 → 왼쪽 → 오른쪽**

주로 트리 구조를 복사하거나 직렬화할 때 사용한다.

```
결과: 50 → 30 → 20 → 40 → 70 → 60 → 80
```

```java
private void preorderRecursive(Node node) {
    if (node == null) return;
    System.out.print(node.value + " ");  // 1. 현재
    preorderRecursive(node.left);         // 2. 왼쪽
    preorderRecursive(node.right);        // 3. 오른쪽
}
```

### 4.2 중위순회 (In-order)

**왼쪽 → 현재 노드 → 오른쪽**

BST에서 사용하면 정렬 순서로 출력된다.

```
결과: 20 → 30 → 40 → 50 → 60 → 70 → 80
```

### 4.3 후위순회 (Post-order)

**왼쪽 → 오른쪽 → 현재 노드**

주로 트리를 삭제하거나 수식 트리를 계산할 때 사용한다. 자식을 모두 처리한 후 부모를 처리한다.

```
결과: 20 → 40 → 30 → 60 → 80 → 70 → 50
```

```java
private void postorderRecursive(Node node) {
    if (node == null) return;
    postorderRecursive(node.left);        // 1. 왼쪽
    postorderRecursive(node.right);       // 2. 오른쪽
    System.out.print(node.value + " ");  // 3. 현재
}
```

### 4.4 레벨순회 (Level-order, BFS)

**루트 레벨부터 위→아래, 왼→오른쪽으로** 방문. BFS(너비 우선 탐색)와 동일한 원리이며, 큐(Queue)를 사용한다.

```
결과: 50 → 30 → 70 → 20 → 40 → 60 → 80
```

```java
public void levelOrder() {
    if (root == null) return;
    Queue<Node> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        Node current = queue.poll();
        System.out.print(current.value + " ");

        if (current.left != null) queue.offer(current.left);
        if (current.right != null) queue.offer(current.right);
    }
    System.out.println();
}
```

### 순회 비교표

| 순회 종류 | 방문 순서 | 활용 |
|----------|----------|------|
| 전위(Pre-order) | 현재 → 왼 → 오 | 트리 복사, 직렬화 |
| 중위(In-order) | 왼 → 현재 → 오 | BST 정렬 출력 |
| 후위(Post-order) | 왼 → 오 → 현재 | 트리 삭제, 수식 계산 |
| 레벨(Level-order) | 레벨 순서대로 | BFS, 최단 경로, 힙 표현 |

## 5. BST 퇴화와 자가균형 트리

### 5.1 퇴화(Degenerate) BST

BST는 입력 데이터가 **이미 정렬된 순서**로 삽입될 때 한쪽으로만 자라는 문제가 발생한다. 이를 **퇴화(Degenerate) BST** 또는 **사향 트리(Skewed Tree)**라고 한다.

**1, 2, 3, 4를 순서대로 삽입했을 때:**

```
삽입 1:  1
삽입 2:  1
          \
           2
삽입 3:  1
          \
           2
            \
             3
삽입 4:  1
          \
           2
            \
             3
              \
               4
```

이 경우 트리는 사실상 **연결 리스트**와 동일한 구조가 된다. 탐색 시 모든 노드를 순회해야 하므로 O(n) 시간이 걸린다.

**균형 트리 vs 퇴화 트리 비교:**

| N (노드 수) | 균형 트리 높이 | 퇴화 트리 높이 | 최악 탐색 비교 횟수 차이 |
|------------|--------------|--------------|----------------------|
| 7          | 3            | 7            | 3회 vs 7회           |
| 31         | 5            | 31           | 5회 vs 31회          |
| 63         | 6            | 63           | 6회 vs 63회          |
| 1,023      | 10           | 1,023        | 10회 vs 1,023회      |
| 1,048,575  | 20           | 1,048,575    | 20회 vs 1,048,575회  |

데이터가 클수록 균형 트리의 이점이 기하급수적으로 커진다.

---

### 5.2 AVL 트리

**AVL 트리(Adelson-Velsky and Landis Tree)**는 1962년 제안된 최초의 자가균형 이진 탐색 트리이다. 모든 노드에서 **균형 인수(Balance Factor)**를 유지함으로써 항상 O(log n) 높이를 보장한다.

**균형 인수(Balance Factor, BF)**:
```
BF(node) = height(node.left) - height(node.right)
```

AVL 트리는 모든 노드에서 BF가 -1, 0, 1 중 하나여야 한다. BF가 이 범위를 벗어나면 **회전(Rotation)**으로 균형을 복구한다.

#### 4가지 회전

회전이 필요한 상황은 불균형이 발생한 위치에 따라 4가지로 나뉜다.

**LL 회전 (Left-Left Case): 우회전**

불균형 노드의 왼쪽 자식의 왼쪽에 삽입된 경우.

```
불균형 전:           우회전 후:
      C (BF=2)            B
     /                   / \
    B (BF=1)            A   C
   /
  A
```

```
회전 방법: B를 루트로 올리고, C는 B의 오른쪽 자식이 된다.
```

**RR 회전 (Right-Right Case): 좌회전**

불균형 노드의 오른쪽 자식의 오른쪽에 삽입된 경우.

```
불균형 전:     좌회전 후:
  A (BF=-2)        B
   \               / \
    B (BF=-1)     A   C
     \
      C
```

**LR 회전 (Left-Right Case): 좌회전 후 우회전**

불균형 노드의 왼쪽 자식의 오른쪽에 삽입된 경우.

```
불균형 전:           1단계(좌회전):       2단계(우회전):
      C (BF=2)           C (BF=2)              B
     /                  /                     / \
    A (BF=-1)          B                     A   C
     \                /
      B              A
```

**RL 회전 (Right-Left Case): 우회전 후 좌회전**

불균형 노드의 오른쪽 자식의 왼쪽에 삽입된 경우.

```
불균형 전:     1단계(우회전):   2단계(좌회전):
  A (BF=-2)       A (BF=-2)         B
   \               \               / \
    C (BF=1)        B             A   C
   /                 \
  B                   C
```

**균형 인수와 회전 요약:**

| 상황 | 불균형 노드 BF | 자식 BF | 회전 종류 |
|------|--------------|---------|---------|
| 왼쪽-왼쪽 | +2 | +1 (왼쪽) | 우회전(LL) |
| 오른쪽-오른쪽 | -2 | -1 (오른쪽) | 좌회전(RR) |
| 왼쪽-오른쪽 | +2 | -1 (왼쪽) | LR(좌회전→우회전) |
| 오른쪽-왼쪽 | -2 | +1 (오른쪽) | RL(우회전→좌회전) |

AVL 트리는 삽입/삭제 후 최대 O(log n)번의 회전으로 균형을 복구하므로, 탐색·삽입·삭제 모두 O(log n)을 보장한다.

### 5.3 레드블랙 트리와 TreeMap

**레드블랙 트리(Red-Black Tree)**는 각 노드에 색상(빨강/검정)을 부여하는 자가균형 BST이다. Java의 `TreeMap`, `TreeSet`, 그리고 `HashMap`의 버킷 충돌 처리에서도 활용된다.

#### 레드블랙 트리의 5가지 규칙

1. **색상 규칙**: 모든 노드는 빨강(Red) 또는 검정(Black)이다.
2. **루트 규칙**: 루트는 항상 검정이다.
3. **리프 규칙**: 모든 NIL(빈 리프) 노드는 검정이다.
4. **연속 빨강 금지**: 빨간 노드의 자식은 반드시 검정이다 (빨강 노드가 연속으로 올 수 없다).
5. **검정 높이 규칙**: 루트에서 임의의 NIL 리프까지 경로에 있는 검정 노드의 수가 모두 같다.

```
레드블랙 트리 예시 (R=빨강, B=검정):
         13(B)
        /     \
      8(R)    17(R)
     /   \    /   \
   1(B) 11(B) 15(B) 25(B)
         \
         9(R)
```

이 5가지 규칙에 의해 레드블랙 트리의 높이는 최대 **2 × log₂(n+1)**을 넘지 않는다. 따라서 탐색·삽입·삭제 모두 O(log n)을 보장한다.

#### AVL 트리 vs 레드블랙 트리

| 구분 | AVL 트리 | 레드블랙 트리 |
|------|---------|------------|
| 균형 기준 | 엄격 (BF ∈ {-1, 0, 1}) | 느슨 (검정 높이 동일) |
| 탐색 성능 | 약간 더 빠름 (더 균형적) | 약간 느림 |
| 삽입/삭제 회전 수 | 더 많음 | 최대 3번 |
| 활용 | 읽기 작업이 많은 경우 | 쓰기 작업이 많은 경우 |
| Java 사용 | (기본 라이브러리 미사용) | TreeMap, TreeSet |

Java의 `TreeMap`과 `TreeSet`이 레드블랙 트리를 선택한 이유는 삽입·삭제 회전 수가 적어 쓰기 작업이 많은 일반적인 사용 패턴에서 더 효율적이기 때문이다.

**HashMap과 트리**: `HashMap`은 해시 충돌이 없으면 O(1)이지만, 같은 버킷에 8개 이상의 요소가 쌓이면 연결 리스트를 레드블랙 트리로 변환(`treeify`)한다. 이로써 최악의 경우도 O(log n)으로 개선된다.

## 6. 최소 힙(MinHeap)과 PriorityQueue

### 6.1 힙의 정의와 완전이진트리

**최소 힙(Min-Heap)**은 다음 두 성질을 만족하는 완전 이진 트리이다.

1. **힙 성질(Heap Property)**: 모든 부모 노드의 값은 자식 노드의 값보다 작거나 같다.
   ```
   parent.value ≤ child.value  (모든 부모-자식 쌍에 대해)
   ```
2. **완전 이진 트리 성질**: 마지막 레벨을 제외한 모든 레벨이 꽉 차 있고, 마지막 레벨은 왼쪽부터 채워진다.

이 두 성질로부터 중요한 결론이 도출된다: **루트 노드는 항상 전체 최솟값이다.**

```
최소 힙 예시:
         1
        / \
       3   2
      / \ / \
     8  9 5  7

부모 ≤ 자식 관계 확인:
  1 ≤ 3, 1 ≤ 2  ✓
  3 ≤ 8, 3 ≤ 9  ✓
  2 ≤ 5, 2 ≤ 7  ✓
```

힙은 BST와 달리 왼쪽/오른쪽 자식 간의 순서가 정해져 있지 않다. 위 예시에서 3과 2의 위치가 바뀌어도 힙 성질은 유지된다. 이것이 핵심적인 차이점이다.

---

### 6.2 배열 매핑

완전 이진 트리이기 때문에 힙은 **배열 하나**로 표현할 수 있다. 이 표현법은 포인터가 필요 없어 공간 효율적이고 캐시 친화적이다.

**배열 인덱스 규칙 (0-based index)**

```
부모(i)의 왼쪽 자식 = 2i + 1
부모(i)의 오른쪽 자식 = 2i + 2
자식(i)의 부모 = (i - 1) / 2  (정수 나눗셈)
```

**트리와 배열 대응:**

```
트리:
         1          ← 인덱스 0
        / \
       3   2        ← 인덱스 1, 2
      / \ / \
     8  9 5  7      ← 인덱스 3, 4, 5, 6

배열: [ 1, 3, 2, 8, 9, 5, 7 ]
       [0][1][2][3][4][5][6]
```

**수식 유도:**

레벨 k에는 최대 2^k개의 노드가 있다. 레벨 0(루트)부터 레벨 k-1까지의 총 노드 수는 `2^0 + 2^1 + ... + 2^(k-1) = 2^k - 1`이다. 따라서 레벨 k의 첫 번째 노드는 인덱스 `2^k - 1`에 위치한다. 인덱스 i의 왼쪽 자식이 `2i+1`, 오른쪽 자식이 `2i+2`임은 이로부터 귀납적으로 증명된다.

**배열 기반 인덱스 계산 예:**

```
인덱스 1 (값=3)의 자식:
  왼쪽: 2×1+1 = 3 (값=8)  ✓
  오른쪽: 2×1+2 = 4 (값=9)  ✓

인덱스 3 (값=8)의 부모:
  (3-1)/2 = 1 (값=3)  ✓
```

---

### 6.3 삽입: sift-up

힙에 새 값을 삽입하는 절차:

1. 배열의 맨 끝에 새 값을 추가 (완전 이진 트리 성질 유지)
2. 추가된 위치에서 **위로(sift-up)** 올라가며 부모와 비교
3. 부모보다 작으면 교환, 아니면 중지

```
초기 힙:
배열: [ 1, 3, 2, 8, 9, 5, 7 ]

값 4 삽입 (끝에 추가):
배열: [ 1, 3, 2, 8, 9, 5, 7, 4 ]
인덱스                         7

         1
        / \
       3   2
      / \ / \
     8  9 5  7
    /
   4          ← 인덱스 7 추가

sift-up:
  인덱스 7, 값 4
  부모 = (7-1)/2 = 3, 부모값 = 8
  4 < 8 → 교환

배열: [ 1, 3, 2, 4, 9, 5, 7, 8 ]

  인덱스 3, 값 4
  부모 = (3-1)/2 = 1, 부모값 = 3
  4 ≥ 3 → 중지 (힙 성질 만족)

최종:
         1
        / \
       3   2
      / \ / \
     4  9 5  7
    /
   8
```

**Java 구현**

```java
class MinHeap {
    private int[] heap;
    private int size;
    private int capacity;

    MinHeap(int capacity) {
        this.capacity = capacity;
        heap = new int[capacity];
        size = 0;
    }

    public void add(int value) {
        if (size >= capacity) throw new IllegalStateException("힙이 가득 찼습니다");
        heap[size] = value;
        siftUp(size);
        size++;
    }

    private void siftUp(int index) {
        while (index > 0) {
            int parentIndex = (index - 1) / 2;
            if (heap[index] < heap[parentIndex]) {
                swap(index, parentIndex);
                index = parentIndex;
            } else {
                break;
            }
        }
    }

    private void swap(int i, int j) {
        int temp = heap[i];
        heap[i] = heap[j];
        heap[j] = temp;
    }
}
```

### 6.4 삭제: sift-down

최솟값(루트) 추출 절차:

1. 루트(인덱스 0)의 값을 반환용으로 저장
2. 배열의 **마지막 원소를 루트로 이동** (완전 이진 트리 성질 유지)
3. 배열 크기를 1 감소
4. 루트에서 **아래로(sift-down)** 내려가며 자식과 비교
5. 더 작은 자식보다 크면 교환, 아니면 중지

```
초기 힙:
배열: [ 1, 3, 2, 8, 9, 5, 7 ]

루트(1) 제거:
  1을 결과로 저장
  마지막 원소(7)를 루트로 이동

배열: [ 7, 3, 2, 8, 9, 5 ]
size = 6

         7
        / \
       3   2
      / \ /
     8  9 5

sift-down:
  인덱스 0, 값 7
  왼쪽 자식: 인덱스 1, 값 3
  오른쪽 자식: 인덱스 2, 값 2
  더 작은 자식: 인덱스 2, 값 2
  7 > 2 → 교환

배열: [ 2, 3, 7, 8, 9, 5 ]

         2
        / \
       3   7
      / \ /
     8  9 5

  인덱스 2, 값 7
  왼쪽 자식: 인덱스 5, 값 5
  오른쪽 자식: 인덱스 6 → 범위 초과
  더 작은 자식: 인덱스 5, 값 5
  7 > 5 → 교환

배열: [ 2, 3, 5, 8, 9, 7 ]

  인덱스 5 → 자식 없음 (2×5+1=11 ≥ size=6) → 중지

최종:
         2
        / \
       3   5
      / \ /
     8  9 7
```

**Java 구현**

```java
public int peekMin() {
    if (size == 0) throw new NoSuchElementException("힙이 비어있습니다");
    return heap[0];
}

public int pollMin() {
    if (size == 0) throw new NoSuchElementException("힙이 비어있습니다");
    int min = heap[0];
    size--;
    heap[0] = heap[size];  // 마지막 원소를 루트로
    siftDown(0);
    return min;
}

private void siftDown(int index) {
    while (true) {
        int left = 2 * index + 1;
        int right = 2 * index + 2;
        int smallest = index;

        if (left < size && heap[left] < heap[smallest]) {
            smallest = left;
        }
        if (right < size && heap[right] < heap[smallest]) {
            smallest = right;
        }

        if (smallest == index) break;  // 힙 성질 만족

        swap(index, smallest);
        index = smallest;
    }
}
```

### 6.5 build-heap과 O(n) 상환분석

임의 배열이 주어졌을 때 이를 힙으로 변환하는 가장 직관적인 방법은 원소를 하나씩 `add()`로 삽입하는 것이다. 그러나 이 방법은 O(n log n)이다.

더 효율적인 방법은 **build-heap** 알고리즘으로, **O(n)**에 힙을 구성할 수 있다.

**build-heap 알고리즘:**

```
배열의 마지막 내부 노드(= (n/2 - 1))부터 루트(0)까지 역순으로 sift-down을 수행
```

```java
public void buildHeap(int[] arr) {
    heap = arr;
    size = arr.length;
    // 마지막 내부 노드부터 역순으로 sift-down
    for (int i = size / 2 - 1; i >= 0; i--) {
        siftDown(i);
    }
}
```

**왜 O(n)인가? (상환분석 스케치)**

직관적으로 O(n log n)이라고 생각하기 쉽지만, 각 노드에서 sift-down이 내려갈 수 있는 최대 거리는 해당 노드의 **높이**에 비례한다. 리프에 가까울수록 높이가 낮아 sift-down 비용이 적다.

높이 h인 노드의 수 ≤ ⌈n / 2^(h+1)⌉

총 비용 = Σ (높이 h인 노드 수) × h = Σ ⌈n/2^(h+1)⌉ × h

이 합을 계산하면:
```
= n × Σ(h / 2^(h+1)), h=0 to log n
≤ n × Σ(h / 2^h), h=0 to ∞
= n × 2          (무한급수 Σ(h/2^h) = 2)
= O(n)
```

즉 대부분의 노드(리프)는 sift-down 비용이 0 또는 1이고, 비용이 많이 드는 노드(루트 근처)는 극히 소수이다. 이 비대칭성이 전체 비용을 O(n)으로 만든다.

**비교: add() 반복 vs build-heap**

| 방법 | 시간 복잡도 | 설명 |
|------|-----------|------|
| n번 add() | O(n log n) | 각 삽입이 O(log n), n번 반복 |
| build-heap | O(n) | 상환분석으로 O(n) 증명 |

## 7. 힙 정렬

**힙 정렬(Heap Sort)**은 힙 자료구조를 이용한 비교 기반 정렬 알고리즘이다.

### 최소 힙을 이용한 정렬

1. n개의 원소를 MinHeap에 삽입: O(n log n)
2. n번 `pollMin()` 호출하여 오름차순으로 꺼냄: O(n log n)
3. 전체 시간 복잡도: **O(n log n)**

```java
public static int[] heapSort(int[] arr) {
    PriorityQueue<Integer> pq = new PriorityQueue<>();
    for (int val : arr) pq.offer(val);

    int[] sorted = new int[arr.length];
    for (int i = 0; i < arr.length; i++) {
        sorted[i] = pq.poll();
    }
    return sorted;
}
```

### 제자리 힙 정렬 (In-place)

build-heap으로 최대 힙을 만든 후, 루트와 마지막 원소를 교환하고 힙 크기를 줄이는 방식으로 O(1) 추가 공간만 사용한다.

```java
// 최대 힙 기반 제자리 힙 정렬
public static void heapSortInPlace(int[] arr) {
    int n = arr.length;

    // 1단계: 최대 힙 구성 (build-heap)
    for (int i = n / 2 - 1; i >= 0; i--) {
        siftDownMax(arr, n, i);
    }

    // 2단계: 루트(최댓값)를 끝으로 보내며 정렬
    for (int i = n - 1; i > 0; i--) {
        // 최댓값(루트)을 배열 끝으로 이동
        int temp = arr[0]; arr[0] = arr[i]; arr[i] = temp;
        // 힙 크기를 줄이고 sift-down
        siftDownMax(arr, i, 0);
    }
}

private static void siftDownMax(int[] arr, int n, int i) {
    while (true) {
        int largest = i;
        int left = 2 * i + 1;
        int right = 2 * i + 2;
        if (left < n && arr[left] > arr[largest]) largest = left;
        if (right < n && arr[right] > arr[largest]) largest = right;
        if (largest == i) break;
        int temp = arr[i]; arr[i] = arr[largest]; arr[largest] = temp;
        i = largest;
    }
}
```

**힙 정렬 단계별 시각화 (배열 [4, 10, 3, 5, 1]):**

```
초기 배열: [4, 10, 3, 5, 1]

1단계: 최대 힙 구성
  build-heap 후: [10, 5, 3, 4, 1]

          10
         /  \
        5    3
       / \
      4   1

2단계: 정렬
  교환 arr[0]↔arr[4]: [1, 5, 3, 4, | 10]  → sift-down
  교환 arr[0]↔arr[3]: [1, 4, 3, | 5, 10]  → sift-down
  교환 arr[0]↔arr[2]: [1, 3, | 4, 5, 10]  → sift-down (이미 최대 힙)
  교환 arr[0]↔arr[1]: [1, | 3, 4, 5, 10]  → sift-down

최종: [1, 3, 4, 5, 10]  ← 오름차순 정렬 완료
```

## 9. 복잡도 표

### BST 연산 복잡도

|------|-------------------|----------------|
| 탐색 | O(log n) | O(n) |
| 삽입 | O(log n) | O(n) |
| 삭제 | O(log n) | O(n) |
| 중위순회 | O(n) | O(n) |
| 공간 | O(n) | O(n) |

### 자가균형 트리 (AVL, 레드블랙)

| 연산 | 최선 | 평균 | 최악 |
|------|------|------|------|
| 탐색 | O(log n) | O(log n) | O(log n) |
| 삽입 | O(log n) | O(log n) | O(log n) |
| 삭제 | O(log n) | O(log n) | O(log n) |

### 힙 연산 복잡도

| 연산 | 최선 | 평균 | 최악 | 설명 |
|------|------|------|------|------|
| peekMin | O(1) | O(1) | O(1) | 루트를 읽기만 함 |
| add | O(1) | O(log n) | O(log n) | sift-up, 트리 높이에 비례 |
| pollMin | O(log n) | O(log n) | O(log n) | sift-down, 트리 높이에 비례 |
| build-heap | O(n) | O(n) | O(n) | 상환분석으로 O(n) |
| 임의 탐색 | O(n) | O(n) | O(n) | 힙은 전체 순서 보장 안 함 |

### 힙 정렬 복잡도

| 구분 | 시간 | 공간 |
|------|------|------|
| 최선 | O(n log n) | O(1) 제자리 |
| 평균 | O(n log n) | O(1) 제자리 |
| 최악 | O(n log n) | O(1) 제자리 |

힙 정렬은 시간 복잡도가 항상 O(n log n)이고 추가 공간이 O(1)이지만, 캐시 지역성(Cache Locality)이 좋지 않아 실전에서는 퀵 정렬보다 느린 경우가 많다.

## 10. ⚠️ 흔한 오해와 함정

### ⚠️ 오해 1: "BST는 항상 O(log n)이다"

**사실**: BST는 삽입 순서에 따라 퇴화할 수 있으며, 최악의 경우 O(n)이다. O(log n)을 **보장**하는 것은 AVL 트리나 레드블랙 트리 같은 **자가균형 BST**이다. Java의 `TreeMap`과 `TreeSet`이 레드블랙 트리를 사용하는 이유가 바로 이것이다. 단순 BST를 구현하고 "O(log n)"이라고 주장하는 것은 오류이다.

### ⚠️ 오해 2: "힙은 정렬된 배열이다"

**사실**: 힙은 **부분 순서(Partial Order)**만 보장한다. 즉, 부모-자식 관계에서만 순서가 정해지며, 형제 노드 간에는 순서가 없다.

```
최소 힙:
         1
        / \
       3   2    ← 3과 2는 형제. 어느 쪽이 더 작아도 됨.
      / \ / \
     8  9 5  7

배열: [1, 3, 2, 8, 9, 5, 7]
      정렬 배열이었다면: [1, 2, 3, 5, 7, 8, 9] 이어야 함.
```

힙에서 두 번째로 작은 값이 인덱스 1 또는 2에 있다는 것은 보장되지만(부모가 자식보다 작으므로 2번째 최솟값은 루트의 직접 자식 중 하나), 정확히 어느 위치인지는 탐색 없이 알 수 없다.

### ⚠️ 오해 3: "힙에서 특정 값 탐색은 O(log n)이다"

**사실**: BST는 구조적 성질 덕분에 탐색이 O(log n)이지만, 힙은 **부분 순서만 보장**하므로 특정 값을 찾으려면 배열 전체를 순회해야 한다. 즉, 힙에서 임의 값 탐색은 **O(n)**이다.

Java의 `PriorityQueue.contains()`도 O(n)이다. 최솟값을 빠르게 꺼내는 것이 힙의 목적이지, 임의 탐색이 목적이 아니다.

### ⚠️ 오해 4: "build-heap은 n번 삽입이므로 O(n log n)이다"

**사실**: n번 `add()`를 호출하는 방식은 O(n log n)이지만, **build-heap 알고리즘**은 **O(n)**이다. 핵심은 대부분의 노드가 리프에 가까워 sift-down 거리가 짧다는 점이다. 상환분석으로 총 비용이 2n = O(n)임을 증명할 수 있다.

실무에서 n개의 데이터로 힙을 초기 구성할 때는 build-heap을 사용해야 한다.

### ⚠️ 오해 5: "중위순회만이 정렬 순서를 보장한다"

**사실**: 맞다. BST에서 중위순회(In-order Traversal)만이 오름차순 정렬 결과를 제공한다.

- 전위순회(Pre-order): 트리 구조 직렬화
- 중위순회(In-order): BST에서 정렬 순서
- 후위순회(Post-order): 자식 먼저 처리 (삭제, 수식 계산)
- 레벨순회(Level-order): 너비 우선 순서

임의 이진 트리에서 "트리를 순회하면 정렬 결과가 나온다"고 말할 때는 반드시 BST + 중위순회 조합을 명시해야 한다.

### 문제 1 — BST 삭제(후계자) 구현 [난이도: 중]

위에서 설명한 BST 삭제 연산을 직접 구현하라. 특히 **자식이 2개인 노드 삭제 시 중위 후계자(오른쪽 서브트리의 최솟값)**를 사용하는 로직을 포함해야 한다.

다음 시나리오를 테스트하라:
1. 리프 노드 삭제: BST(50, 30, 70, 20, 40, 60, 80)에서 20 삭제
2. 자식 1개인 노드 삭제: 위 트리에서 30 삭제 (40만 남음)
3. 자식 2개인 노드 삭제: BST(50, 30, 70, 20, 40, 60, 80)에서 70 삭제

힌트: `findMin(node.right)`로 후계자를 찾고, 후계자 값으로 현재 노드를 교체한 뒤 후계자를 삭제하라.

<details>
<summary>해설 보기</summary>

```java
private Node deleteRecursive(Node node, int value) {
    if (node == null) return null;

    if (value < node.value) {
        node.left = deleteRecursive(node.left, value);
    } else if (value > node.value) {
        node.right = deleteRecursive(node.right, value);
    } else {
        // 삭제 대상 발견
        if (node.left == null && node.right == null) return null;  // 리프
        if (node.left == null) return node.right;   // 오른쪽 자식만 있음
        if (node.right == null) return node.left;   // 왼쪽 자식만 있음

        // 자식 2개: 중위 후계자 사용
        Node successor = findMin(node.right);
        node.value = successor.value;
        node.right = deleteRecursive(node.right, successor.value);
    }
    return node;
}

private Node findMin(Node node) {
    while (node.left != null) node = node.left;
    return node;
}
```

시나리오 3에서 70을 삭제할 때, 오른쪽 서브트리의 최솟값인 80이 후계자가 된다. 80은 리프 노드이므로 이를 삭제하면 트리는 {50, 30, 80, 20, 40, 60}이 된다.
</details>

### 문제 2 — 트리 높이 및 균형 판정 [난이도: 중]

다음 두 함수를 구현하라:

1. `int height(Node node)`: 트리의 높이를 반환 (리프=0, null=-1)
2. `boolean isBalanced(Node node)`: AVL 조건(모든 노드에서 왼쪽-오른쪽 높이 차이가 1 이하)을 만족하는지 판정

힌트: `height()` 함수를 재귀로 구현하고, `isBalanced()`에서 이를 활용하라. 단, 순진한 구현은 O(n²)이 된다. 이를 O(n)으로 최적화하는 방법도 고민하라.

<details>
<summary>해설 보기</summary>

```java
// O(n²) 순진한 구현
int height(Node node) {
    if (node == null) return -1;
    return 1 + Math.max(height(node.left), height(node.right));
}

boolean isBalanced(Node node) {
    if (node == null) return true;
    int leftH = height(node.left);
    int rightH = height(node.right);
    if (Math.abs(leftH - rightH) > 1) return false;
    return isBalanced(node.left) && isBalanced(node.right);
}

// O(n) 최적화: -1을 "불균형" 신호로 활용
int checkHeight(Node node) {
    if (node == null) return 0;
    int leftH = checkHeight(node.left);
    if (leftH == -1) return -1;  // 이미 불균형
    int rightH = checkHeight(node.right);
    if (rightH == -1) return -1;
    if (Math.abs(leftH - rightH) > 1) return -1;  // 현재 불균형
    return Math.max(leftH, rightH) + 1;
}

boolean isBalancedOptimal(Node root) {
    return checkHeight(root) != -1;
}
```

O(n²) 해법은 각 노드에서 `height()`를 O(n)으로 호출하므로 전체 O(n²). O(n) 해법은 높이를 계산하면서 동시에 균형을 체크하여 한 번의 순회로 완료한다.
</details>

### 문제 3 — 최대 힙으로 변환 [난이도: 하]

주어진 배열을 **최대 힙(Max-Heap)**으로 변환하라. 최대 힙은 부모 ≥ 자식인 힙이다.

입력: `[3, 1, 6, 5, 2, 4]`  
기대 출력 예시: `[6, 5, 4, 1, 2, 3]` (다른 유효한 최대 힙도 정답)

힌트: build-heap 알고리즘에서 비교 방향만 바꾸면 된다. 마지막 내부 노드부터 역순으로 siftDownMax를 호출하라.

<details>
<summary>해설 보기</summary>

```java
void buildMaxHeap(int[] arr) {
    int n = arr.length;
    for (int i = n / 2 - 1; i >= 0; i--) {
        siftDownMax(arr, n, i);
    }
}

void siftDownMax(int[] arr, int n, int i) {
    while (true) {
        int largest = i;
        int left = 2 * i + 1;
        int right = 2 * i + 2;
        if (left < n && arr[left] > arr[largest]) largest = left;
        if (right < n && arr[right] > arr[largest]) largest = right;
        if (largest == i) break;
        int temp = arr[i]; arr[i] = arr[largest]; arr[largest] = temp;
        i = largest;
    }
}
```

`[3, 1, 6, 5, 2, 4]` 변환 과정:
1. n/2-1 = 2, arr[2]=6: 자식 arr[5]=4, arr[6] 없음 → 6 > 4이므로 그대로
2. i=1, arr[1]=1: 자식 arr[3]=5, arr[4]=2 → 최대 5 > 1 → 교환 → [3, 5, 6, 1, 2, 4]
3. i=0, arr[0]=3: 자식 arr[1]=5, arr[2]=6 → 최대 6 > 3 → 교환 → [6, 5, 3, 1, 2, 4]
   - i=2, arr[2]=3: 자식 arr[5]=4 → 4 > 3 → 교환 → [6, 5, 4, 1, 2, 3]

최종: `[6, 5, 4, 1, 2, 3]`
</details>

### 문제 4 — 힙으로 K번째 최솟값 찾기 [난이도: 중]

정렬되지 않은 배열에서 **K번째로 작은 값**을 찾아라. 힙을 이용해 O(n + k log n) 또는 O(n log k)로 구현하라.

입력: `arr = [7, 2, 5, 1, 8, 3]`, `k = 3`  
기대 출력: `3` (1, 2, 3 순서로 3번째)

힌트 1 (최소 힙): build-heap 후 k번 pollMin()  
힌트 2 (최대 힙): 크기 k의 최대 힙 유지. 각 원소를 힙에 추가하고, 크기가 k를 초과하면 최댓값 제거.

<details>
<summary>해설 보기</summary>

**방법 1: 최소 힙, O(n + k log n)**

```java
int kthSmallest_minHeap(int[] arr, int k) {
    PriorityQueue<Integer> pq = new PriorityQueue<>();
    for (int val : arr) pq.offer(val);  // build: O(n)

    int result = -1;
    for (int i = 0; i < k; i++) {
        result = pq.poll();  // 각 poll: O(log n), k번
    }
    return result;
}
```

**방법 2: 최대 힙 (크기 k 유지), O(n log k)**

```java
int kthSmallest_maxHeap(int[] arr, int k) {
    // 최대 힙 (Java PriorityQueue는 최소 힙이므로 역순 비교자 사용)
    PriorityQueue<Integer> pq = new PriorityQueue<>(Collections.reverseOrder());

    for (int val : arr) {
        pq.offer(val);
        if (pq.size() > k) pq.poll();  // 크기 k 초과 시 최댓값 제거
    }
    return pq.peek();  // 크기 k인 최대 힙의 루트 = k번째 최솟값
}
```

방법 2는 힙 크기를 k로 유지하므로 메모리 사용량이 O(k)이고, 각 연산이 O(log k)이다. n이 매우 크고 k가 작을 때 유리하다.

`arr = [7, 2, 5, 1, 8, 3]`, k=3:
- 삽입 과정: [7], [7,2], [7,5,2]→크기3 OK, [7,5,2]+1=4→최대7제거→[5,2,1], [8,5,2,1]→최대8제거→[5,2,1], [5,3,2,1]→최대5제거→[3,2,1]
- 결과: pq.peek() = 3 ✓
</details>

### 문제 5 — build-heap O(n) 구현 및 검증 [난이도: 중상]

임의 배열을 입력받아 build-heap으로 최소 힙을 구성한 뒤, 결과가 올바른 힙인지 검증하는 함수를 작성하라.

입력: `[5, 3, 8, 1, 2, 7, 4]`

검증 함수 `boolean isValidMinHeap(int[] arr)`: 모든 인덱스 i에 대해 자식(2i+1, 2i+2)이 존재하면 `arr[i] ≤ arr[child]`임을 확인.

또한, n번 `add()` 반복과 build-heap의 실행 시간을 n=100, 1000, 10000으로 측정해 비교하라.

힌트: `System.nanoTime()`으로 시간 측정.

<details>
<summary>해설 보기</summary>

```java
void buildMinHeap(int[] arr) {
    int n = arr.length;
    for (int i = n / 2 - 1; i >= 0; i--) {
        siftDown(arr, n, i);
    }
}

void siftDown(int[] arr, int n, int i) {
    while (true) {
        int smallest = i;
        int left = 2 * i + 1, right = 2 * i + 2;
        if (left < n && arr[left] < arr[smallest]) smallest = left;
        if (right < n && arr[right] < arr[smallest]) smallest = right;
        if (smallest == i) break;
        int tmp = arr[i]; arr[i] = arr[smallest]; arr[smallest] = tmp;
        i = smallest;
    }
}

boolean isValidMinHeap(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n / 2; i++) {  // 내부 노드만 검사
        int left = 2 * i + 1, right = 2 * i + 2;
        if (left < n && arr[i] > arr[left]) return false;
        if (right < n && arr[i] > arr[right]) return false;
    }
    return true;
}

// 시간 측정 예시 (n=10000)
// build-heap:   보통 수십 µs
// n번 add():   보통 수백 µs ~ 수 ms
// 차이가 커질수록 n이 커짐에 따라 더욱 뚜렷해짐
```

`[5, 3, 8, 1, 2, 7, 4]` build-heap 결과: `[1, 2, 4, 3, 5, 7, 8]` (유효한 최소 힙 중 하나)
- 검증: 0→(1,2), 1→(3,4), 2→(5,6): 1≤2,1≤4, 2≤3,2≤5, 4≤7,4≤8 → 모두 통과 ✓
</details>

## 12. 요약

이 장에서 학습한 핵심 내용을 정리한다.

### 이진 탐색 트리(BST)

- **BST 성질**: 왼쪽 < 부모 < 오른쪽. 이 성질로 탐색 시 매 단계 탐색 공간이 절반으로 줄어든다.
- **탐색/삽입**: 트리 높이에 비례. 균형 트리에서 O(log n), 퇴화 트리에서 O(n).
- **삭제 3경우**: 리프 → 단순 제거 / 자식 1개 → 자식으로 대체 / 자식 2개 → 중위 후계자로 교체 후 삭제.
- **중위순회**: BST에서 오름차순 정렬 결과를 출력하는 유일한 순회 방법.

### 트리 순회 4종

전위(현재→왼→오), 중위(왼→현재→오), 후위(왼→오→현재), 레벨(BFS). 각각 다른 목적에 활용.

### 자가균형 트리

- **BST 퇴화**: 정렬된 순서로 삽입하면 연결 리스트 구조로 퇴화. 높이 = n.
- **AVL 트리**: 균형 인수(BF) ∈ {-1, 0, 1} 유지. LL/RR/LR/RL 회전으로 복구.
- **레드블랙 트리**: 5가지 색상 규칙으로 높이 ≤ 2log₂(n+1) 보장. Java `TreeMap`, `TreeSet`의 내부 구현.

### 최소 힙(MinHeap)

- **성질**: 부모 ≤ 자식 + 완전 이진 트리. 루트 = 최솟값.
- **배열 매핑**: 인덱스 i의 왼쪽 자식 = 2i+1, 오른쪽 자식 = 2i+2, 부모 = (i-1)/2.
- **sift-up (삽입)**: 끝에 추가 후 부모보다 작으면 올라감. O(log n).
- **sift-down (삭제)**: 루트 제거 → 끝 원소를 루트로 → 더 작은 자식과 교환하며 내려감. O(log n).
- **build-heap**: 상환분석으로 O(n). n번 add()의 O(n log n)과 다름.

### 힙 정렬

- build-heap(O(n)) + n번 poll(O(n log n)) = O(n log n). 추가 공간 O(1).

### Java 연결

- `TreeMap`, `TreeSet` → 레드블랙 트리 → O(log n) 보장
- `PriorityQueue` → 이진 최소 힙 → peekMin O(1), add/poll O(log n)
- `HashMap` 버킷 충돌 ≥ 8 → 레드블랙 트리(`treeify`)
