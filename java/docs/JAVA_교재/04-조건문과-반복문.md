프로그램은 기본적으로 위에서 아래로 한 줄씩 순서대로 실행된다. 하지만 실제 프로그램은 "점수가 60점 이상이면 합격", "10개의 데이터를 하나씩 처리", "원하는 값을 찾으면 멈춤"처럼 **상황에 따라 실행 순서를 바꾸어야** 한다. 이렇게 실행 순서를 조절하는 문법을 **제어문(control statement)**이라고 한다.

제어문은 크게 두 종류로 나뉜다.

- **조건문(selection)** : 조건에 따라 실행할 코드를 고른다. → `if`, `switch`
- **반복문(iteration)** : 같은 코드를 여러 번 실행한다. → `for`, `while`, `do-while`

그리고 반복문 안에서 흐름을 세밀하게 조절하는 **분기문(branch)**으로 `break`(즉시 탈출)와 `continue`(이번 회차 건너뛰기)가 있다.

이번 장에서는 이 문법들을 하나씩 살펴본다. 각 소절에는 실행 가능한 실습 코드가 연결되어 있으니, 개념을 읽은 뒤 직접 실행해 결과를 눈으로 확인하며 익히자.

## 4.1 실행 흐름 제어

코드의 실행 순서를 다루는 방식을 **제어 흐름(control flow)**이라고 한다. 제어 흐름은 세 가지 기본 형태로 이루어진다.

1. **순차(sequence)** : 특별한 제어문이 없으면 작성한 순서대로 위에서 아래로 한 문장씩 실행된다.
2. **조건(selection)** : 조건이 참일 때만 특정 코드를 실행한다. 갈림길에서 하나의 길을 고르는 것과 같다.
3. **반복(iteration)** : 같은 코드를 정해진 횟수나 조건만큼 되풀이한다.

우리가 작성하는 거의 모든 프로그램은 이 세 가지의 조합으로 만들어진다. 아래 코드는 세 가지 흐름이 한 프로그램 안에서 어떻게 어우러지는지 보여준다.

```java
// [1] 순차: 위에서 아래로
System.out.println("첫 번째 문장");
System.out.println("두 번째 문장");

// [2] 조건: 참일 때만 실행
int temperature = 30;
if (temperature >= 28) {
    System.out.println("덥습니다.");
}

// [3] 반복: 같은 작업을 되풀이
for (int i = 1; i <= 3; i++) {
    System.out.println("반복 " + i + "회차");
}
```

## 4.2 if 문

`if` 문은 **조건이 참(true)일 때만** 코드 블록을 실행한다. 조건식의 결과는 반드시 `boolean`(참/거짓)이어야 한다. 형태는 목적에 따라 네 가지로 나뉜다.

- **단순 if** : 조건이 참이면 실행, 거짓이면 아무것도 하지 않는다.
- **if-else** : 참이면 앞 블록, 거짓이면 `else` 블록을 실행한다. (두 갈래)
- **else if 사슬** : 여러 조건을 위에서부터 순서대로 검사해, **가장 먼저 참이 되는** 블록 하나만 실행한다.
- **중첩 if** : `if` 안에 또 `if`를 넣어 조건을 단계적으로 확인한다.

```java
int point = 82;
if (point >= 90) {
    System.out.println("등급: A");
} else if (point >= 80) {
    System.out.println("등급: B");   // 82 → 여기서 걸려 실행됨
} else if (point >= 70) {
    System.out.println("등급: C");
} else {
    System.out.println("등급: F");
}
```

`else if` 사슬은 위에서부터 검사하다가 처음으로 참이 되는 곳에서 멈춘다. 그래서 조건의 **순서가 중요**하다. 예컨대 위 코드에서 `>= 80`을 `>= 90`보다 위에 두면, 95점도 B로 판정되어 버린다.

## 4.3 switch 문

`switch` 문은 **하나의 값을 여러 case와 비교**하여, 일치하는 곳부터 실행한다. "이 값이 A인가? B인가? C인가?"처럼 **같은지(==)만** 따질 때, 긴 `else if` 사슬보다 훨씬 읽기 쉽다.

**전통 switch**는 각 `case` 끝에 `break`를 붙여 흐름을 끊는다. 만약 `break`를 생략하면 다음 `case`로 그대로 흘러 내려가는데, 이를 **fall-through**라고 한다. 이 성질을 일부러 이용하면 여러 `case`를 하나로 묶을 수 있다.

```java
int month = 4;
switch (month) {
    case 3:
    case 4:
    case 5:
        System.out.println("봄");   // 3,4,5월이 하나로 묶임
        break;
    default:
        System.out.println("그 외");
}
```

**switch 표현식(JDK 14+)**은 `case 값 -> 결과;` 형태로 값을 바로 돌려줄 수 있다. `break`가 필요 없고 fall-through도 일어나지 않아 실수가 줄어든다. 결과를 계산하는 데 여러 문장이 필요하면 `{ ... yield 값; }` 블록을 쓴다.

```java
int grade = 2;
String result = switch (grade) {
    case 1 -> "1학년";
    case 2 -> "2학년";   // grade=2 → 이 값이 result에 들어감
    default -> "학년 미상";
};
```

## 4.4 for 문

`for` 문은 **반복 횟수가 정해져 있을 때** 가장 흔히 쓰는 반복문이다. 구성 요소는 세 가지다.

```java
for (초기화; 조건; 증감) { 반복할 코드 }
```

- **초기화** : 반복 시작 전 딱 한 번만 실행된다. 보통 카운터 변수를 선언한다.
- **조건** : 매 반복 전에 검사한다. 참이면 본문을 실행하고, 거짓이 되면 반복을 끝낸다.
- **증감** : 본문을 실행한 뒤 매번 실행된다. 보통 카운터를 1씩 바꾼다.

```java
for (int i = 1; i <= 5; i++) {
    System.out.println("i = " + i);   // 1,2,3,4,5 출력
}
```

배열이나 컬렉션의 요소를 **처음부터 끝까지 하나씩** 꺼낼 때는 **향상된 for(for-each)**가 더 간결하다. 인덱스 변수 없이 요소를 바로 다룰 수 있다.

```java
String[] fruits = {"사과", "바나나", "체리"};
for (String fruit : fruits) {
    System.out.println("과일: " + fruit);
}
```

`for` 안에 `for`를 넣는 **중첩 for**는 구구단이나 표(2차원) 같은 반복에 쓰인다. 참고로 `for (;;)`처럼 조건을 비우면 무한 반복이 되므로 반드시 `break`로 탈출해야 한다.

> 💻 실습: `./run.sh ForStatement`  📁 `chapter-java-textbook/src/main/java/com/edu/javabook/ch04/ForStatement.java`

## 4.5 while 문

`while` 문은 **조건이 참인 동안** 코드를 반복한다. `for`와 달리 반복 횟수를 미리 정하지 않고, **어떤 조건이 만족될 때까지** 돌릴 때 적합하다.

```java
while (조건) { 반복할 코드 }
```

핵심은 두 가지다. 첫째, **조건을 먼저 검사**하므로 처음부터 거짓이면 본문이 한 번도 실행되지 않는다. 둘째, 반복 안에서 **언젠가 조건을 거짓으로 만들어 주어야** 한다. 그러지 않으면 영원히 도는 **무한 루프**에 빠진다.

```java
int i = 1;
while (i <= 5) {
    System.out.println("i = " + i);
    i++;   // 이 증가가 없으면 무한 루프! 조건을 거짓으로 만드는 역할
}
```

`i++`를 빠뜨리면 `i`는 계속 1에 머물러 조건이 영원히 참이 된다. `while`을 쓸 때는 항상 "이 조건은 언제 거짓이 되는가?"를 스스로 물어야 한다. 반복 종료를 `boolean` 플래그 변수로 제어하는 방식도 자주 쓰인다.

## 4.6 do-while 문

`do-while` 문은 `while`과 비슷하지만 **실행 순서가 반대**다. 본문을 **먼저 한 번 실행한 뒤** 조건을 검사한다. 그래서 조건이 처음부터 거짓이어도 **최소 한 번은 반드시** 실행된다.

```java
do { 반복할 코드 } while (조건);   // 끝에 세미콜론(;) 필수
```

| 구분 | 순서 | 최소 실행 횟수 |
|------|------|----------------|
| `while` | 조건 검사 → 실행 | 0번 |
| `do-while` | 실행 → 조건 검사 | 1번 |

```java
int b = 10;
do {
    System.out.println("최소 1회 실행됨 b=" + b);   // 조건이 거짓이어도 1번 출력
    b++;
} while (b < 5);
```

"메뉴를 보여주고 입력을 받는" 것처럼 **일단 한 번은 실행한 뒤** 계속할지 판단하는 상황에 잘 맞는다. 문법 끝에 붙는 세미콜론(`;`)을 빠뜨리기 쉬우니 주의하자.

> 💻 실습: `./run.sh DoWhileStatement`  📁 `chapter-java-textbook/src/main/java/com/edu/javabook/ch04/DoWhileStatement.java`

## 4.7 break 문

`break` 문은 반복문(또는 `switch`)을 **즉시 빠져나온다**. 원하는 조건을 만족하는 순간 더 이상 반복할 필요가 없을 때 사용한다. 예를 들어 특정 값을 찾으면 나머지는 확인할 필요 없이 멈추면 된다.

기본 `break`는 자신을 감싸는 **가장 안쪽 반복문 하나만** 탈출한다. 그래서 중첩 반복에서 안쪽 `for`에 `break`를 쓰면 바깥 `for`는 계속 돈다.

```java
for (int i = 1; i <= 10; i++) {
    if (i == 5) {
        System.out.println("5 발견 → 중단");
        break;   // for를 즉시 탈출 (6~10은 실행 안 됨)
    }
    System.out.println("i = " + i);
}
```

중첩 반복을 **한 번에 모두** 탈출하고 싶다면 **레이블 break**를 쓴다. 바깥 반복문에 `이름:` 형태로 이름을 붙인 뒤 `break 이름;`으로 지정한다.

```java
outer:
for (int row = 1; row <= 3; row++) {
    for (int col = 1; col <= 3; col++) {
        if (row == 2 && col == 2) {
            break outer;   // 'outer'가 붙은 바깥 for 전체를 탈출
        }
    }
}
```

## 4.8 continue 문

`continue` 문은 현재 반복의 **남은 부분을 건너뛰고 다음 회차로** 넘어간다. `break`가 반복을 완전히 끝낸다면, `continue`는 **이번 회차만** 건너뛸 뿐 반복 자체는 계속된다. "짝수는 건너뛰고 홀수만 처리" 같은 필터링에 유용하다.

```java
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) {
        continue;   // 짝수면 아래를 건너뛰고 다음 i로
    }
    System.out.println("홀수 i = " + i);
}
```

`continue`가 다음으로 이동하는 위치는 반복문마다 다르다. `for`에서는 **증감식**으로, `while`에서는 **조건 검사**로 되돌아간다. 이 차이 때문에 `while`에서는 주의가 필요하다 — 증감을 `continue` 뒤에 두면 그 문장을 영영 못 만나 무한 루프에 빠질 수 있다.

중첩 반복에서 바깥 반복의 다음 회차로 바로 점프하고 싶다면 `break`처럼 **레이블 continue**(`continue 이름;`)를 쓴다.

## ⚠️ 흔한 오해와 함정

- **`=`와 `==`를 헷갈린다.** `if (x = 5)`는 대입이고 `if (x == 5)`가 비교다. 자바에서는 `boolean`이 아닌 대입식을 조건에 쓰면 컴파일 오류가 나 어느 정도 막아 주지만, `boolean` 변수에서는 `if (flag = true)`가 실수로 통과할 수 있으니 주의하자.
- **switch에서 break를 빠뜨린다.** 전통 `switch`에서 `break`를 잊으면 아래 `case`로 계속 흘러(fall-through) 의도치 않은 코드까지 실행된다. fall-through를 원하는 게 아니라면 각 `case`에 `break`를 꼭 붙이자. (화살표 `->` 표현식에는 이 문제가 없다.)
- **`while`/`for` 조건을 거짓으로 만드는 것을 잊어 무한 루프에 빠진다.** 반복 안에서 카운터 증감이나 플래그 변경을 반드시 넣어야 한다. `while`에서 `continue`를 쓸 때는 증감식을 `continue` **앞**에 두는지 특히 확인하자.
- **중괄호 `{}`를 생략한 뒤 실수한다.** `if (조건) 문장;`처럼 중괄호를 생략하면 **바로 다음 한 문장만** 조건에 속한다. 나중에 문장을 한 줄 더 추가하면 그 줄은 조건과 무관하게 항상 실행되어 버린다. 가독성과 안전을 위해 항상 `{}`를 쓰자.
- **`break`와 `continue`를 혼동한다.** `break`는 반복을 **완전히 끝내고**, `continue`는 **이번 회차만 건너뛴다.** 목적에 맞는 것을 골라야 한다.
- **기본 `break`가 모든 반복을 빠져나온다고 착각한다.** 중첩 반복에서 기본 `break`는 **가장 안쪽 하나만** 탈출한다. 바깥까지 나가려면 레이블을 써야 한다.

## ✍️ 직접 작성해보기

지금까지는 완성된 코드를 **읽고 실행**하며 개념을 익혔다. 이제는 직접 **키보드로 타이핑**해 볼 차례다. 코드는 눈으로 볼 때와 손으로 칠 때 남는 것이 완전히 다르다. 아래 과제를 순서대로 풀어 보자. 쉬운 것에서 시작해 점점 응용으로 나아간다.

**푸는 방법**

1. 각 과제의 **스펙(무엇을 만들지)**과 **힌트**를 읽는다.
2. 여러분만의 클래스를 직접 작성하고 실행해 결과가 스펙과 맞는지 확인한다.
3. 막히거나 다 풀었다면, 각 과제 끝의 `> 답안 비교` 명령으로 **모범 답안 코드를 실행**해 내 결과와 비교해 본다. 답안 클래스는 이 장의 실습 코드와 같은 위치에 있다.

> 팁: 먼저 종이나 주석으로 "어떤 조건일 때 무엇을 할지"를 말로 적어 본 뒤 코드로 옮기면 훨씬 수월하다. 정답을 바로 열어 보지 말고, 최소 한 번은 스스로 작성해 실행까지 해 보자.

### 과제 1. 학점 판별기 (조건문)

### 과제 7. 입력 검증 반복 (do-while)

"1부터 5 사이의 숫자만 유효하다"고 가정하고, **일단 한 번은 값을 확인한 뒤** 유효 범위를 벗어나면 계속 다음 값을 시도하는 흐름을 `do-while`로 작성하라. (실제 사용자 입력 대신, 시도할 값들을 배열이나 카운터로 흉내 내도 좋다.) 유효한 값을 만나면 `유효한 값: N`을 출력하고 반복을 끝낸다.

- 힌트: `do { ... } while (조건);` 은 **본문을 먼저 한 번 실행**하고 조건을 검사하므로, "일단 값을 받아 보고 유효한지 판단"하는 상황에 잘 맞는다.
- 힌트: 문법 끝의 세미콜론(`;`)을 빠뜨리지 말자. 무한 루프를 막기 위해 시도 횟수 **상한**(예: 최대 10회)을 함께 두는 것도 좋은 습관이다.
- 응용: `while`로 같은 로직을 짜 보고, 첫 값이 처음부터 유효할 때와 유효하지 않을 때 두 방식의 동작 차이를 관찰해 보자.

### 과제 8. 최대공약수(GCD) 구하기 (반복 + 나머지)

두 정수(예: 48과 36)를 변수에 담아 두고, 두 수의 **최대공약수**를 반복문으로 구해 출력하는 프로그램을 작성하라. 48과 36의 최대공약수는 `12`이다.

- 힌트(단순 방법): 1부터 두 수 중 작은 값까지 반복하며 **둘 다 나누어떨어지는(`a % i == 0 && b % i == 0`) 가장 큰 수**를 기록한다.
- 힌트(유클리드 호제법, 권장): 큰 수를 작은 수로 나눈 **나머지**로 계속 바꿔 나간다. `while (b != 0) { int t = a % b; a = b; b = t; }` 를 마치면 `a`가 최대공약수다.
- 응용: 최소공배수는 `두 수의 곱 / 최대공약수`로 구할 수 있다. 이어서 계산해 출력해 보자.

정수 높이 `h`(예: 5)를 변수에 담아 두고, 아래처럼 **왼쪽 정렬 직각삼각형**을 별(`*`)로 출력하는 프로그램을 작성하라. `h`가 5이면 다음과 같다.

### 과제 10. 곱셈표에서 목표 값 찾기 (레이블 break)

1단부터 9단까지의 곱셈표를 중첩 `for`로 훑으면서, **곱이 처음으로 어떤 목표 값 이상이 되는 (단, 곱하는 수)** 조합을 찾으면 그 조합을 출력하고 **중첩 반복 전체를 즉시 종료**하는 프로그램을 작성하라. (예: 목표 값 50이면 `7 x 8 = 56`처럼 처음 50 이상이 되는 조합에서 멈춘다.)

## 확인문제

> 개념 서술, 출력 예측, 오류 찾기, 빈칸 채우기, 참·거짓 등 여러 유형을 섞어 두었다. 먼저 스스로 풀어 본 뒤 `정답 및 해설`을 펼쳐 확인하자. 특히 출력 예측 문제는 **눈으로만 읽지 말고** 실제 실행 순서를 한 줄씩 따라가며 손으로 적어 보는 것이 좋다.

### 개념 서술

**1.** 제어 흐름의 세 가지 기본 형태(순차·조건·반복)를 설명하고, 각각에 해당하는 자바 문법을 하나씩 예로 드시오.

<details>
<summary>정답 및 해설</summary>

- **순차(sequence)**: 특별한 제어문이 없으면 작성한 순서대로 위에서 아래로 한 문장씩 실행된다. (예: 연속된 `System.out.println(...)`)
- **조건(selection)**: 조건이 참일 때만 특정 코드를 실행한다. (예: `if`, `switch`)
- **반복(iteration)**: 같은 코드를 정해진 횟수나 조건만큼 되풀이한다. (예: `for`, `while`, `do-while`)

거의 모든 프로그램은 이 세 형태의 조합으로 만들어진다.
</details>

**2.** 전통 `switch`에서 각 `case`에 `break`를 붙이지 않으면 어떤 현상이 일어나는가? 이 현상을 부르는 이름과, 이를 일부러 활용하는 경우를 설명하시오.

<details>
<summary>정답 및 해설</summary>

일치한 `case`부터 아래 `case`로 **break를 만나거나 switch가 끝날 때까지 계속 흘러 내려가며 실행**된다. 이 현상을 **fall-through**라고 한다. 일부러 활용하는 예는 `case 3: case 4: case 5:`처럼 **여러 case를 하나의 처리로 묶을 때**이다(3·4·5월을 모두 "봄"으로). 화살표(`->`) switch 표현식에는 fall-through가 없다.
</details>

**3.** `while` 문과 `do-while` 문의 **가장 큰 차이**는 무엇인가? 조건이 처음부터 거짓일 때 각각 본문이 몇 번 실행되는지 함께 답하시오.

<details>
<summary>정답 및 해설</summary>

- `while`: **조건을 먼저 검사**한 뒤 실행한다 → 조건이 처음부터 거짓이면 본문은 **0번** 실행.
- `do-while`: **본문을 먼저 실행**한 뒤 조건을 검사한다 → 조건이 처음부터 거짓이어도 본문은 **최소 1번** 실행.

"메뉴를 보여주고 입력받기"처럼 일단 한 번은 실행해야 하는 상황에 `do-while`이 적합하다.
</details>

**4.** `break`와 `continue`의 차이를 설명하고, 중첩 반복에서 **바깥 반복까지 한 번에** 빠져나오려면 어떤 문법을 써야 하는지 답하시오. 또한 `for`와 `while`에서 `continue`가 각각 어디로 이동하는지도 함께 설명하시오.

<details>
<summary>정답 및 해설</summary>

- `break`: 반복을 **완전히 끝내고** 빠져나온다. 기본형은 **가장 안쪽 반복문 하나만** 탈출한다.
- `continue`: **이번 회차의 남은 부분만 건너뛰고** 다음 회차로 넘어간다. 반복 자체는 계속된다.
- 바깥 반복까지 한 번에 탈출하려면 **레이블 break**(`break 이름;`)를 사용한다(바깥 반복문에 `이름:`을 붙임).
- `continue`의 이동 위치: `for`에서는 **증감식**으로, `while`에서는 **조건 검사**로 되돌아간다. 그래서 `while`에서 증감식을 `continue` 뒤에 두면 그 문장을 영영 못 만나 **무한 루프**에 빠질 수 있다.
</details>

**5.** 중첩 반복(nested loop)이 실제로 유용하게 쓰이는 상황을 두 가지 이상 예로 들고, 안쪽 반복 본문이 총 몇 번 실행되는지 계산하는 방법을 설명하시오.

<details>
<summary>정답 및 해설</summary>

예: 구구단 전체 출력(단 × 곱하는 수), 2차원 표/행렬 처리, 별 찍기 같은 도형 출력, 모든 쌍(pair) 비교 등. 바깥 반복이 M번, 안쪽 반복이 매 회차마다 N번 돈다면 안쪽 본문은 총 **M × N번** 실행된다(안쪽 횟수가 바깥 변수에 따라 달라지면 각 회차의 합으로 계산). 이 때문에 중첩이 깊어질수록 실행 횟수가 급격히 늘어난다.
</details>

### 출력 예측

**6.** 다음 `else if` 사슬에서 `point`가 95일 때 출력되는 값은 무엇인가? 그리고 조건의 **순서**가 왜 중요한지 설명하시오.
```java
int point = 95;
if (point >= 80) { System.out.println("B"); }
else if (point >= 90) { System.out.println("A"); }
else { System.out.println("F"); }
```

<details>
<summary>정답 및 해설</summary>

출력: **`B`**

`else if` 사슬은 위에서부터 검사하다 **처음으로 참이 되는 블록 하나만** 실행하고 멈춘다. 95는 첫 조건 `>= 80`에서 이미 참이 되어 `B`가 출력되고, `>= 90`은 검사조차 되지 않는다. 따라서 등급 판별은 반드시 **높은 점수(더 좁은 조건)부터** 검사해야 한다. `>= 90`을 위로 올려야 95가 A로 올바르게 판정된다.
</details>

**7.** 다음 코드의 출력을 예측하시오. (switch fall-through 함정)
```java
int n = 2;
switch (n) {
    case 1:
        System.out.println("하나");
    case 2:
        System.out.println("둘");
    case 3:
        System.out.println("셋");
        break;
    case 4:
        System.out.println("넷");
    default:
        System.out.println("기타");
}
```

<details>
<summary>정답 및 해설</summary>

```
둘
셋
```

`n`이 2이므로 `case 2`부터 시작한다. `case 2`에 `break`가 없으므로 **fall-through**가 일어나 `case 3`까지 흘러 내려가 `둘`, `셋`을 출력하고, `case 3`의 `break`에서 멈춘다. `case 1`은 진입 자체를 하지 않았고, `case 4`·`default`는 `break`로 이미 빠져나온 뒤라 실행되지 않는다.
</details>

**8.** 다음 중첩 반복과 `break`가 섞인 코드의 출력을 예측하시오. (기본 `break`는 안쪽만 탈출)
```java
for (int row = 1; row <= 3; row++) {
    for (int col = 1; col <= 3; col++) {
        if (col == 2) {
            break;
        }
        System.out.println("row=" + row + ", col=" + col);
    }
}
```

<details>
<summary>정답 및 해설</summary>

```
row=1, col=1
row=2, col=1
row=3, col=1
```

`col == 2`가 되는 순간 `break`가 실행되지만, 기본 `break`는 **가장 안쪽 for(col)만** 탈출한다. 그래서 바깥 for(row)는 계속 돌고, 매 row마다 `col=1`만 출력된 뒤 `col=2`에서 안쪽 반복이 끊긴다. 만약 `break outer;`(레이블 break)였다면 `row=1, col=1` 한 줄만 출력되고 전체가 종료된다.
</details>

**9.** 다음 코드의 출력을 예측하시오. (do-while은 최소 1회 실행)
```java
int b = 10;
do {
    System.out.println("b = " + b);
    b++;
} while (b < 5);
System.out.println("종료 b = " + b);
```

<details>
<summary>정답 및 해설</summary>

```
b = 10
종료 b = 11
```

`do-while`은 **본문을 먼저 실행한 뒤** 조건을 검사한다. 처음 진입 시 `b`가 10이라 `b = 10`을 출력하고 `b++`로 11이 된 뒤, 조건 `b < 5`(11 < 5)가 거짓이므로 반복이 끝난다. 즉 조건이 처음부터 거짓이어도 본문은 **정확히 한 번** 실행된다. (같은 코드를 `while`로 썼다면 본문은 한 번도 실행되지 않는다.)
</details>

**10.** 다음 코드의 출력을 예측하시오. (continue가 건너뛰는 위치)
```java
for (int i = 1; i <= 6; i++) {
    if (i % 3 == 0) {
        continue;
    }
    System.out.print(i + " ");
}
```

<details>
<summary>정답 및 해설</summary>

```
1 2 4 5 
```

`i`가 3의 배수(3, 6)일 때 `continue`가 실행되어 아래 `System.out.print`를 건너뛰고 바로 증감식으로 넘어간다. 그래서 3과 6은 출력되지 않고 `1 2 4 5`만 출력된다.
</details>

### 오류 찾기

**11.** 다음 코드에는 **무한 루프** 버그가 있다. 원인을 지적하고 고치시오.
```java
int i = 1;
int sum = 0;
while (i <= 10) {
    if (i % 2 == 0) {
        continue;
    }
    sum += i;
    i++;
}
System.out.println(sum);
```

<details>
<summary>정답 및 해설</summary>

**원인**: `i`가 짝수(예: 2)가 되면 `continue`가 실행되는데, 이때 `i++`가 `continue` **뒤**에 있어 실행되지 못한다. 결국 `i`는 2에 머물러 계속 짝수 → `continue`만 반복하는 **무한 루프**에 빠진다.

**핵심**: `for`와 달리 `while`에서 `continue`는 **조건 검사로 되돌아갈 뿐 증감을 자동으로 해 주지 않는다.**

**수정**(증감을 `continue`보다 앞에 두거나 홀수만 걸러 더한다):
```java
int i = 1, sum = 0;
while (i <= 10) {
    if (i % 2 != 0) {   // 홀수만 더함
        sum += i;
    }
    i++;                // continue와 무관하게 항상 실행됨
}
System.out.println(sum);   // 25
```
</details>

**12.** 다음 코드는 배열의 모든 원소를 출력하려는 의도였으나 실행하면 예외(`ArrayIndexOutOfBoundsException`)가 발생한다. **off-by-one** 오류를 찾아 고치시오.
```java
int[] arr = {10, 20, 30, 40, 50};
for (int i = 0; i <= arr.length; i++) {
    System.out.println(arr[i]);
}
```

<details>
<summary>정답 및 해설</summary>

**원인**: 반복 조건이 `i <= arr.length`이다. 배열 인덱스는 `0`부터 `arr.length - 1`(=4)까지인데, `i`가 `arr.length`(=5)일 때도 조건이 참이 되어 존재하지 않는 `arr[5]`에 접근 → 예외가 발생한다. 경계를 1 넘어선 전형적인 **off-by-one** 실수다.

**수정**: 조건을 `<=`에서 `<`로 바꾼다.
```java
for (int i = 0; i < arr.length; i++) {   // i는 0..4
    System.out.println(arr[i]);
}
```
</details>

### 빈칸 채우기

**13.** 다음은 1부터 N까지의 합을 구하는 코드다. 빈칸 `(가)`, `(나)`, `(다)`를 채워 반복문을 완성하시오. (N=100일 때 결과는 5050)
```java
int N = 100;
int sum = 0;
int i = 1;
while ( (가) ) {
    sum (나) i;
    (다) ;
}
System.out.println(sum);
```

<details>
<summary>정답 및 해설</summary>

- `(가)`: `i <= N`  (반복을 계속할 조건)
- `(나)`: `+=`      (`sum = sum + i`와 같음)
- `(다)`: `i++`     (카운터 증가 — 없으면 무한 루프)

`(다)`의 증감이 조건 `(가)`를 언젠가 거짓으로 만들어 주는 역할을 한다. 셋 중 하나라도 빠지면 결과가 틀리거나 무한 루프에 빠진다.
</details>

**14.** 다음은 `2` 이상의 정수 `num`이 소수인지 판별하는 코드다. 빈칸 `(가)`, `(나)`를 채워 완성하시오.
```java
int num = 13;
boolean isPrime = true;
for (int i = 2; i < num; i++) {
    if ( (가) ) {
        isPrime = false;
        (나) ;          // 약수를 찾았으니 더 볼 필요 없음
    }
}
System.out.println(isPrime ? "소수" : "소수 아님");
```

<details>
<summary>정답 및 해설</summary>

- `(가)`: `num % i == 0`  (i가 num의 약수인지 검사)
- `(나)`: `break`         (약수를 하나라도 찾으면 즉시 반복 종료)

하나라도 나누어떨어지면 소수가 아님이 확정되므로 `break`로 곧장 빠져나오면 된다(끝까지 돌 필요 없음). 참고로 `i < num` 대신 `i * i <= num`(또는 `i <= Math.sqrt(num)`)까지만 검사해도 결과는 같으며 훨씬 빠르다.
</details>

### 참·거짓

**15.** 다음 서술이 참인지 거짓인지 판단하고, 거짓이면 이유를 밝히시오.

1. `while` 문은 조건이 처음부터 거짓이면 본문을 한 번도 실행하지 않는다.
2. 전통 `switch`에서 `break`를 빠뜨리면 컴파일 오류가 난다.
3. 중첩 반복 안에서 기본 `break` 하나로 바깥 반복까지 모두 빠져나올 수 있다.
4. `for (int i = 0; i < 5; i++)` 반복문의 본문은 5번 실행된다.
5. `do-while` 문법의 끝에는 세미콜론(`;`)이 필요하다.

<details>
<summary>정답 및 해설</summary>

1. **참**. `while`은 조건을 먼저 검사하므로 처음부터 거짓이면 본문은 0번 실행된다.
2. **거짓**. `break` 생략은 컴파일 오류가 아니라 **fall-through**를 일으킬 뿐이다(문법상 정상 코드). 논리 버그가 되기 쉬울 뿐 컴파일은 된다.
3. **거짓**. 기본 `break`는 **가장 안쪽 반복문 하나만** 탈출한다. 바깥까지 나가려면 **레이블 break**가 필요하다.
4. **참**. `i`가 0,1,2,3,4일 때 실행되어 총 5번 반복한다.
5. **참**. `do { ... } while (조건);` 처럼 끝에 세미콜론이 반드시 필요하다(빠뜨리기 쉬운 실수).
</details>

## 요약

- **제어 흐름**은 순차·조건·반복 세 가지 형태로 이루어지며, 대부분의 프로그램은 이들의 조합이다. (`ExecutionFlow`)
- **`if` 문**은 조건이 참일 때 실행한다. `if-else`, `else if` 사슬, 중첩 `if`로 확장되며, `else if`는 **먼저 참이 되는** 블록 하나만 실행하므로 순서가 중요하다. (`IfStatement`)
- **`switch` 문**은 하나의 값을 여러 `case`와 비교한다. 전통 방식은 `break`로 끊으며 생략 시 **fall-through**가 일어난다. 화살표 표현식(`->`)은 값을 돌려주고 fall-through가 없다. (`SwitchStatement`)
- **`for` 문**은 반복 횟수가 정해진 반복에 쓰며(초기화·조건·증감), 배열/컬렉션 순회에는 **for-each**가 간결하다. (`ForStatement`)
- **`while` 문**은 조건이 참인 동안 반복하고 **조건을 먼저** 검사한다. 조건을 거짓으로 만들지 않으면 무한 루프에 빠진다. (`WhileStatement`)
- **`do-while` 문**은 본문을 **먼저 실행한 뒤** 조건을 검사하므로 **최소 1회**는 실행된다. (`DoWhileStatement`)
- **`break`**는 반복을 즉시 끝내고, 기본형은 **가장 안쪽 하나만** 탈출한다. 레이블로 바깥까지 탈출할 수 있다. (`BreakStatement`)
- **`continue`**는 **이번 회차만 건너뛰고** 다음 회차로 넘어간다. `while`에서는 증감 위치에 주의해야 한다. (`ContinueStatement`)
