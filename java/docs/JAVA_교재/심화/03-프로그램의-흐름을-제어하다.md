# 3장. 프로그램의 흐름을 제어하다

> **🐳 실습 환경 — 이 장은 `java-sandbox` 컨테이너에서 실습한다**
> ```bash
> cd java && docker compose up -d              # 컨테이너 켜기 (이미 떠 있으면 생략)
> docker exec -it java-sandbox ./run.sh ControlFlow   # 예제 실행
> ```
> 처음이라면 [java/README.md](../../../README.md)의 "빠른 시작"을 먼저 보라.

지금까지 우리가 배운 프로그램은 마치 한 줄로 늘어선 도미노와 같았습니다. 첫 번째 줄부터 마지막 줄까지, 위에서 아래로 하나씩 순서대로 실행되는 것이죠. 이것을 **순차 실행(sequential execution)** 이라고 부릅니다. 컴퓨터는 기본적으로 코드를 이 순서대로만 읽습니다.

그런데 잠깐, 현실 세계는 그렇게 단순하지 않습니다.

아침에 일어나서 창밖을 봅니다. "비가 오는가?" 비가 오면 우산을 챙기고, 오지 않으면 그냥 나갑니다. 이건 **상황에 따라 다른 행동**을 하는 것입니다. 또 어떤가요? 팔굽혀펴기를 100번 한다고 생각해봅시다. 우리는 "팔을 굽힌다"는 똑같은 동작을 100번 **반복**합니다.

만약 프로그램이 오직 순차 실행만 할 수 있다면, "비가 오면"이라는 판단을 내릴 수 없고, "100번 반복"을 위해 똑같은 코드를 100번 복사해서 붙여넣어야 할 것입니다. 이건 프로그램이라고 부르기도 민망한 수준이겠죠.

그래서 우리에게는 **흐름 제어(control flow)** 가 필요합니다. 흐름 제어란 프로그램이 순차 실행이라는 직선 도로를 벗어나, **갈림길에서 방향을 선택**하거나(조건문), **같은 길을 여러 번 돌게**(반복문) 만드는 기술입니다.

이 장을 다 읽고 나면, 여러분은 프로그램에게 "만약 ~라면 이렇게 해"와 "이 조건이 만족될 때까지 계속 해"라고 명령할 수 있게 됩니다. 이것이 바로 프로그램을 진짜 '살아 움직이게' 만드는 핵심입니다. 왜 이런 문법이 필요한지, 왜 이런 형태로 발전해 왔는지 하나씩 풀어가 봅시다.

## 3.1 조건문: 갈림길에서의 선택

### 정의

**조건문(conditional statement)** 은 주어진 조건이 참(true)인지 거짓(false)인지에 따라, 서로 다른 코드를 실행하도록 만드는 문법입니다. 프로그램의 실행 흐름을 갈림길로 나누는 도구라고 생각하면 됩니다.

### 원리: 왜 조건문이 필요한가?

프로그램은 원래 위에서 아래로 한 줄씩 순차 실행됩니다. 하지만 현실의 문제는 대부분 "상황에 따라 다르게 행동해야" 합니다.

> "비가 오면 우산을 챙기고, 안 오면 그냥 나간다."

여기서 "비가 오는가?"가 바로 **조건**이고, "예/아니오"에 따라 갈라지는 두 갈래 길이 바로 조건문입니다. 조건문이 없다면 프로그램은 그저 미리 정해진 대본만 읽는 배우일 뿐, 상황을 판단하는 지능적인 존재가 될 수 없습니다.

```
        [프로그램 시작]
              │
              ▼
        ┌───────────┐
        │ 비가 오는가? │  ← 조건 판단
        └───────────┘
          예 ╱     ╲ 아니오
           ▼         ▼
      [우산 챙기기]  [그냥 나가기]
           │         │
           └────┬────┘
                ▼
          [집을 나선다]  ← 다시 흐름이 합쳐짐
```

### if-else: 인생의 분기점

가장 기본적인 조건문은 `if-else`입니다. "예/아니오"로 갈리는 모든 상황을 표현할 수 있습니다.

```java
boolean 비가온다 = true;

if (비가온다) {
    System.out.println("우산을 챙긴다");
} else {
    System.out.println("그냥 나간다");
}
```

`if` 뒤 괄호 안의 조건이 `true`이면 첫 번째 블록을, `false`이면 `else` 블록을 실행합니다. 두 블록 중 **오직 하나만** 실행된다는 점이 핵심입니다.

조건이 세 갈래 이상으로 나뉘면 `else if`를 이어 붙입니다.

```java
int 점수 = 85;

if (점수 >= 90) {
    System.out.println("A 학점");
} else if (점수 >= 80) {
    System.out.println("B 학점");   // ← 85점은 여기서 걸림
} else if (점수 >= 70) {
    System.out.println("C 학점");
} else {
    System.out.println("재수강");
}
```

여기서 중요한 원리가 하나 있습니다. `if-else if` 사슬은 **위에서부터 순서대로 검사하다가, 처음으로 참이 되는 조건을 만나면 나머지는 건너뛴다**는 것입니다. 85점은 `>= 90`은 실패하지만 `>= 80`에서 참이 되므로 "B 학점"이 출력되고, 그 아래 `>= 70`은 아예 검사조차 하지 않습니다. 그래서 조건의 순서가 매우 중요합니다.

### switch: 엘리베이터 버튼

`if-else`가 "예/아니오"의 갈림길이라면, `switch`는 **여러 개의 선택지 중 하나를 고르는** 구조입니다. 엘리베이터 버튼을 떠올려 보세요. 1층, 2층, 3층... 버튼이 여러 개 있고, 누른 버튼에 해당하는 층으로 이동합니다.

선택지가 많을 때 `if-else if`를 길게 늘어놓으면 읽기 힘듭니다. 이럴 때 `switch`가 훨씬 깔끔합니다.

```java
int 요일 = 3;

switch (요일) {
    case 1:
        System.out.println("월요일");
        break;
    case 2:
        System.out.println("화요일");
        break;
    case 3:
        System.out.println("수요일");   // ← 실행됨
        break;
    default:
        System.out.println("주말이거나 잘못된 값");
}
```

```
      switch (요일)
          │
   ┌──────┼──────┬───────┐
   ▼      ▼      ▼       ▼
 case 1  case 2  case 3  default
 (월)    (화)    (수)    (그 외)
```

`switch`는 괄호 안의 값과 일치하는 `case`로 곧장 점프합니다. 어떤 `case`에도 맞지 않으면 `default`로 갑니다.

### 전통 switch의 함정: break를 빼먹으면?

전통적인 `switch` 문에는 오래된 골칫거리가 하나 있습니다. 바로 **`break`를 빼먹으면 다음 case로 줄줄이 흘러 내려간다**는 것입니다. 이것을 **fall-through(폴 스루)** 라고 부릅니다.

```java
int 요일 = 1;

switch (요일) {
    case 1:
        System.out.println("월요일");
        // break 를 깜빡했다!
    case 2:
        System.out.println("화요일");   // 원치 않게 실행됨!
        break;
    case 3:
        System.out.println("수요일");
        break;
}
// 출력:
// 월요일
// 화요일    ← case 1로 들어왔는데 화요일까지 출력되는 버그
```

`case 1`에 `break`가 없으니, 흐름이 멈추지 않고 `case 2`의 코드까지 실행되어 버립니다. 이런 실수는 눈으로 발견하기 어려워서 악명 높은 버그의 원천이었습니다.

### Java 14+ switch expression: 왜 새로운 방식이 나왔나?

전통 switch에는 두 가지 문제가 있었습니다.

1. `break`를 빼먹으면 fall-through 버그가 생긴다.
2. 값을 하나 얻으려고 해도 코드가 장황하다.

Java는 이런 문제가 반복되자, 아예 **새로운 방식**을 만들었습니다. 이것이 Java의 발전 방식입니다. 기존 방식에 고질적인 문제가 있으면, 억지로 참지 않고 더 나은 문법을 도입합니다.

Java 14부터 정식 도입된 **switch expression(스위치 표현식)** 은 화살표 `->` 를 사용합니다.

```java
int 요일 = 3;

String 이름 = switch (요일) {
    case 1 -> "월요일";
    case 2 -> "화요일";
    case 3 -> "수요일";     // ← break 불필요, 값을 바로 반환
    default -> "주말";
};

System.out.println(이름);   // 수요일
```

무엇이 개선되었을까요?

- **`break`가 필요 없다.** 화살표 `->` 오른쪽만 실행되고 다음 case로 새지 않으므로, fall-through 버그가 원천 봉쇄됩니다.
- **값을 직접 반환한다.** `switch` 전체가 하나의 값을 만들어내는 '표현식'이 되어, 변수에 바로 대입할 수 있습니다.
- **장황함이 사라졌다.** 반복되던 `System.out.println`과 `break`가 줄어듭니다.

여러 case를 묶을 수도 있어서 더욱 편리합니다.

```java
int 월 = 4;
String 계절 = switch (월) {
    case 12, 1, 2  -> "겨울";
    case 3, 4, 5   -> "봄";      // ← 3,4,5월을 한 번에
    case 6, 7, 8   -> "여름";
    default        -> "가을";
};
```

정리하면, 기존 switch가 틀린 것은 아니지만 실수하기 쉬웠고, Java는 그 문제를 해결한 새 방식을 제공한 것입니다. 새 코드를 짤 때는 switch expression을 우선 고려하세요.

### 삼항 연산자: 아주 짧은 if-else

조건에 따라 두 값 중 하나를 고르기만 하면 될 때는, 굳이 여러 줄의 `if-else`를 쓰지 않고 **삼항 연산자(ternary operator)** 로 한 줄에 끝낼 수 있습니다.

```
조건 ? 참일_때_값 : 거짓일_때_값
```

```java
int 나이 = 20;

// if-else 로 쓰면
String 구분1;
if (나이 >= 19) {
    구분1 = "성인";
} else {
    구분1 = "미성년자";
}

// 삼항 연산자로 쓰면 한 줄
String 구분2 = (나이 >= 19) ? "성인" : "미성년자";
```

삼항 연산자는 '값을 고르는' 상황에 잘 맞습니다. 다만 조건 안에 조건을 중첩하면 급격히 읽기 어려워지므로, 복잡해지면 그냥 `if-else`로 쓰는 것이 낫습니다.

### 짧은 회로 평가(short-circuit): &&와 ||의 숨은 원리

조건을 여러 개 결합할 때 `&&`(그리고)와 `||`(또는)를 씁니다. 그런데 이 두 연산자에는 알아두면 유용한 성질이 있습니다. 바로 **짧은 회로 평가(short-circuit evaluation)** 입니다.

- `A && B` 에서, `A`가 이미 `false`라면 `B`는 아예 검사하지 않습니다. (거짓이 하나라도 있으면 전체가 거짓이니 볼 필요가 없죠.)
- `A || B` 에서, `A`가 이미 `true`라면 `B`는 아예 검사하지 않습니다. (참이 하나라도 있으면 전체가 참이니까요.)

이 성질은 단순한 최적화를 넘어, **위험한 코드를 안전하게 막는 데** 쓰입니다.

```java
String 이름 = null;

// 만약 && 가 왼쪽부터 검사하지 않았다면,
// 이름이 null인데 이름.length()를 호출해서 오류(NullPointerException)가 났을 것이다.
if (이름 != null && 이름.length() > 3) {
    System.out.println("긴 이름입니다");
}
// 이름 != null 이 false이므로, 오른쪽 이름.length()는 실행되지 않아 안전하다.
```

왼쪽 조건(`이름 != null`)이 `false`이면 오른쪽(`이름.length()`)을 건너뛰기 때문에, null인 값에 접근하는 사고를 막을 수 있습니다. 순서가 반대였다면 오류가 났을 것입니다. 그래서 "먼저 안전한지 확인하고, 그 다음에 값을 검사한다"는 순서가 중요합니다.

## 3.2 반복문: 같은 일을 반복하는 기계

### 정의

**반복문(loop)** 은 특정 코드 블록을 조건이 만족되는 동안 여러 번 반복해서 실행하는 문법입니다.

### 원리: 왜 반복문이 필요한가?

컴퓨터의 가장 큰 장점 중 하나는 **같은 일을 지치지도, 실수하지도 않고 반복**할 수 있다는 것입니다. 사람은 "1부터 1000까지 더하라"고 하면 힘들어하지만, 컴퓨터는 눈 깜짝할 사이에 해냅니다.

만약 반복문이 없다면 어떻게 될까요? "안녕"을 100번 출력하려면 `System.out.println("안녕");`을 100줄 복사해야 합니다. 상상만 해도 끔찍하죠. 반복문은 이런 반복 작업을 **단 몇 줄로 표현**하게 해주는 도구입니다.

### for: 횟수가 정해진 반복

`for` 문은 "몇 번 반복할지 횟수가 정해져 있을 때" 사용합니다. "100번 절하기"처럼요.

```java
for (int i = 0; i < 5; i++) {
    System.out.println((i + 1) + "번째 절");
}
```

`for` 문의 괄호 안은 세 부분으로 나뉩니다.

```
for ( 초기화 ; 조건 ; 증감 )
       ①       ②      ③
```

이들이 실행되는 순서를 다이어그램으로 보면 이렇습니다.

```
  ① 초기화 (int i = 0)   ← 딱 한 번만 실행
        │
        ▼
  ② 조건 확인 (i < 5) ──── false ──▶ [반복 종료]
        │ true
        ▼
     [블록 실행]
        │
        ▼
  ③ 증감 (i++)
        │
        └──────▶ 다시 ②로
```

핵심은 **초기화는 한 번만**, 그 뒤로는 **조건 확인 → 실행 → 증감**을 조건이 거짓이 될 때까지 뱅글뱅글 돈다는 것입니다. `i`가 0, 1, 2, 3, 4일 때 실행되고, `i`가 5가 되면 `i < 5`가 거짓이 되어 멈춥니다. 즉 총 5번 실행됩니다.

### while: 조건이 충족될 때까지

`while` 문은 "횟수가 아니라 조건이 중요할 때" 사용합니다. "배부를 때까지 먹기"처럼, 몇 번을 먹어야 하는지는 모르지만 "배부르다"는 조건이 될 때까지 계속하는 것이죠.

```java
int 배고픔 = 100;

while (배고픔 > 0) {          // 배고픔이 남아 있는 동안
    System.out.println("한 입 먹는다. 남은 배고픔: " + 배고픔);
    배고픔 -= 30;             // 한 입 먹을 때마다 배고픔 감소
}
System.out.println("배부르다!");
```

```
    ┌──────────────────┐
    ▼                  │
  조건 확인 (배고픔 > 0)   │
    │ true             │
    ▼                  │
 [한 입 먹기]            │
 [배고픔 감소] ───────────┘
    │ (조건이 false가 되면)
    ▼
 [반복 종료]
```

`for`와 `while`의 차이는 무엇일까요? 사실 둘은 서로 바꿔 쓸 수 있지만, **의도**가 다릅니다. `for`는 "몇 번"이라는 횟수가 명확할 때, `while`은 "어떤 상태가 될 때까지"라는 조건이 명확할 때 쓰는 것이 자연스럽습니다.

### do-while: 최소 한 번은 실행

`do-while`은 `while`과 거의 같지만, **순서가 다릅니다.**

- `while`: 조건을 **먼저 확인**하고 → 실행. (조건이 처음부터 거짓이면 한 번도 실행 안 됨)
- `do-while`: **먼저 실행**하고 → 조건 확인. (조건과 무관하게 **무조건 최소 한 번은 실행**)

"식당에 가면 배가 고프든 안 고프든 메뉴판은 일단 한 번은 본다"는 상황을 생각하면 됩니다.

```java
int 숫자;
Scanner scanner = new Scanner(System.in);

do {
    System.out.print("1~10 사이 숫자를 입력하세요: ");
    숫자 = scanner.nextInt();
} while (숫자 < 1 || 숫자 > 10);   // 잘못된 값이면 다시 물어봄

System.out.println("입력한 숫자: " + 숫자);
```

```
  ┌────────────────┐
  ▼                │
[숫자 입력받기]  ← 먼저 실행!  │
  │                │
  ▼                │
조건 확인          │
(1~10 벗어남?) ─ 예 ─┘
  │ 아니오
  ▼
[반복 종료]
```

사용자에게 입력을 받는 상황이 대표적입니다. 최소 한 번은 물어봐야 하고, 잘못된 값이면 다시 물어봐야 하니까요. 실무에서 `do-while`은 `for`나 `while`보다 훨씬 덜 쓰이지만, "무조건 한 번은 실행해야 하는" 상황에서 그 진가를 발휘합니다.

### 무한 루프: 축복이자 저주

반복문에서 가장 조심해야 할 것이 **무한 루프(infinite loop)** 입니다. 탈출 조건이 없거나 절대 거짓이 되지 않으면, 프로그램은 영원히 멈추지 않습니다.

```java
while (true) {
    System.out.println("멈추지 않아요...");   // 영원히 출력
}
```

여기서 중요한 질문은 이것입니다. **"이 무한 루프는 의도한 것인가, 실수인가?"**

- **의도한 경우:** 서버 프로그램은 일부러 무한 루프를 돕니다. "요청이 올 때까지 계속 대기하다가, 요청이 오면 처리하고, 다시 대기"를 끝없이 반복해야 하니까요. 이때는 보통 안에 탈출 조건(`break`)이나 대기 로직이 함께 있습니다.
- **실수인 경우:** 반복문의 종료 조건을 잘못 짜서 조건이 절대 거짓이 되지 않으면, 프로그램이 얼어붙습니다.

```java
int i = 0;
while (i < 5) {
    System.out.println(i);
    // i++ 를 깜빡했다! i는 영원히 0이라 i < 5는 영원히 참 → 무한 루프
}
```

그래서 반복문을 쓸 때는 항상 스스로에게 물어야 합니다. **"이 반복은 언제 끝나는가?"** 이 질문에 답할 수 없다면, 그 반복문에는 버그가 있는 것입니다.

### break와 continue: 반복의 흐름을 조종하기

반복문 안에서 흐름을 더 정교하게 조종하는 두 명령이 있습니다.

- **`break`**: 반복문을 **즉시 완전히 탈출**합니다.
- **`continue`**: 현재 회차를 **건너뛰고** 다음 반복으로 넘어갑니다. (반복문 자체는 계속됨)

```java
for (int i = 1; i <= 10; i++) {
    if (i == 5) {
        break;      // i가 5가 되면 반복문 전체 종료
    }
    System.out.println(i);
}
// 출력: 1 2 3 4  (5에서 멈춤)
```

```java
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) {
        continue;   // 짝수면 이번 회차 건너뛰기
    }
    System.out.println(i);
}
// 출력: 1 3 5 7 9  (홀수만)
```

`break`와 `continue`의 차이를 그림으로 보면 명확합니다.

```
break:                       continue:
 [반복 시작]                   [반복 시작]
     │                            │
     ▼                            ▼
  조건? ── break ──▶ [탈출]     조건? ── continue ─┐
     │                            │                │
     ▼                            ▼                │
  [나머지 실행]                 [나머지 실행]        │
     │                            │                │
     └──▶ 다음 반복               └──▶ 다음 반복 ◀───┘
                                 (나머지 실행을 건너뛰고 바로 다음으로)
```

### 중첩 반복문과 라벨(labeled) break

반복문 안에 반복문을 넣을 수 있습니다. 이를 **중첩 반복문(nested loop)** 이라고 합니다. 구구단이 대표적인 예입니다.

```java
for (int 단 = 2; 단 <= 4; 단++) {
    for (int 곱 = 1; 곱 <= 3; 곱++) {
        System.out.println(단 + " x " + 곱 + " = " + (단 * 곱));
    }
}
```

그런데 중첩 반복문 안에서 `break`를 쓰면 **가장 가까운 안쪽 반복문 하나만** 빠져나옵니다. 만약 특정 조건에서 바깥 반복문까지 한꺼번에 탈출하고 싶다면 어떻게 할까요? 이때 **라벨(label)** 을 사용합니다.

```java
outer:  // 바깥 반복문에 이름을 붙임
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        if (i * j == 4) {
            System.out.println("i=" + i + ", j=" + j + " 에서 전체 탈출");
            break outer;   // 바깥 반복문까지 한 번에 탈출
        }
        System.out.println("i=" + i + ", j=" + j);
    }
}
```

`break outer;`는 `outer`라는 이름이 붙은 반복문 전체를 빠져나갑니다. 만약 그냥 `break`였다면 안쪽 `for`만 빠져나오고 바깥 `for`는 계속 돌았을 것입니다. 라벨은 자주 쓰이지는 않지만, 깊게 중첩된 반복에서 한 번에 탈출해야 할 때 유용합니다.

## 3.3 왜 enhanced for를 선호하는가?

### 두 가지 for 문

리스트나 배열의 모든 요소를 하나씩 훑는 것을 **순회(traversal)** 라고 합니다. Java에는 순회를 위한 두 가지 `for` 문이 있습니다.

**일반 for (인덱스 기반):**

```java
List<String> list = List.of("사과", "바나나", "체리");

for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));   // 인덱스로 하나씩 꺼냄
}
```

**enhanced for (향상된 for, for-each):**

```java
for (String item : list) {
    System.out.println(item);          // 요소를 직접 받음
}
```

`for (String item : list)`는 "리스트의 각 항목(`item`)에 대하여"라고 읽습니다. 인덱스 없이, 요소 자체를 하나씩 자동으로 건네줍니다.

```
일반 for:  i=0 ─▶ list.get(0) ─▶ "사과"
           i=1 ─▶ list.get(1) ─▶ "바나나"
           i=2 ─▶ list.get(2) ─▶ "체리"
           (인덱스를 직접 다뤄야 함)

enhanced for:  list ─▶ 사과 ─▶ 바나나 ─▶ 체리
               (요소가 알아서 하나씩 들어옴)
```

### 왜 enhanced for를 선호하는가?

세 가지 이유가 있습니다.

**① 인덱스 실수를 원천 봉쇄한다.**
일반 for는 `i <= list.size()`처럼 등호 하나만 잘못 써도 범위를 벗어나는 **off-by-one 오류**가 발생합니다. enhanced for는 인덱스를 아예 다루지 않으므로 이런 실수 자체가 불가능합니다.

```java
// 흔한 실수: <= 를 쓰면 마지막에 없는 인덱스에 접근 → IndexOutOfBoundsException
for (int i = 0; i <= list.size(); i++) {   // 버그!
    System.out.println(list.get(i));
}
```

**② 의도가 명확하다.**
`for (String item : list)`는 그 자체로 "리스트의 각 항목에 대해 무언가를 한다"는 뜻이 그대로 드러납니다. 반면 일반 for는 인덱스 계산 코드에 의도가 가려집니다.

**③ 짧고 깔끔하다.**
초기화, 조건, 증감, `list.get(i)` 같은 군더더기가 사라집니다. 코드가 짧으면 읽기 쉽고, 읽기 쉬우면 버그도 줄어듭니다.

### 그렇다면 일반 for는 언제 쓰는가?

enhanced for가 항상 정답은 아닙니다. 일반 for가 꼭 필요한 상황도 있습니다.

- **인덱스 값 자체가 필요할 때** — 예: "3번째 요소"라는 번호를 함께 출력해야 할 때.
- **역순으로 순회할 때** — 뒤에서 앞으로 가는 것은 enhanced for로 표현할 수 없습니다.
- **한 칸씩이 아니라 2칸씩 건너뛰며 순회할 때.**
- **순회 도중에 요소를 삭제할 때** — 다만 이 경우에는 enhanced for도 일반 for도 위험하며, `Iterator`를 쓰는 것이 안전합니다.

```java
// 인덱스가 필요한 경우 - 일반 for가 적합
for (int i = 0; i < list.size(); i++) {
    System.out.println((i + 1) + "번: " + list.get(i));
}

// 역순 순회 - 일반 for만 가능
for (int i = list.size() - 1; i >= 0; i--) {
    System.out.println(list.get(i));
}
```

정리하면, **기본은 enhanced for를 쓰되, 인덱스가 꼭 필요하거나 특수한 순회가 필요할 때만 일반 for를 쓴다**는 원칙을 기억하세요.

## ⚠️ 흔한 오해와 함정

**1. `=`와 `==`를 헷갈린다.**
`=`는 대입, `==`는 비교입니다. `if (x = 5)`는 x에 5를 대입하려는 것이라 대부분 컴파일 오류가 납니다. 조건 비교에는 반드시 `==`를 써야 합니다.

**2. 문자열 비교에 `==`를 쓴다.**
`if (name == "홍길동")`은 두 문자열이 **같은 객체인지**를 비교하는 것이라 예상과 다르게 동작할 수 있습니다. 문자열 내용 비교는 반드시 `name.equals("홍길동")`을 쓰세요. (자세한 이유는 뒤 장에서 다룹니다.)

**3. 전통 switch에서 break를 빼먹는다.**
`break` 없는 case는 다음 case로 흘러 내려가는 fall-through가 일어납니다. 이 함정을 피하려면 새 switch expression(`->`)을 쓰는 것이 안전합니다.

**4. 반복문의 증감을 빼먹어 무한 루프에 빠진다.**
`while (i < 5)` 안에서 `i++`를 깜빡하면 프로그램이 영원히 멈추지 않습니다. 반복문을 짤 때는 항상 "언제 끝나는가?"를 확인하세요.

**5. off-by-one 오류.**
`i <= list.size()`는 존재하지 않는 인덱스에 접근합니다. 크기가 3인 리스트의 유효한 인덱스는 0, 1, 2뿐이므로 조건은 `i < size`여야 합니다.

**6. enhanced for로 리스트를 수정하려 한다.**
enhanced for로 순회하는 중에 리스트에 요소를 추가/삭제하면 `ConcurrentModificationException`이 발생할 수 있습니다. 순회 중 수정이 필요하면 `Iterator`를 사용하세요.

**7. 삼항 연산자를 남용한다.**
`a ? (b ? c : d) : (e ? f : g)`처럼 삼항을 겹치면 아무도 못 읽습니다. 복잡하면 그냥 `if-else`로 푸세요.

## 연습문제

**문제 1.** 다음 코드의 출력 결과는 무엇일까요?

```java
int x = 7;
if (x > 10) {
    System.out.println("A");
} else if (x > 5) {
    System.out.println("B");
} else {
    System.out.println("C");
}
```

<details>
<summary>힌트</summary>
if-else if는 위에서부터 검사하다가 처음 참이 되는 조건에서 멈춥니다.
</details>

<details>
<summary>해설</summary>
정답은 <b>B</b>입니다. x는 7이므로 `x > 10`은 거짓, 다음 `x > 5`는 참이 되어 "B"가 출력됩니다. 그 아래 `else`는 검사조차 하지 않습니다.
</details>

**문제 2.** 다음 전통 switch 문에서 `등급 = 'B'`일 때 무엇이 출력될까요?

```java
char 등급 = 'B';
switch (등급) {
    case 'A':
        System.out.println("우수");
    case 'B':
        System.out.println("보통");
    case 'C':
        System.out.println("노력");
        break;
    default:
        System.out.println("미분류");
}
```

<details>
<summary>힌트</summary>
case 'B'에 break가 없습니다. fall-through가 일어나면 어디까지 흘러갈까요?
</details>

<details>
<summary>해설</summary>
출력은 <b>보통</b>과 <b>노력</b> 두 줄입니다. `case 'B'`로 진입한 뒤 break가 없어 `case 'C'`까지 흘러 내려가고, 거기서 break를 만나 멈춥니다. 이것이 fall-through 함정이며, switch expression(`->`)을 쓰면 이런 문제가 없습니다.
</details>

**문제 3.** 1부터 100까지의 정수 중, 3의 배수의 합을 구하는 코드를 작성하세요. (반복문과 조건문 사용)

<details>
<summary>힌트</summary>
for 문으로 1~100을 돌면서, `i % 3 == 0`인 경우에만 합에 더하세요. `continue`를 활용해도 좋습니다.
</details>

<details>
<summary>해설</summary>

```java
int 합 = 0;
for (int i = 1; i <= 100; i++) {
    if (i % 3 == 0) {
        합 += i;
    }
}
System.out.println(합);   // 1683
```
`i % 3 == 0`은 i가 3으로 나누어떨어진다(즉 3의 배수)는 뜻입니다. 3, 6, 9, ..., 99를 모두 더하면 1683입니다.
</details>

**문제 4.** 다음 코드의 문제점을 찾고, 왜 문제인지 설명하세요.

```java
int i = 0;
while (i < 3) {
    System.out.println("반복 중");
}
```

<details>
<summary>힌트</summary>
반복문에게 물어보세요. "이 반복은 언제 끝나는가?"
</details>

<details>
<summary>해설</summary>
`i`를 증가시키는 코드(`i++`)가 없어서 `i`는 영원히 0입니다. 따라서 `i < 3`은 항상 참이 되어 <b>무한 루프</b>에 빠집니다. 반복 안에 `i++;`를 추가해야 3번 실행 후 종료됩니다. 반복문에는 반드시 종료로 이어지는 변화가 있어야 합니다.
</details>

**문제 5.** 다음 중첩 반복문에서, `i * j`가 처음으로 6이 되는 순간 두 반복문을 모두 탈출하도록 코드를 완성하세요.

```java
for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= 5; j++) {
        // 여기를 완성하세요
    }
}
```

<details>
<summary>힌트</summary>
그냥 break는 안쪽 반복문만 탈출합니다. 바깥까지 한 번에 나가려면 무엇이 필요할까요?
</details>

<details>
<summary>해설</summary>

```java
outer:
for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= 5; j++) {
        if (i * j == 6) {
            System.out.println("i=" + i + ", j=" + j);
            break outer;   // 라벨을 이용해 바깥까지 탈출
        }
    }
}
```
바깥 for에 `outer:` 라벨을 붙이고 `break outer;`를 쓰면 두 반복문을 한 번에 탈출합니다. 일반 `break`였다면 안쪽 for만 빠져나와 바깥 for가 계속 돌았을 것입니다.
</details>

## 요약

- 프로그램은 기본적으로 위에서 아래로 순차 실행되지만, **조건문**과 **반복문**을 통해 흐름을 제어할 수 있다.
- **조건문**: `if-else`는 예/아니오의 갈림길, `switch`는 여러 선택지 중 하나를 고르는 구조다. Java 14+의 switch expression(`->`)은 break 없이 값을 반환하여 fall-through 버그를 없앤다. 삼항 연산자는 값을 고를 때 유용하고, `&&`/`||`의 짧은 회로 평가는 안전한 조건 검사를 가능하게 한다.
- **반복문**: `for`는 횟수가 정해진 반복, `while`은 조건이 만족될 때까지, `do-while`은 최소 한 번은 실행해야 할 때 쓴다. `break`/`continue`와 라벨로 흐름을 세밀하게 조종할 수 있다.
- 반복문을 쓸 때는 항상 **"언제 끝나는가?"** 를 자문하여 무한 루프를 피해야 한다.
- 순회는 기본적으로 **enhanced for**를 쓰되(실수 방지·명확·간결), 인덱스가 꼭 필요하거나 역순·특수 순회일 때만 일반 for를 쓴다.
