# 마무리: 자바 학습 로드맵

> **🐳 실습 환경 — 이 장은 `java-sandbox` 컨테이너에서 실습한다**
> ```bash
> cd java && docker compose up -d              # 컨테이너 켜기 (이미 떠 있으면 생략)
> docker exec -it java-sandbox ./run.sh TwosComplement   # 예제 실행
> ```
> 처음이라면 [java/README.md](../../../README.md)의 "빠른 시작"을 먼저 보라.

> 문법은 잊어도 다시 찾으면 된다. 하지만 "왜?"를 이해하면, 어떤 언어를 배우든 빠르게 적응할 수 있다.

---

여기까지 온 여러분에게 축하를 보낸다. 이 마지막 장은 새로운 개념을 가르치지 않는다.
대신 지금까지 배운 것의 **의미를 정리**하고, 앞으로 어디로 나아가야 할지 **지도**를 그려준다.

## 지금까지 배운 것의 의미

이 책을 통해 여러분은 Java의 **"왜?"**를 이해했다.

- **왜** Java는 JVM 위에서 돌아가는가 (1장)
- **왜** 변수에 타입이 필요한가 (2장)
- **왜** 기본형과 참조형이 다른가 (2장)
- **왜** `0.1 + 0.2`가 `0.3`이 아닌가 (2장)
- **왜** Java는 항상 pass-by-value인가 (2장)
- **왜** 객체지향이 등장했는가 (4장)
- **왜** 캡슐화, 상속, 다형성이 중요한가 (4장)
- **왜** 컬렉션 프레임워크가 필요한가 (5장)
- **왜** HashMap이 평균 O(1)인가 (5장)
- **왜** 함수형 프로그래밍 요소가 추가되었는가 (6장)
- **왜** 예외처리가 필요한가 (7장)
- **왜** record와 sealed 같은 현대 기능이 생겼는가 (8장)

이것은 **문법을 외운 것이 아니라, 철학을 이해한 것**이다.

```
  이 책을 읽기 전                    이 책을 읽은 후
  ┌──────────────────┐              ┌──────────────────┐
  │  "그냥 이렇게     │              │  "이건 이래서     │
  │   쓰는 거래"      │   ────────>  │   이렇게 동작해"  │
  │                   │              │                   │
  │  점(dot)들의 나열  │              │  선(line)으로 연결 │
  └──────────────────┘              └──────────────────┘
```

전에는 흩어진 문법 지식이 **점(dot)**들이었다면, 이제는 그 점들이
"메모리 → 바이트코드 → JIT → GC"처럼 하나의 **선(line)**으로 연결되었을 것이다.

## 초급 → 중급 → 고급으로 가는 길

```
  ┌─────────────────────────────────────────────────────────┐
  │                     Java 학습 로드맵                     │
  │                                                          │
  │  [초급] 이 책의 내용                                     │
  │  ├── 기본 문법 (변수, 조건문, 반복문)                    │
  │  ├── 객체지향 기초 (클래스, 객체, 상속)                  │
  │  ├── 컬렉션 기초 (List, Set, Map)                        │
  │  ├── 예외처리 기초                                       │
  │  └── JVM 원리 (바이트코드/JIT/클래스로딩/GC 1장)         │
  │       │                                                  │
  │       v                                                  │
  │  [중급] 실전 능력                                        │
  │  ├── 제네릭 심화 (와일드카드, PECS)                      │
  │  ├── Stream API & Lambda 실전                            │
  │  ├── 멀티스레드 & 동시성 (Thread, ExecutorService)       │
  │  ├── I/O & NIO                                           │
  │  ├── 디자인 패턴 (Strategy, Observer, Factory 등)        │
  │  └── 단위 테스트 (JUnit, Mockito)                        │
  │       │                                                  │
  │       v                                                  │
  │  [고급] 깊은 이해                                        │
  │  ├── GC 튜닝 & GC 알고리즘 (G1, ZGC) 심화                │
  │  ├── 성능 최적화 & 프로파일링                            │
  │  ├── Reflection & Annotation                             │
  │  ├── Java 모듈 시스템                                    │
  │  └── 함수형 프로그래밍 심화                              │
  │       │                                                  │
  │       v                                                  │
  │  [프레임워크] 실무 진입                                  │
  │  ├── Spring / Spring Boot                                │
  │  ├── JPA / Hibernate                                     │
  │  ├── Spring Security                                     │
  │  └── 마이크로서비스 아키텍처                             │
  └─────────────────────────────────────────────────────────┘
```

이 책은 그림의 **[초급]** 전체를 다뤘다. 그리고 그 초급을 단순 문법이 아니라
**"왜?"의 관점**에서 다뤘다는 점이 중요하다. 왜냐하면 중급/고급으로 갈수록,
문법이 아니라 바로 이 **원리에 대한 이해**가 실력을 가르기 때문이다.

예를 들어, [중급]의 "제네릭 심화"는 5장의 **타입 소거**를 이해했기에 자연스럽게 이어지고,
[고급]의 "GC 튜닝"은 1장의 **세대별 GC와 STW**를 알아야 시작할 수 있다.
당신은 이미 그 토대를 놓았다.

## 추천 학습 순서

1. **이 책의 내용을 완전히 이해하라.** 개념 없이 코드를 작성하면 모래 위에 집을 짓는 것과 같다.

2. **직접 코드를 작성하라.** 읽기만 해서는 절대 프로그래밍을 배울 수 없다. 자전거 타기를 책으로만 배울 수 없는 것과 같다. 이 책 곳곳의 `./run.sh` 실행 데모를 반드시 직접 돌려보라.

3. **에러 메시지를 두려워하지 마라.** 에러 메시지는 적이 아니라 선생님이다. "여기가 잘못되었어"라고 알려주는 것이다.

4. **"왜?"를 계속 물어보라.** "이렇게 하니까 동작한다"에서 멈추지 말고, "왜 이렇게 동작하는가?"까지 파고들어야 한다. 그것이 진짜 실력이다.

5. **작은 프로젝트를 만들어보라.** 계산기, 주소록, 간단한 게임... 직접 만들면서 배우는 것이 가장 빠르다.

## 실행 데모로 개념을 몸에 새기기

이 책에서 소개한 실행 데모들을 한자리에 모았다. 개념을 읽은 뒤, 각 데모를 돌려
**"글로 읽은 것"을 "눈으로 본 것"으로** 바꿔보자.

| 데모 | 실행 | 확인할 개념 |
|------|------|-------------|
| 2의 보수 | `./run.sh TwosComplement` | 음수 저장, `(byte)300`이 44가 되는 이유 (§2.5) |
| 부동소수점 비트 | `./run.sh FloatingPointBits` | `0.1 + 0.2 != 0.3`, IEEE-754 (§2.6) |
| 값에 의한 전달 | `./run.sh PassByValue` | Java는 항상 pass-by-value (§2.8) |
| Big-O 측정 | `./run.sh BigOTiming` | n이 커질 때 실제 시간 변화 (§5.6) |
| HashMap 내부 | `./run.sh HashMapInternals` | 버킷, resize, treeify (§5.7) |
| 타입 소거 | `./run.sh TypeErasureDemo` | 런타임에 제네릭 타입이 사라짐 (§5.9) |
