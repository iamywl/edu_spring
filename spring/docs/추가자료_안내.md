# 추가 자료 안내

이 교육 프로젝트에는 학습을 돕는 보조 자료가 추가되어 있습니다. 막히거나 API를 테스트하고 싶을 때 활용하세요.

## 1. 트러블슈팅 & FAQ

- 파일: [`docs/트러블슈팅.md`](../../docs/트러블슈팅.md)
- 내용: 처음 실습할 때 자주 만나는 오류를 **증상 → 원인 → 해결** 순서로 정리했습니다.
  - Docker 미실행, 포트 충돌(8080/8081/5432/6379), 첫 빌드가 느린 이유
  - Chapter 01~03 컴파일/실행(`compile.sh` → `run.sh`)
  - DB 연결 실패(인프라를 먼저 안 띄운 경우)
  - JWT 401/403, 토큰 만료, Validation 400, JPA LazyInitialization/N+1/스키마 문제
  - 스택 트레이스 읽는 법, 로그 보는 법, `docker compose down -v`로 초기화하기
- 오류가 났을 때 가장 먼저 펼쳐보면 좋은 문서입니다.

## 2. API 요청 모음 (`.http` 파일)

curl 명령을 일일이 외우지 않아도, HTTP 요청 파일로 클릭 한 번에 API를 테스트할 수 있습니다.
HTTP API가 있는 Spring 챕터마다 `requests.http` 파일을 추가했습니다.

| 챕터 | 파일 | 포함 내용 |
|------|------|-----------|
| 05 Spring Web | [`chapter05-spring-web/requests.http`](../chapter05-spring-web/requests.http) | Todo CRUD, Validation 400 체험 |
| 06 Spring Data JPA | [`chapter06-spring-data-jpa/requests.http`](../chapter06-spring-data-jpa/requests.http) | Team/Member CRUD, 검색, 페이징 |
| 07 Spring Security | [`chapter07-spring-security/requests.http`](../chapter07-spring-security/requests.http) | 회원가입/로그인 → JWT로 보호 API 호출, 401/403 체험 |
| 09 종합 프로젝트 | [`chapter09-final-project/requests.http`](../chapter09-final-project/requests.http) | 회원가입 → 로그인 → 게시글/댓글 전체 흐름 |

### 사용법 — VS Code
1. 확장 마켓플레이스에서 **"REST Client"** (게시자: Huachao Mao) 확장을 설치합니다.
2. `requests.http` 파일을 엽니다.
3. 각 요청(`### ...`) 바로 위에 나타나는 **"Send Request"**를 클릭합니다.
4. 오른쪽에 응답(상태 코드 + 본문)이 표시됩니다.

### 사용법 — IntelliJ IDEA
1. 별도 설치 없이 `requests.http` 파일을 엽니다. (HTTP Client 내장)
2. 각 요청 왼쪽의 **초록색 ▶ 실행 버튼**을 클릭합니다.
3. 하단 Run 창에 응답이 표시됩니다.

### JWT 토큰 자동 재사용 (07, 09 챕터)
로그인 요청에 `# @name login` 주석이 붙어 있습니다.
이 덕분에 이후 보호된 요청에서 `Authorization: Bearer {{login.response.body.$.token}}` 처럼
**로그인 응답의 토큰을 자동으로 가져다 사용**합니다. (REST Client / IntelliJ HTTP Client 공통 지원)

> 만약 사용하는 도구에서 자동 참조가 동작하지 않는다면, 로그인 응답의 `token` 값을 복사해
> `Bearer ` 뒤에 직접 붙여넣어도 됩니다. (각 파일 상단 주석에 안내되어 있습니다.)

  자세한 실행 순서는 각 챕터 README와 [`docs/트러블슈팅.md`](../../docs/트러블슈팅.md) 5번 항목을 참고하세요.
- DB 접속정보: `edu` / `edu1234` / `edu_spring` (Adminer: `http://localhost:8081`)
